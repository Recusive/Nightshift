# System Design v2 — Sub-Agent Driven Architecture

## The Shift

```
v1 (current):
  Engine DECIDES  ->  Force-loads ONE operator  ->  Agent does everything
  Agent overrides  ->  Prompt guard reverts  ->  Cycles wasted

v2 (proposed):
  Engine ADVISES  ->  Brain agent DECIDES  ->  Sub-agents EXECUTE in worktrees
  Brain REVIEWS PRs  ->  Brain MERGES  ->  Brain REPORTS
```

## Core Principles

1. **Brain never writes code.** It reads, thinks, delegates, reviews, merges, reports.
2. **Sub-agents never merge.** They create PRs. Only the brain merges after review.
3. **Each sub-agent works in a git worktree.** Physical isolation. No conflicts.
4. **Zone enforcement is structural.** Project sub-agents can't touch framework. Framework sub-agents can't touch project. Enforced by worktree scope.
5. **The human's local repo stays clean.** Sub-agents work in /tmp/ worktrees. The human can work on the main repo while the daemon runs.

 These files control agent execution.
 No sub-agent can modify them, including evolve.
 Changes require: GitHub Issue with needs-human label.

 ├── .recursive/engine/daemon.sh
 ├── .recursive/engine/lib-agent.sh

 If found: reject PR, create needs-human issue.

─────────────────────────────────────────────────
 These files affect agent behavior but not safety.
 Only the evolve and audit sub-agents may modify.

   -> Brain flags [FRAMEWORK MODIFIED] in report

─────────────────────────────────────────────────
 Implementation details, not identity.

 ├── .recursive/lib/*.py
 ├── .recursive/templates/*
 ├── .recursive/scripts/*
 ├── .recursive/tests/*

## Self-Improvement Loop

```
═══════════════════════════════════════════════════════
 HOW THE AGENT EVOLVES ITSELF
═══════════════════════════════════════════════════════

Session N (brain + project sub-agents):
    │
    │  Security sub-agent finds framework issue:
    │  "daemon.sh has a PYTHONPATH shadow bug"
    │
    │  Sub-agent logs to .recursive/friction/log.md
    │  Brain sees friction in Phase 4 (report)
    │  Brain adds to handoff: "Friction: 1 entry (PYTHONPATH)"
    │
    ▼
.recursive/friction/log.md fills up
    │
    │  Next cycle, advisory engine sees:
    │    friction_entries: 1
    │    friction_has_security: true
    │  Recommends: evolve
    │
    ▼
Session N+1 (brain decides to evolve):
    │
    │  Brain launches evolve sub-agent (worktree):
    │  "Fix friction: PYTHONPATH shadow in daemon.sh"
    │
    │  Evolve sub-agent:
    │  ┌──────────────────────────────────────┐
    │  │ 1. Read friction log                 │
    │  │ 2. Hypothesis: "removing $REPO_DIR   │
    │  │    from PYTHONPATH closes bypass"    │
    │  │ 3. Implement fix                     │
    │  │ 4. make check (847 tests pass)       │
    │  │ 5. Create PR #193                    │
    │  └──────────────────────────────────────┘
    │
    │  Brain reviews PR #193:
    │  ├─ Code quality: PASS
    │  ├─ Zone compliance: PASS (Tier 2 only)
    │  ├─ Tests pass: PASS
    │  └─ Merge PR #193
    │
    │  Brain logs in decision journal:
    │  "Evolved: PYTHONPATH fix. Hypothesis: shadow bypass closed.
    │   Measure: next security scan should not find this exploit."
    │
    ▼
Session N+2, N+3, ...:
    │
    │  Signals include cooling notice:
    │  "Framework modified in session N+1: daemon.sh
    │   Monitoring for regression (2 of 3 observed)"
    │
    ▼
Better framework -> less friction -> better sessions
```

## Cost Model

```
═══════════════════════════════════════════════════════
 COST COMPARISON
═══════════════════════════════════════════════════════

v1 (single agent):
  1 Opus session x 500 turns x 12 min     = ~$0.50/cycle

v2 (sub-agent driven):
  1 Opus brain x 40 turns x 5 min         = ~$0.10
  1-2 Sonnet execution x 200 turns x 8 min = ~$0.06 each
  1-2 Sonnet review x 30 turns             = ~$0.01 each
  Total:                                   = ~$0.20-0.25/cycle

  CHEAPER because Opus only thinks, Sonnet executes.
  BETTER because each agent has fresh, focused context.
```

## Structural Invariants (bash, not prompt)

```
═══════════════════════════════════════════════════════
 CANNOT BE DISABLED BY ANY PROMPT CHANGE
═══════════════════════════════════════════════════════

1. Handoff file must exist after session
   [ -f .recursive/handoffs/LATEST.md ] || FAIL

2. Cost per session cannot exceed hard cap
   Daemon enforces, agent cannot override

3. Circuit breaker: 3 consecutive failures = STOP
   Daemon enforces, agent cannot restart

4. All .recursive/ state committed after every session
   Daemon does git add .recursive/, not agent

5. Sub-agents work in /tmp/ worktrees only
   Never modify the main working directory

6. Brain cannot push to main directly
   Only merge via gh pr merge (CI must pass)
```
