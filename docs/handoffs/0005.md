## What I Built
- **Test writing incentives** -- Three-part system to encourage agents to write tests during hardening shifts:
  1. `tests_written` counter in `Counters` TypedDict, tracked by `append_cycle_state()` via `_is_test_file()` helper in `state.py`
  2. `build_test_escalation()` in `cycle.py` injects "Test writing directive" into the prompt after `test_incentive_cycle` (default 3) cycles if no tests have been written
  3. Test bonus in `score_diff()` increased from +1 to +2 to reward test-writing cycles with higher scores
- New config option: `test_incentive_cycle` (default 3) controls when the escalation kicks in
- Files: `nightshift/types.py`, `nightshift/constants.py`, `nightshift/state.py`, `nightshift/cycle.py`, `nightshift/scoring.py`, `nightshift/config.py`, `nightshift/__init__.py`, `tests/test_nightshift.py`, `.nightshift.json.example`
- Tests: +15 new (6 state tracking, 6 escalation logic, 3 prompt integration), 170 total passing

## Decisions Made
- Test file detection (`_is_test_file`) lives in `state.py` rather than being imported from `scoring.py` to respect the dependency flow (state cannot import from scoring)
- The escalation uses `config.get("test_incentive_cycle", 3)` with a fallback default for robustness with partial configs in tests
- Test bonus increased from +1 to +2 (not +3) to incentivize without making test-only cycles automatically pass the score threshold
