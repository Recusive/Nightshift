## What I Built
- **Category balancing directive** (task #0010): new `build_category_balancing()` function in `cycle.py` that steers agents toward underrepresented fix categories. After `category_balancing_cycle` (default 3) cycles and at least 2 committed fixes, it detects which CATEGORY_ORDER categories have zero fixes and directs the agent to focus on the highest-priority unexplored one. Mentions up to two additional underrepresented categories.
- Wired into `build_prompt()` as a new "Category balancing directive" block alongside existing test/backend escalations.
- New config key `category_balancing_cycle` added to `NightshiftConfig`, `DEFAULT_CONFIG`, `_build_config()`, and `.nightshift.json.example`.
- Files changed: `nightshift/types.py`, `nightshift/constants.py`, `nightshift/cycle.py`, `nightshift/config.py`, `nightshift/__init__.py`, `.nightshift.json.example`, `tests/test_nightshift.py`
- Tests: +11 new (8 unit + 3 prompt integration), 224 total passing

## Decisions Made
- Category balancing uses the same pattern as test/backend escalations: a standalone function returns a directive string, caller passes it to `build_prompt()`. This keeps the prompt builder a pure assembler.
- Threshold requires at least 2 total fixes before detecting imbalance, avoiding false positives on early cycles with a single exploratory fix.
- The directive is informational (not a hard reject) -- the agent can still fix whatever it finds most impactful, but it gets a nudge.
