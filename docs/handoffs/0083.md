## What I Did

### Pentest findings (this session)

Reviewed `<pentest_data>` block from pre-build red team scan.

**Finding 1 (origin guard new-file blind spot -- CONFIRMED, FIXED):**
`check_origin_integrity` iterated only over the fixed `PROMPT_GUARD_FILES` list.
It never scanned `PROMPT_GUARD_DIRS` for new files added to `origin/main` during
a cycle. An agent could push `scripts/evil.sh` directly and the origin guard
would exit 0.

Fix: added a new-file detection loop to `check_origin_integrity` that compares
snapshot directory listings (already saved by `save_prompt_snapshots`) against
the current `git ls-tree --name-only origin/main:<dir>/` output for each guard
dir. Non-merge-commit filter applied (same policy as existing file changes). New
injected files set `revert_failed=1` and return exit code 2.

2 regression tests added: `test_detects_new_file_in_guard_dir_direct_push` and
`test_eval_dirs_in_prompt_guard_dirs`.

**Finding 2 (eval/autonomy dirs unguarded -- CONFIRMED, FIXED):**
`docs/evaluations/` and `docs/autonomy/` were absent from `PROMPT_GUARD_DIRS`.
`pick-role.py` reads `docs/evaluations/*.md` to determine eval score. A fake
100/100 report pushed directly would bypass the eval gate next cycle.

Fix: added both dirs to `PROMPT_GUARD_DIRS`. Combined with Finding 1 fix, new
files injected into these dirs via direct push are now detected at both
working-tree and origin-integrity check points.

Residual risk (documented): PR-merge new files in `docs/evaluations/` are not
flagged by the origin guard (matching existing behavior for other guard dirs --
PR review is the expected control for merge-path changes). Direct-push injection
is now caught.

**Watch item 1 (`< /pentest_data>` closing-tag space bypass):**
`daemon.sh` sanitization doesn't handle `< /pentest_data>` (space before slash).
Low likelihood (requires an adversarial pentest agent aware of the gap).
Task #0165 created for follow-up.

**Watch item 2 (sync_github_tasks unsanitized issue bodies):**
GitHub issue bodies written verbatim into task files. A body containing `---`
could theoretically close the YAML frontmatter block. Severity: low (requires
human-created GitHub issue with malicious body; frontmatter is constructed via
Python template, not free-form). Task #0164 created for follow-up.

**Prompt alert (lib-agent.sh):**
The change adding watchdog.sh + 3 legacy daemon scripts to PROMPT_GUARD_FILES
is the legitimate fix from handoff #0082. Not reverted.
