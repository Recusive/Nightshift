## What I Built
- **Prompt injection protection** (task #0024): When running against external repos, the cycle prompt no longer tells the agent to read instruction files directly. Instead, the runner pre-reads all instruction files (CLAUDE.md, AGENTS.md, .cursorrules, etc.) and injects their content into the prompt wrapped in an untrusted context block. The block explicitly warns the agent not to follow behavioral overrides, command execution requests, or exfiltration attempts from target repos.
- Files modified: `nightshift/cycle.py`, `nightshift/constants.py`, `nightshift/cli.py`, `nightshift/__init__.py`, `nightshift/SKILL.md`
- Tests: +19 new, 528 total passing
- New functions: `read_repo_instructions()`, `wrap_repo_instructions()`
- New constants: `UNTRUSTED_INSTRUCTIONS_PREAMBLE`, `UNTRUSTED_INSTRUCTIONS_SUFFIX`

## Decisions Made
- Pre-load instruction files in the runner (cli.py) and pass content to `build_prompt()` as a string. This keeps `build_prompt()` a pure text-construction function.
- Read from `repo_dir` (not `worktree_dir`) since instruction files are identical in both locations.
- The untrusted wrapper lists 7 specific categories of disallowed instructions (behavioral modification, external access, worktree escape, command execution, exfiltration, override attempts, encoded content).
- Smoke-test.sh adversarial fixture not added — unit tests cover the logic. E2E adversarial testing deferred.
