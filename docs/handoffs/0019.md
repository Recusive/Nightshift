## What I Built
- **Prompt self-modification guard** (task #0025): All four daemon scripts now snapshot prompt/control files before each cycle and compare after. If an agent modifies CLAUDE.md, evolve.md, or any other prompt file during a cycle, the daemon logs a full diff and writes an alert file. The next cycle injects that alert into the agent's prompt so it reviews the changes before building.
- Files modified: `scripts/lib-agent.sh`, `scripts/daemon.sh`, `scripts/daemon-review.sh`, `scripts/daemon-overseer.sh`, `scripts/daemon-strategist.sh`
- New functions in lib-agent.sh: `save_prompt_snapshots()`, `check_prompt_integrity()`, `cleanup_prompt_snapshots()`
- New constant: `PROMPT_GUARD_FILES` array (7 files monitored)
- Tests: 528 total passing (no new Python tests -- this is pure shell)

## Decisions Made
- Guard functions live in `lib-agent.sh` (shared by all daemons) rather than duplicated in each script.
- Snapshots stored in `/tmp` (via `mktemp`) to survive `git clean -fd` within a cycle.
- Alert file stored in `docs/sessions/prompt-alert.md` (gitignored dir) to persist across cycles.
- Session index entries include `[PROMPT MODIFIED]` flag for visibility.
- All 7 prompt files are monitored: CLAUDE.md, evolve.md, evolve-auto.md, review.md, overseer.md, strategist.md, harden-daemon.md.
