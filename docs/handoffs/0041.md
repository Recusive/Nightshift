## Decisions Made
- Scoring is pure computation. Each scorer takes `ShiftArtifacts` (parsed state + shift log) and returns a `DimensionScore`. No I/O in scorers -- highly testable.
- Used `dict[str, object]` for the state field in `ShiftArtifacts` because the target repo's state file may be malformed (the whole point is to score quality). This is a JSON deserialization boundary.
- Evaluation failure is non-blocking: `evaluate()` catches clone/run failures and returns zero scores. The daemon shell wrapper also suppresses errors.
- Placed evaluation.py between coordination and feature in the dependency chain (per pyproject.toml per-file-ignores pattern).
