## Decisions Made
- E2E runs BEFORE final verification. If E2E fails (tests or smoke test), feature is marked failed immediately without running lint. This saves time and gives clearer error messages.
- `infer_test_command` is separate from `infer_verify_command` (config.py) because: (a) E2E module should be self-contained, (b) adds Makefile detection which `infer_verify_command` doesn't have, (c) doesn't take a config parameter.
- Smoke test detection skips symlinks (security) and directories (robustness).
- The test command passed to `run_e2e_tests` from `build_feature` is `state["profile"]["test_runner"]` (the profiler's detection), with `infer_test_command` as fallback inside the module.
