## What I Built
- **Task #0083** (Sub-agent coordination): Built `nightshift/coordination.py` that provides pre-wave overlap detection with prompt injection and post-wave file conflict detection for Loop 2's sub-agent pipeline.
- Added `FileOverlap`, `FileConflict`, `ConflictReport` TypedDicts to `types.py`

- Eight public functions: `extract_file_references()` (regex extraction of path-like strings), `detect_overlaps()` (finds shared file refs across wave tasks), `generate_coordination_hints()` (per-task hint strings), `inject_hints()` (appends hints to work order prompts), `detect_file_conflicts()` (post-wave check for files touched by 2+ tasks), `format_conflict_report()` (human-readable output), `coordinate_wave()` (main pre-spawn entry point), `log_conflicts()` (post-wave logging entry point)
- Wired into `build_feature()` wave loop: `coordinate_wave()` before `spawn_wave()`, `log_conflicts()` after
- 47 new tests covering all functions, types, constants, state round-trip, and integration
- Marked #0086 as duplicate of #0083

## Decisions Made
- Coordination is advisory, not blocking. Overlaps generate hints in prompts; conflicts are logged but don't fail the build. The integrator already handles actual merge conflicts.
- Used lookarounds (`(?<!\w)`, `(?!\w)`) instead of `\b` in the file reference regex so dot-prefixed paths like `.github/workflows/ci.yml` are matched.
- `generate_coordination_hints()` takes only overlaps (not the full wave) since it doesn't need work order data.
- Named the test helper `_make_coord_order()` to avoid collision with the existing `_make_work_order()` helper at line 5126.
