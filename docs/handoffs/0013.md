## Decisions Made
- Decomposer reuses `execution_order()` from planner -- avoids duplicating topological sort logic.
- Work order prompts include both "done" and "blocked" JSON output schemas -- sub-agents can self-report when stuck instead of failing silently.
- Dependency context in prompts describes what predecessor tasks produce ("Assume their work is already merged") -- gives sub-agents enough context to build on prior work without needing to see the actual files.
- `_format_frameworks()` is duplicated from planner rather than extracted to a shared utility -- it's 7 lines and extracting it would create an unnecessary shared module for a trivial function. Can be refactored later if more shared utilities emerge.
