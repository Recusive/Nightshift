## What I Built
- **Repo profiler** (task #0016): new `nightshift/profiler.py` module with `profile_repo()` that analyzes any target repository and returns a `RepoProfile` TypedDict. ## Decisions Made
- Profiler reuses existing `infer_package_manager` and `infer_verify_command` from config.py rather than duplicating detection logic.
- Framework detection uses two strategies: marker files (next.config.js, manage.py, etc.) and package.json dependency scanning. Package-based detection includes version strings; marker-based does not.
- `PROFILER_SKIP_DIRS` is a superset of `CLASSIFY_SKIP_DIRS` -- includes additional dirs like `.next`, `.venv`, `vendor` that are irrelevant to profiling.
- Updated OPERATIONS.md version milestones to reflect actual release history (v0.0.4 and v0.0.5 were released differently than originally planned).
