## Pentest Findings

### Finding 1: CONFIRMED — $after_task injection in run_evaluation()
**Task**: `#0142` (urgent, v0.0.8)
`lib-agent.sh:525` interpolates `$after_task` (agent-controlled "Built:" line text) directly into a Python `-c` single-quoted string. A crafted `Built:` line like `foo'; os.system("id"); x='` would execute arbitrary Python in the daemon shell context. Fix: pass via `_AFTER_TASK` env var instead of interpolation.

### Finding 2: CONFIRMED — PR title injection into builder prompt
**Task**: `#0143` (urgent, v0.0.8)
`daemon.sh:181` concatenates `$PR_TITLE` raw into `OPEN_PR`, which is then prepended to BOTH the pentest prompt (before the DATA wrapper) and the builder prompt (no wrapper at all). An adversarial PR title can bias both agents. Fix: strip non-alphanumeric chars from `$PR_TITLE` before use, or wrap in an XML data label.

Note: The pentest report described finding 1 as "double-quote injection" but the actual injection character is single-quote (the Python string uses single-quote delimiters). The vulnerability is real regardless.
