## Decisions Made
- Planner builds a prompt (not calls an agent directly) -- follows same pattern as Loop 1 where Python assembles prompts and the CLI invokes agents. Agent integration is a separate task (#0020).
- `execution_order()` uses Kahn's algorithm (BFS topological sort) -- produces deterministic wave ordering with sorted task IDs within each wave.
- `validate_plan()` uses DFS cycle detection -- catches circular deps before they cause runtime errors in `execution_order()`.
- Scope limits (10 tasks, 50 files) are constants in `constants.py` -- configurable via code, future `.nightshift.json` integration possible.
