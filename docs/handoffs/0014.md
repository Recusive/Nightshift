## Decisions Made
- Sub-agent command building is in `subagent.py` rather than reusing `command_for_agent()` from `cycle.py` -- the Loop 2 version uses the task schema path relative to the repo dir (not the nightshift package) and doesn't need the cycle-specific schema/message path conventions.
- `spawn_wave()` runs tasks sequentially within a wave. Parallel execution is a future optimization -- the wave concept already enables parallelism at the orchestrator level.
- Agent "blocked" status is NOT retried -- only parse failures/crashes trigger retries. If the agent explicitly reports blocked, that's a deliberate decision worth respecting.
- For codex, the spawner prefers the `--output-last-message` file over stdout for parsing, since codex stdout may contain verbose logs mixed with the JSON.
