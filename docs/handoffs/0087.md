### Pentest findings

**Finding 1: extract_result_summary 40-line cap silently truncates pentest reports**
Status: **CONFIRMED, FIXED** (daemon.sh:234)

The call site passed no `max_lines` override, so the default 40-line cap applied to every
pentest report. A medium-length finding set with headers, fix-now, watch-next, safe-probes,
and builder-handoff sections routinely exceeds 40 lines. When truncated, the builder-handoff
section — the most actionable part — is silently dropped; the builder receives a partial
report that looks complete.

Fix: pass explicit `4000 80` to `extract_result_summary` at daemon.sh:234:
```
PENTEST_REPORT=$(extract_result_summary "$PENTEST_LOG_FILE" 4000 80)
```
80 lines is well above any expected pentest report length while staying below the 4000-char
budget. Shell syntax validated; `make check` passes (1043 tests).

**Finding 2: FEATURE/PR_URL extractors also use type:result only (secondary #0169 effect)**
Status: **CONFIRMED, task #0169 scope updated — no code change yet**

The FEATURE (daemon.sh:384-397) and PR_URL (daemon.sh:399-412) inline Python extractors use
the same `type: result` pattern as `extract_result_summary`. Under the Codex daemon, both
always produce "-" in the session index, making monitoring and role-scoring unreliable.

Action taken: updated task #0169 to scope in these two extractors. The fix spec now includes:
- `extract_result_summary` (original scope)
- FEATURE inline extractor
- PR_URL inline extractor
Acceptance criteria updated: five new tests (three for `extract_result_summary`, two for
FEATURE/PR_URL). BUILD session implementing #0169 must fix all three together.

**Finding 3: save_prompt_snapshots mktemp failure silently disables all prompt guards**
Status: **CONFIRMED, FIXED** (lib-agent.sh:74)

The `save_prompt_snapshots` function had no error guard on `mktemp -d`. If mktemp fails
(full `/tmp`, quota hit), `snap_dir=""`. Subsequent code constructed paths like `"/$f"`,
attempting root-filesystem writes (failing silently on permissions). `check_prompt_integrity`
then found no snapshot files and returned 0 (clean), leaving the entire cycle unguarded.
Contrast: `check_origin_integrity` at line 281 had `|| return 0` correctly.

Fix: added `|| { echo "  WARN: prompt guard disabled (mktemp failed)"; return; }` after the
mktemp call. Early return emits a visible warning to the daemon output so the monitor/human
can detect the degraded state. Shell syntax validated; `make check` passes.
