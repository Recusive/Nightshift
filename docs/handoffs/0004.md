## What I Built
- **Cycle-to-cycle state injection** -- `build_state_summary()` in `cycle.py` reads prior cycle state (category counts, paths touched, fix counters) and generates a human-readable paragraph injected into the agent prompt between "Cycle context" and "Hard limits". On cycle 1 (no prior state), the block is omitted entirely so the prompt is unchanged.
- The summary includes: fixes by category, unexplored categories (from `CATEGORY_ORDER`), paths already visited, and running totals.
- Integrated into `build_prompt()` as a "Prior cycle intelligence" block.
- Exported from `__init__.py` and added to `__all__`.
- Files: `nightshift/cycle.py`, `nightshift/__init__.py`, `tests/test_nightshift.py`
- Tests: +9 new (6 unit for `build_state_summary`, 3 integration for prompt injection), 154 passing (155 `def test_` counted by validator due to string literal)

## Decisions Made
- State summary reads from `state["category_counts"]` (aggregated by `append_cycle_state`) rather than re-parsing individual cycle fixes. This avoids double-counting and stays consistent with the state machine.
- Paths are extracted from `verification["files_touched"]` across all cycles, split to top-level directory.
- Unexplored categories are ordered by `CATEGORY_ORDER` priority (Security first, Polish last).
- The block header is "Prior cycle intelligence:" to distinguish it from the "Cycle context:" metadata block.
