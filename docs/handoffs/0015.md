## Decisions Made
- `run_test_suite()` uses subprocess.run directly (not shell.run_command) because it needs both exit code and output in a structured way -- run_capture only returns stdout, not the exit code.
- Diagnosis is file-mention-based: counts how many times each task's files appear in test output, the most-mentioned task is the suspect. Simple but effective for matching failures to tasks.
- When diagnosis cannot identify a suspect (no file mentions match), fix attempts stop immediately rather than spawning blind fix agents.
- `integrator.py` added to ruff per-file-ignores for S603/S607 (same pattern as shell.py, cycle.py, worktree.py).
