**Date**: 2026-04-05

## What I Built
- **Task #0059** (Feature summary generation): Built `nightshift/summary.py` that generates structured summaries after Loop 2 feature builds complete.
- Added `FeatureSummary` TypedDict to `types.py` with: files_created, files_modified, tests_added, total_tasks, completed_tasks, failed_tasks, patterns_detected, description
- Added `summary` field to `FeatureState` TypedDict
- `generate_feature_summary()` extracts data from wave results, deduplicates files, detects patterns (API endpoints, CLI commands, database changes, config changes, new Python modules, new test files), and generates a one-paragraph NL description
- Wired into `build_feature()` as post-verification step -- summary is generated for both completed and failed builds
- Added `_build_feature_summary()` deserializer in `feature.py` for state persistence round-trip
- Updated `format_feature_status()` to display the summary section
- 22 new tests covering: file collection, deduplication, task counts, pattern detection (API, CLI, DB, config), NL description, empty states, state round-trip

## Decisions Made
- Summary uses path-segment-set-intersection for pattern detection rather than regex -- more robust across path depths and separators
- Files appearing in both `files_created` and `files_modified` are placed only in `files_created` (creation is the stronger signal)
- Summary is generated even for failed builds -- partial progress information is valuable for debugging
