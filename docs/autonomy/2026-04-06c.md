## Dependency Eliminated

**Auth failures stop the daemon when they should pause and retry.**

Root cause: Claude CLI returns "Not logged in · Please run /login" when unauthenticated.
The daemon recorded exit code 1 and incremented CONSECUTIVE_FAILURES identically to
a code bug. With MAX_CONSECUTIVE_FAILURES=3, three auth failures tripped the circuit
breaker (confirmed: two circuit-break events at 11:48 and 11:54 in today's session index).
After the circuit breaker fired, the daemon stopped. The human had to restart it after
re-authenticating.

Fix: Added `is_auth_failure()` to `scripts/lib-agent.sh`. Parses stream-json log for
"not logged in" / "please run /login" in `type:result` events (Claude) or
`item.completed/agent_message` events (Codex). In `scripts/daemon.sh` circuit breaker,
call `is_auth_failure` before incrementing the counter. If auth failure: `notify_human`,
sleep 300s, `continue` (no counter change). The daemon survives indefinitely without
restart; the human only needs to run /login.
