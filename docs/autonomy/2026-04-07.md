## Dependency Eliminated

**Role selection silently falls back to BUILD if pick-role.py emits unexpected stdout.**

Root cause: `daemon.sh:83` used `role_output=$(python3 "$PICK_ROLE" "$REPO_DIR" 2>&1)`,
merging stdout and stderr. `tail -1` then took the last line of the merged stream to
determine SESSION_ROLE. `pick-role.py` intentionally separates concerns -- reasoning
goes to stderr, role name to stdout -- but the `2>&1` merge defeated that separation.
Any unexpected stdout line (atexit handler, uncaught exception emitted after
`sys.exit`, future library print) would overwrite SESSION_ROLE with garbage, falling
through to the `build` default silently. The daemon would run only BUILD for all
subsequent cycles, losing OVERSEE, STRATEGIZE, and ACHIEVE scheduling until a human
debugged the issue.

Fix: `pick_session_role()` now uses `mktemp` to capture stderr separately:
- `role_stdout=$(python3 "$PICK_ROLE" "$REPO_DIR" 2>"$_pick_stderr" || true)`
- `cat "$_pick_stderr"` prints reasoning to daemon output
- `SESSION_ROLE` is derived from `$role_stdout` only (stdout-clean)
- `tail -1` is kept for defense against any future extra stdout lines
- Temp file cleaned up with `rm -f`
