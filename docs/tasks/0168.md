# Add CRLF line-ending test for has_urgent_tasks frontmatter regex

The frontmatter extraction regex in `has_urgent_tasks()` in `scripts/pick-role.py`
uses `^---\n(.*?)\n---` which requires Unix (`\n`) line endings.  Task files
written with Windows-style CRLF (`\r\n`) line endings would fail to match the
frontmatter block, causing the task to be silently skipped rather than detected
as urgent.

On macOS/Linux this is low risk because task files are created by the daemon
itself using shell heredocs and Python `write_text()` calls (which default to
Unix endings).  However, the gap should be documented or closed for robustness.

Note from safety review of PR #160.
