# Production-readiness checker module

Loop 2 "Production-readiness checker" is at 0% in the tracker with no existing task. After `nightshift build` completes a feature and runs tests, there is no structured check for production-readiness beyond "tests pass."

Build a `nightshift/readiness.py` module that runs a configurable set of production-readiness checks after build completion:

- All tests pass (already done -- verify)
- No lint errors (ruff check)
- Type check passes (mypy)
- No secrets or credentials in changed files (basic pattern scan)
- No debug/print statements left in production code
- Changed files have corresponding test coverage

Wire it into `build_feature()` as a post-verification step. Return a structured `ReadinessReport` (TypedDict in types.py) with pass/fail per check and an overall verdict.

## Acceptance Criteria
- [ ] `nightshift/readiness.py` exists with `check_production_readiness()` function
- [ ] Returns `ReadinessReport` TypedDict with per-check results
- [ ] Integrated into `build_feature()` pipeline
- [ ] Configurable checks via `.nightshift.json` (which checks to enable/disable)
- [ ] Tests cover each individual check and the aggregate verdict
- [ ] `make check` passes
