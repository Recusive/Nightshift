# Add README consistency checks for shipped entry points and config

This session found README drift large enough to document a bare `nightshift`
command that the repo does not actually install, plus an outdated config
surface. The repo already validates some docs, but it does not currently guard
operator-facing README contracts.

## Acceptance Criteria
- [ ] `scripts/validate-docs.sh` or an equivalent check fails if README advertises a bare `nightshift` command without a shipped console script
- [ ] README config guidance is checked against `.nightshift.json.example`, or the README is generated/sourced so that drift cannot happen silently
- [ ] The new README consistency rule runs in an existing verification path such as `make check` or `bash scripts/validate-docs.sh`
