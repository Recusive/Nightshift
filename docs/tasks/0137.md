# Add session-end notification plumbing distinct from notify_human

Task `#0032` is blocked on live webhook validation, but the internal code path
for structured success/failure notifications is still actionable.

Nightshift already has `notification_webhook` config plus `notify_human`
best-effort escalation for exceptional conditions. The missing piece is a
separate session-end notification path that reports routine outcomes without
pretending every completion or failure needs human escalation.

## Acceptance Criteria
- [ ] Add a best-effort session-end notification path for both success and
      failure cases
- [ ] Payload includes session ID, role, exit status, duration, task or feature
      summary, and PR URL when present
- [ ] Document the session-end payload contract and add regression coverage for
      formatting plus non-crashing behavior when the webhook is unset or fails
