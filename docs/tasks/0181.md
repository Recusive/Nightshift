# Evaluate whether docs/prompt/unified.md should be deleted

## Context

PR #170 added `docs/prompt/unified.md` to `PROMPT_GUARD_FILES` to close a
guard inconsistency. The reviewer noted the file is not loaded by any active
script (0 grep matches in `scripts/`), making it purely historical.

Guarding a dead file is safe but creates minor maintenance overhead: any future
file content changes will trigger guard alerts. Deleting it would be cleaner
and remove the dead code, but should be confirmed intentional.

## Decision criteria

- Confirm with `grep -r "unified.md" scripts/ docs/prompt/` — if zero matches,
  the file is safe to delete
- Check git log to understand why it exists (was it the predecessor to evolve.md
  or the per-role prompts?)
- If confirmed dead and not referenced anywhere: delete the file AND remove it
  from PROMPT_GUARD_FILES

## Acceptance Criteria

- [ ] Research confirms file is unused (or documents why it should be kept)
- [ ] Either: file deleted + PROMPT_GUARD_FILES entry removed, OR: a comment
      added explaining why the file is retained despite not being loaded
- [ ] `make check` passes
