# Harden persist_healer_changes error paths

Code review (PR #49) flagged that `persist_healer_changes()` error paths
in `scripts/lib-agent.sh` could leave the repo on a non-main branch if
`git checkout main` fails silently. The daemon loop top does its own
`git checkout main && git reset --hard`, so this is defense-in-depth,
not a functional bug.

## Implementation

- On each error path, verify the branch state and force-reset if needed
- Consider adding a final unconditional `git checkout main` at function end
- Add cleanup of the remote branch if push succeeded but PR/merge failed

## Acceptance Criteria
- [ ] All error paths in persist_healer_changes leave repo on main branch
- [ ] Remote branch cleaned up on partial failure (push ok, PR/merge fail)
- [ ] No functional regression in happy path
