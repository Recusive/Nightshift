# Extract repo path-casing helpers if path resolution spreads beyond Nightshift artifacts

Session `#0055` added `canonical_repo_relative_path()` and
`resolve_nightshift_dir()` to `nightshift/worktree.py` to fix `Docs/` vs
`docs/` handling. That is the right place while the helpers only serve
Nightshift artifact handling, but the architecture review noted that they should
move into a dedicated utility module if more subsystems start depending on them.

## Acceptance Criteria
- [ ] Audit all callers of Nightshift path-casing helpers across `nightshift/*.py`
- [ ] If a second cross-cutting use case exists beyond worktree/runtime artifact handling, extract the shared helpers into a dedicated utility module
- [ ] `nightshift/worktree.py` keeps only worktree-specific APIs after the extraction
- [ ] Regression coverage still passes for `Docs/Nightshift` path resolution after the refactor
