# Task file frontmatter validator -- scripts/validate-tasks.sh

During task scanning this session, tasks #0024 and #0036 were found with malformed YAML frontmatter (missing or misformatted status field). Broken frontmatter causes tasks to be invisible to the queue scanner, meaning work gets silently lost.

## What to build

A `scripts/validate-tasks.sh` script that checks every task file in docs/tasks/ for valid YAML frontmatter with all required fields. Could be integrated into `make check` or run standalone.

## Checks

1. Every `docs/tasks/NNNN.md` file has YAML frontmatter (--- delimiters)
2. Required fields present: `status`, `priority`, `target`, `created`
3. `status` is one of: `pending`, `in-progress`, `done`, `blocked`
4. `priority` is one of: `urgent`, `normal`, `low`
5. `created` is a valid date (YYYY-MM-DD)
6. If `status: blocked`, then `blocked_reason` must be present
7. If `status: done`, then `completed` must be a valid date

## Acceptance Criteria
- [ ] `scripts/validate-tasks.sh` exists and runs without errors
- [ ] Checks all 7 validation rules above
- [ ] Reports specific file and field for each violation
- [ ] Exit code 0 if all valid, non-zero if any violations
- [ ] Skips README.md, GUIDE.md, .next-id, and archive/ directory
- [ ] Integrated into validate-docs.sh or standalone
