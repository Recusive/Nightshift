# Task queue cap: stop generating tasks when 50+ pending

## Problem

Generate Work step creates 1-5 tasks per session. At 8 sessions/day that's 8-40 new tasks daily. Queue is at 126+ and growing. Most are low-priority noise.

## Fix

Add to evolve-auto.md and evolve.md Step 6o:

> QUEUE CAP: If there are 50+ pending tasks, Generate Work creates 0 tasks. The queue already has enough work. Only create new tasks if you find something urgent that isn't already tracked.

Also: the overseer daemon should cull stale low-priority tasks — mark as `status: wontfix` if pending 20+ sessions with no progress.

## Acceptance Criteria

- [ ] Generate Work skips when 50+ pending tasks exist
- [ ] Overseer culls stale low-priority tasks (20+ sessions)
- [ ] Queue stabilizes or shrinks over time
- [ ] Docs updated
