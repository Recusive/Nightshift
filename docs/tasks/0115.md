# Split healer retention out of cleanup.py

`nightshift.cleanup` now handles session-log rotation, healer-log retention, and
branch pruning. The review panel flagged that the daemon-maintenance boundary is
still acceptable, but healer retention is a clear extraction point if the module
keeps growing.

## Acceptance Criteria
- [ ] Healer retention logic moves into a dedicated module without changing the public CLI/daemon behavior
- [ ] `nightshift.cleanup` keeps only log rotation and branch pruning entry points, or delegates clearly to the extracted module
- [ ] Existing healer-retention tests still pass after the extraction
- [ ] Module-map, ops docs, and package re-exports are updated to reflect the new module boundary
