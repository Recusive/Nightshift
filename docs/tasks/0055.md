# Add healer log rotation

`docs/healer/log.md` grows without bound. Each healer cycle appends observations.
Over weeks/months this will become large and slow to read.

## Implementation

Add rotation to the daemon housekeeping phase (alongside `cleanup_old_logs`):
- Keep the last N entries (e.g., 50) or last N days of observations
- Archive older entries to `docs/healer/archive/` or simply truncate
- Could be Python (like cleanup.py) or shell

## Acceptance Criteria
- [ ] Healer log entries older than threshold are rotated/archived
- [ ] Rotation runs in daemon housekeeping (all 3 looping daemons)
- [ ] Recent observations preserved for healer to read
- [ ] Tests for rotation logic
