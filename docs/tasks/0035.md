# Configurable model + effort for Nightshift product (command_for_agent)

The Nightshift product (`nightshift/cycle.py:command_for_agent()`) has hardcoded agent flags. It needs the same configurability that the Recurse daemons have via `lib-agent.sh`.

## What to fix

1. **Update Codex flags**: replace old `-c approval_policy="never" -s workspace-write` with `--dangerously-bypass-approvals-and-sandbox --json` (matching lib-agent.sh)

2. **Add model config to `.nightshift.json`**:
   ```json
   {
     "agent": "claude",
     "claude_model": "claude-opus-4-6",
     "claude_effort": "max",
     "codex_model": "o3",
     "codex_thinking": "extra_high"
   }
   ```

3. **Update `command_for_agent()` to read these from config**:
   - Claude: add `--model` and `--effort` flags from config
   - Codex: add `--model` and `-c reasoning_effort` from config
   - Defaults match what `lib-agent.sh` uses (opus, max effort)

4. **Update types**: add the new config keys to `NightshiftConfig` in `types.py` and `DEFAULT_CONFIG` in `constants.py`

5. **Update `.nightshift.json.example`** with the new keys

6. **Environment variable overrides** (same pattern as lib-agent.sh):
   - `NIGHTSHIFT_CLAUDE_MODEL`
   - `NIGHTSHIFT_CODEX_MODEL`
   - `NIGHTSHIFT_CODEX_THINKING`
   - Config file wins over env var. Env var wins over default.

## Acceptance Criteria
- [ ] `command_for_agent()` uses updated Codex flags (--full-auto or --dangerously-bypass)
- [ ] Model and effort/thinking configurable via `.nightshift.json`
- [ ] Environment variable overrides work
- [ ] Defaults match lib-agent.sh (opus, o3, extra_high, max)
- [ ] `.nightshift.json.example` updated
- [ ] types.py and constants.py updated
- [ ] Tests cover model/effort injection into command list
- [ ] `nightshift test --agent codex --dry-run` shows correct flags
