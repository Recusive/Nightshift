# Apply frontmatter-only scope to count_pending_tasks and count_stale_tasks in pick-role.py

`count_pending_tasks()` and `count_stale_tasks()` in `scripts/pick-role.py`
still read `head[:500]` of each task file and search for `^status:\s*pending`
using `re.MULTILINE`.  This means a task body containing `status: pending`
within the first 500 bytes could inflate the pending count, potentially
biasing role selection toward OVERSEE (which triggers at 50+ pending tasks).

While the immediate injection surface is narrower than the `priority: urgent`
path fixed in this session (an attacker needs `status: pending` in the body
within 500 bytes AND another frontmatter-matching condition), the same
frontmatter-extraction technique applied to `has_urgent_tasks` should be
applied here for consistency and defense in depth.

Note from code review of PR #160.
