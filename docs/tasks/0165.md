# Harden daemon.sh pentest_data tag sanitization against space-before-slash bypass

The `<pentest_data>` closing-tag sanitization in `daemon.sh` uses a regex that
does not tolerate a space before the slash (e.g., `< /pentest_data>`).  A
compromised pentest agent could craft output that appears to close the wrapper
early, causing subsequent content to land in the builder prompt as uninhibited
text.

Severity: low (requires an adversarial pentest agent that is aware of the gap
AND produces output that survives the JSON log pipeline).  No active evidence
of exploitation.

## Acceptance Criteria

- [ ] The `sed` (or equivalent) pattern used to strip the `<pentest_data>`
      wrapper in `daemon.sh` also matches `< /pentest_data>` (with a space
      before the slash)
- [ ] A shell test verifies that `echo "x < /pentest_data> y"` is correctly
      sanitized by the updated pattern (the closing tag is removed, not passed
      through)
- [ ] Test suite and `make check` remain green after the change
