# Fix XML-wrapper escape: sanitize PENTEST_REPORT before injection

## Vulnerability

`daemon.sh:244-249` injects `$PENTEST_REPORT` (raw output from the pentest
agent via `extract_result_summary()`) directly into the `<pentest_data>` XML
wrapper without sanitizing the content for the closing tag.

If the pentest agent's result text contains the literal string `</pentest_data>`
-- which occurs naturally when the agent reads and quotes `daemon.sh` source
(the file contains that exact tag as a template literal) -- the data boundary
breaks. Content after the tag lands in the builder prompt outside any protective
wrapper, treated as authoritative instructions rather than data.

This is self-demonstrating: the session that found this finding itself quoted
`daemon.sh`, likely including the closing tag in its report.

Reported by the pentest preflight agent on 2026-04-06.

## Fix

After extracting `PENTEST_REPORT` from the log, sanitize before wrapping:

```bash
PENTEST_REPORT=$(printf '%s' "$PENTEST_REPORT" | sed 's|</pentest_data>|[/pentest_data]|g')
```

Added in `daemon.sh` immediately after the
`PENTEST_REPORT=$(extract_result_summary ...)` assignment.
