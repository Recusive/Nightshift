# Open PR from previous session requires full review before merge

## Problem

When the daemon finds an open PR from a previous session, it auto-merges after checking CI. No diff review, no review panel. This bypasses the quality gate.

## Fix

Add to evolve-auto.md:

> OPEN PR RULE: When you find an open PR from a previous session, do NOT auto-merge. First: read the full diff with `gh pr diff`, run `make check` on the branch, and verify CI passes. If the PR touches `scripts/`, `docs/prompt/`, or `nightshift/` core modules, run the full review panel before merging. Only fast-merge if it is a docs-only change with < 20 lines changed.

## Acceptance Criteria

- [ ] Rule added to evolve-auto.md
- [ ] Agent reads diff before merging open PRs
- [ ] Non-trivial PRs get review panel
- [ ] Docs-only PRs can fast-merge
