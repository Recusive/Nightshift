# Feature summary generation module

Loop 2 "Feature summary generation" is at 0% in the tracker. After `nightshift build` completes a feature, there is no structured summary of what was built.

Build a `nightshift/summary.py` module that generates a feature summary after build completion. The summary should include: files created/modified, tests added, patterns detected (e.g., new API endpoints, new CLI commands), and a one-paragraph natural language description suitable for changelogs and handoffs.

Wire it into the `build_feature()` pipeline in `nightshift/feature.py` as a post-build step. Store the summary in the feature state file.

## Acceptance Criteria
- [ ] `nightshift/summary.py` module exists with `generate_feature_summary()` function
- [ ] Summary includes: files changed list, test count delta, pattern detection, NL description
- [ ] Integrated into `build_feature()` as a post-build step
- [ ] Summary stored in feature state file under a `summary` key
- [ ] Tests cover summary generation with mock build outputs
- [ ] `make check` passes
