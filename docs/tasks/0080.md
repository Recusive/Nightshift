# Unify path-based file categorization across modules

Three modules independently categorize files by path: `cycle.py` (classify_repo_dirs, high_signal_focus_paths), `profiler.py` (PROFILER_SKIP_DIRS, LANGUAGE_EXTENSIONS), and `summary.py` (pattern detection via path segments). Each uses slightly different approaches: regex patterns, path string matching, and set intersection on path segments.

Extract a shared `nightshift/paths.py` utility with:
- `categorize_file(path) -> set[str]` returning tags like "api", "cli", "test", "config", "db"
- Shared constants for directory-to-category mapping
- All three modules import from the shared utility

This eliminates duplicate keyword sets and ensures consistent categorization across Loop 1 scoring, Loop 2 profiling, and Loop 2 summary generation.

## Acceptance Criteria
- [ ] `nightshift/paths.py` exists with `categorize_file()` function
- [ ] `cycle.py`, `profiler.py`, and `summary.py` import from it (no duplicate keyword sets)
- [ ] Categorization results are identical to current behavior (no behavior change)
- [ ] Tests cover categorization for each file type
- [ ] `make check` passes
