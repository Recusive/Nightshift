# Document TOCTOU window when new injected files are detected on origin/main

When `check_origin_integrity` detects a new file injected into a guard directory
on `origin/main` (e.g., `scripts/evil.sh`), it sets `revert_failed=1` (cannot
auto-revert) and returns exit code 2, causing the daemon to abort.  However, the
injected file remains on `origin/main`.  If the daemon auto-restarts (e.g., via
the `caffeinate` tmux session), each new cycle calls `reset_repo_state` which
does `git reset --hard origin/main`, pulling the injected file to disk before
`check_origin_integrity` runs and aborts again.  The file lands on disk on every
restart iteration.

This is not exploitable through the guard itself (the guard fires before any
agent session starts), but operators may not know to kill the tmux daemon session
entirely before remediating the injected file on GitHub.
