# Auto-release: tag and publish GitHub releases from the daemon

Self-Maintaining "Auto-release" is at 0% in the tracker. Currently the agent manually runs git tag, pushes, and creates a GitHub release. This should be automated.

Build a `nightshift/release.py` module that:
1. Reads the current version from `docs/changelog/README.md` (version table)
2. Checks if all items for that version are done (cross-reference changelog status)
3. If ready: creates git tag, pushes tag, creates GitHub release with changelog body
4. Adds the release URL to the session handoff

Wire into the daemon's post-merge step (Step 11 in evolve.md) as an automated check. The agent currently does this manually -- automate the decision and execution.

## Acceptance Criteria

- [ ] `nightshift/release.py` module with `check_and_release()` function
- [ ] Reads version status from changelog README
- [ ] Creates git tag and GitHub release with proper changelog body
- [ ] Integrated into evolve.md Step 11 as automated check
- [ ] Dry-run mode that reports "would release vX.X.X" without acting
- [ ] Tests cover: version detection, ready/not-ready states, dry-run output
- [ ] `make check` passes
