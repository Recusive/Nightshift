# Backfill missing target on existing task files

`scripts/validate-tasks.sh` now makes queue-frontmatter debt explicit. Its
first run found several live task files missing `target`, including `#0073`,
`#0081`, `#0093`, `#0094`, `#0103`, `#0118`, and `#0119`. Missing targets
weaken release accounting and keep the validator from ever going green against
the real task set.

## Acceptance Criteria

- [ ] Every non-archived task file in `docs/tasks/` has a non-empty `target`
- [ ] The repaired tasks keep their intended status and priority values
- [ ] `bash scripts/validate-tasks.sh docs/tasks` no longer reports
      `target: missing`
- [ ] The follow-up handoff notes which legacy task files were backfilled
