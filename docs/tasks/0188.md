# Harden _is_valid_autonomy_file against **Date**: inside a code block

`_is_valid_autonomy_file()` checks for `**Date**:` using a simple regex with no
structural constraints. A file that embeds `**Date**:` inside a fenced code
block would pass the guard even though it is not a real report header. The same
gap exists in `_is_valid_eval_file()`.

This is a pre-existing pattern in the codebase (not introduced by PR #175), but
consistency and defense-in-depth suggest closing it.

## Acceptance criteria

- Update `_is_valid_autonomy_file()` and/or `_is_valid_eval_file()` to require
  `**Date**:` outside of a fenced code block (i.e., not between ` ``` ` markers).
- Add a test to `TestIsValidAutonomyFile` and `TestIsValidEvalFile` confirming
  that `**Date**:` inside a code block is rejected.
- `make check` passes.

Note: this is low-priority hardening. The attack surface requires a file with
`**Date**:` inside a code block AND a TOTAL line — a narrow edge case.
