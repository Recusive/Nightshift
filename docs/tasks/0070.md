# GitHub Issues sync — human creates tasks as issues, daemon converts to task files

## The problem

The human and daemon share docs/tasks/.next-id for task numbering. If the human creates a task while the daemon is running on a feature branch, both read the same .next-id and collide on merge. The worktree convention reduces this but doesn't eliminate it.

## The fix

Human creates tasks as GitHub Issues with label "task". The daemon syncs them to task files during housekeeping.

### Implementation

Add `sync_github_tasks` function to `scripts/lib-agent.sh`:

```bash
sync_github_tasks() {
    # 1. List open issues labeled "task"
    # 2. For each issue:
    #    a. Read .next-id
    #    b. Create docs/tasks/NNNN.md from issue title + body
    #    c. Map issue labels to frontmatter (priority, environment, target)
    #    d. Increment .next-id
    #    e. Close the issue with comment: "Converted to task #NNNN"
    # 3. git add + commit the new task files (on main, before builder session)
}
```

Add to daemon.sh housekeeping block (after archive_done_tasks):
```bash
sync_github_tasks
```

### Label mapping

| GitHub label | Frontmatter field |
|-------------|-------------------|
| `task` | (required — triggers sync) |
| `urgent` | `priority: urgent` |
| `low` | `priority: low` |
| `integration` | `environment: integration` |
| `loop1` / `loop2` / `self-maintaining` / `meta-prompt` | `vision_section:` |

Default: `priority: normal`, `environment: internal`

### Human workflow

```bash
# Simple task
gh issue create --title "Add dark mode" --label "task"

# Urgent task
gh issue create --title "Fix CI" --label "task,urgent"

# With details
gh issue create --title "Build webhook" --label "task,integration" --body "Description and acceptance criteria here"
```
