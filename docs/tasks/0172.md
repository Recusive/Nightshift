# Validate eval file content before accepting its score in pick-role.py

## Context

Pentest scan (session 0089) identified that `read_latest_eval_score` in
`scripts/pick-role.py` does no content-validity check beyond regex score
extraction. `docs/evaluations/` is in `PROMPT_GUARD_DIRS` but the origin
guard's `--no-merges` filter exempts merge commits. An agent that creates a
PR adding a fabricated high-score eval file (e.g.,
`docs/evaluations/0099.md` with "**Total** | **85/100**") and merges it
would silently update the eval score read by `pick-role.py` on the next
cycle.

`git clean -fd` at cycle start removes untracked eval files, but not
committed ones on `origin/main`.

## Fix

Add a content-validity check in `read_latest_eval_score` (pick-role.py)
before accepting the score. A valid eval file must contain real metadata
markers that cannot be trivially forged:

1. A run timestamp matching the expected format (`**Date**:`)
2. At least 3 dimension score rows in the table (`|` delimited)
3. The session ID in expected format

Reject files missing these markers and fall back to 0 (no score).

## Acceptance Criteria

- [ ] `read_latest_eval_score` validates that the file contains a real
      timestamp, dimension rows, and session ID before accepting its score
- [ ] Files missing validation markers return score 0 (no-eval fallback)
- [ ] Regression tests cover: valid file accepted, file with only a score
      line rejected, empty file rejected
- [ ] `make check` passes
