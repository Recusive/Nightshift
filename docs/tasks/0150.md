# Guard test_revert_failure_returns_exit_code_2 against root CI

Review advisory from PR #140 code-reviewer: the `test_revert_failure_returns_exit_code_2`
test uses `chmod -R a-w` to block the revert push. On Linux CI running as root,
`chmod a-w` does not prevent the root user from writing, so the push would succeed
and the test would see exit code 1 instead of 2, causing an intermittent failure.

## Acceptance Criteria

- [ ] The test either (a) adds a `pytest.mark.skipif(os.geteuid() == 0, reason="...") `
  guard, or (b) uses an alternative push-blocking strategy that works for root
  (e.g., remove the `origin` remote from the repo after the attacker pushes, or
  use a hook-based deny that works regardless of user)
- [ ] All 7 `TestPromptGuardOriginBlindSpot` tests still pass on non-root macOS/Linux
- [ ] `make check` passes
