# Move inline regex patterns from evaluation.py to constants.py

The REVIEW session for `nightshift/evaluation.py` (2026-04-06) left four regex
patterns inline in `score_shift_log` and `score_usefulness`:

- `r"##\s*Fix"` — detects Fixes section header
- `r"\d+\.\s+"` — detects numbered list items
- `r"Fixes committed:\s*\d+"` — detects stats section
- `r"##\s*Recommend"` — detects Recommendations section header

Per CLAUDE.md: "Regex patterns, score maps, category weights, thresholds --
these go in constants.py or a dedicated `*_patterns.py`." These patterns are
used exactly once each, adjacent to their context, so the risk is low but the
convention is not being followed.

- [ ] `evaluation.py` imports them; no inline string literals remain for these patterns
