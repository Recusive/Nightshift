# Evaluation #0001: harden Claude evaluation startup

The first real Phractal evaluation did not start cleanly with the default command. `nightshift test --agent claude` inherited `CLAUDECODE=1`, which Claude interprets as a nested session, and then failed again because the default `claude_effort=max` setting is not supported in the current Claude subscription tier. The scored run only started after manual shell and config overrides.

## Acceptance Criteria
- [ ] Evaluation/test runs sanitize nested Claude session env vars before invoking `claude`
- [ ] Unsupported Claude effort values are downgraded or validated before launch
- [ ] `python3 -m nightshift test --agent claude --cycles 1 --cycle-minutes 5` can start from a fresh Phractal clone without manual env/config edits
- [ ] Regression tests cover the env-sanitization and effort-normalization paths
