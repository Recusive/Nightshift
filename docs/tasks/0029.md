# Pin Phractal to a known-good commit

The eval system clones Phractal at HEAD. If upstream changes break Phractal's own build, every evaluation fails and the agent wastes sessions diagnosing an external repo.

Fix: pin the Phractal clone to a specific commit SHA in a config file (docs/evaluations/targets.json). The eval step uses this SHA instead of HEAD. Update the SHA manually when you want to test against newer code.

## Acceptance Criteria
- [ ] docs/evaluations/targets.json exists with Phractal repo + commit SHA
- [ ] Eval step in evolve.md uses the pinned SHA
- [ ] smoke-test.sh also uses the pinned SHA
