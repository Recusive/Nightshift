# Fix NIGHTSHIFT_PENTEST_AGENT env var interpolation in python3 -c block

## Context

Pentest-2026-04-06 (watch item): `NIGHTSHIFT_PENTEST_AGENT` is string-interpolated
directly into a `python3 -c` block at daemon.sh:373:

```bash
part_agents=['$PENTEST_AGENT', '$AGENT']
```

The learning `2026-04-06-shell-injection-env-var-pattern.md` explicitly prohibits this
pattern ("Pass agent-controlled text via env vars into `python3 -c`, never by string
interpolation"). The risk is low (attack requires pre-session env access; child processes
cannot set parent shell vars), but the code violates the documented contract.

## Fix

Replace the inline interpolation with an environment-variable approach matching the
established pattern used in other `python3 -c` calls (heredoc or `sys.argv`/`os.environ`).
Example:

```bash
PENTEST_AGENT_ARG="$PENTEST_AGENT" AGENT_ARG="$AGENT" python3 -c "
import os
part_agents = [os.environ['PENTEST_AGENT_ARG'], os.environ['AGENT_ARG']]
..."
```

## Acceptance Criteria

- [ ] daemon.sh:373 (costs tracking python3 -c block) no longer interpolates
      `$PENTEST_AGENT` and `$AGENT` directly into the Python source string
- [ ] Values passed via environment variable or sys.argv instead
- [ ] `bash -n scripts/daemon.sh` passes
- [ ] Existing cost tracking behavior is unchanged
