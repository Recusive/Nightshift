# Watch existing files in docs/prompt/feedback for modification

The prompt guard now detects new files in `docs/prompt/feedback/` (added to `PROMPT_GUARD_DIRS`), but modifications to existing files already present at snapshot time are not detected.

Currently only `README.md` exists in that directory, which is low-risk. However, if human feedback files accumulate over time, an agent modifying an existing feedback file without adding new files would not trigger the guard.

Two options:
1. Add specific feedback files to `PROMPT_GUARD_FILES` as they are created (manual, but precise)
2. Enhance the prompt guard to diff existing files in `PROMPT_GUARD_DIRS` subdirectories, not just detect new files (broader, more complex)

## Acceptance Criteria

- [ ] Modifications to existing files inside `docs/prompt/feedback/` are detected by the prompt guard
- [ ] Approach chosen and documented (either option 1 or 2 above)
- [ ] Test coverage for the detection
