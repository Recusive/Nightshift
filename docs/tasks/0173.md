# Add scripts/run.sh and scripts/test.sh to PROMPT_GUARD_FILES

## Context

Pentest scan (session 0089) noted that `scripts/run.sh` and
`scripts/test.sh` are not in `PROMPT_GUARD_FILES` in `lib-agent.sh`.
Both are thin PYTHONPATH wrappers. Modifications to these scripts would
redirect the Python module resolution path for human-initiated runs.

`PROMPT_GUARD_DIRS` covers `scripts/` for NEW files only. Low daemon
impact (daemon invokes Claude/Codex directly via `run_agent`, not through
these scripts) but human-run entry points are unprotected for modifications
to existing files.
