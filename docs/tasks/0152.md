## Context

PR #142 review noted: the ancestor+merge-commit check intentionally passes for any principal
who can create a merge commit directly on origin/main (attacker with direct write access).
The guard's PR-merge shortcircuit relies on merge commits to identify legitimate merges;
an attacker with write access could create a forged merge commit.

This is the acknowledged design boundary: the PR review sub-agents are the protection
for principals with write access. The `check_origin_integrity` guard is specifically for
the "blind spot" scenario (agent pushes without touching the working tree).

## Task

Add a test that explicitly documents this design boundary:
1. Simulate an attacker who has direct push access (can push to origin/main)
2. Have them create a `--no-ff` merge commit directly (not via PR)
3. Assert that `check_origin_integrity` returns 0 (the guard intentionally passes this)
4. Add a docstring comment to the test explaining why this is the intended behavior

This test documents the design contract, not a bug. The assertion should be that rc==0 
and the guard intentionally does not revert because this scenario requires a principal
with sufficient access to bypass PR review entirely.
