# Centralize RepoProfile defaults across fixtures and state rebuilders

Adding `dependencies` and `conventions` to `RepoProfile` required touching `nightshift/feature.py`, `tests/test_feature_build.py`, and multiple fixtures in `tests/test_nightshift.py`. That fan-out will repeat every time the profiler grows unless the schema defaults live in one place.

## Acceptance Criteria

- [ ] One shared helper or default map defines the complete `RepoProfile` shape
- [ ] `tests/test_feature_build.py` and `tests/test_nightshift.py` stop duplicating the full schema inline
- [ ] `nightshift/feature.py::_build_profile()` uses the same defaults or an equivalent single source of truth
- [ ] A regression test proves missing persisted fields still rebuild into a valid `RepoProfile`
- [ ] `make check` passes
