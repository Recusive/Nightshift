# Fix Healer — move from broken daemon.sh call to builder session step

## The bug

The healer was implemented as a separate `run_agent` call in daemon.sh (line 134) that runs between sessions. It NEVER worked. The `$(cat "$HEALER_PROMPT_FILE")` shell expansion mangles the healer prompt because it contains backticks and `$` characters in markdown code blocks. The shell interprets these as command substitution and variable expansion. No healer log files were ever created. `docs/healer/log.md` is empty after 4 sessions.

## The fix

Remove the separate healer `run_agent` call from daemon.sh. Merge the healer's observation logic into the builder's session as a step that runs before "Generate Work" (Step 6n).

The builder already has file access, task creation, and a commit flow. The healer's job (read session index, spot trends, flag avoidance, detect drift) fits naturally as a builder step.

### What to remove

- daemon.sh lines 128-143: the entire healer block (HEALER_PROMPT_FILE, run_agent, persist_healer_changes)
- lib-agent.sh: `persist_healer_changes` function (lines 471-499) — no longer needed
- lib-agent.sh: remove `docs/prompt/healer.md` from PROMPT_GUARD_FILES — it becomes a reference doc, not a control file

### What to add

Add a new step to `docs/prompt/evolve.md` BEFORE the existing "Generate Work" step:

```
STEP 6o — OBSERVE THE SYSTEM (ALWAYS)

Before generating tasks, observe the system like a human checking in:

1. Read docs/sessions/index.md — look at the last 5 entries. Any patterns?
   - Sessions getting slower or more expensive?
   - Same task appearing in handoffs repeatedly without getting done?
   - Consecutive failures?
2. Read docs/healer/log.md — your previous observations. Don't repeat yourself.
3. Check the task queue — any tasks pending 5+ sessions? Any blocked with weak reasons?
4. Check the vision tracker — has it moved? Or are sessions doing work that doesn't advance it?

Write your observations to docs/healer/log.md (append at end):

## YYYY-MM-DD — Session [feature-name]
**System health:** [good / caution / concern]
- [Observation with evidence]
- [Observation with evidence]

Think in TRENDS, not point failures. Connect dots across sessions.
```

The existing "Generate Work" step then creates tasks from these observations.

### What to keep

- docs/prompt/healer.md — keep as reference document for observation methodology
- docs/healer/log.md — keep as persistent journal (now written by builder, not separate agent)
