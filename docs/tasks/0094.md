# E2E validation: run nightshift against Phractal every N sessions

## Problem

The agent builds features and ships PRs but never actually runs nightshift against a real repo (Phractal) to verify the product works end-to-end. Every handoff says "Eval skipped." The evaluation module (evaluation.py) was built but never invoked. We are building in the dark.

## What needs to happen

1. **Wire E2E into the daemon loop.** Every 3-5 sessions (configurable via `eval_frequency` in `.nightshift.json`), the daemon should:
   - Clone `https://github.com/fazxes/Phractal.git` to `/tmp/nightshift-eval`
   - Run `nightshift test --agent claude --cycles 2 --cycle-minutes 5` against it
   - Score the results using `evaluation.py`
   - Write the score to `docs/evaluations/NNNN.md`
   - If score < 40/100, create a `priority: urgent` task and flag in the handoff
   - Clean up the clone

2. **Update evolve.md Step 0** — make evaluation non-skippable when `eval_frequency` triggers. The agent currently always writes "eval skipped" because it rationalizes that its session doesn't need it.

3. **Update evolve-auto.md** — add an EVALUATION RULE similar to the RELEASE RULE. "Evaluation is not optional when the frequency threshold is met."

4. **Update daemon.sh / lib-agent.sh** — add a cycle counter or check against the session index to determine if it's time to evaluate.

5. **Update docs/ops/OPERATIONS.md** — document the E2E validation system.

6. **Update CLAUDE.md** — mention the evaluation cadence.

7. **Verify it works** — after wiring it in, the next daemon run MUST actually produce an evaluation report in `docs/evaluations/`. If it doesn't, the task is not done.

## Acceptance Criteria

- [ ] Daemon runs E2E evaluation against Phractal every N sessions (configurable)
- [ ] Evaluation report written to `docs/evaluations/NNNN.md` with scores
- [ ] Low scores auto-create urgent tasks
- [ ] At least one real evaluation report exists after implementation
- [ ] All relevant docs updated (evolve.md, evolve-auto.md, OPERATIONS.md, CLAUDE.md, DAEMON.md)
