# Learnings verification — require quoting in status report

Learnings are written but never verified as read. An agent can ignore them and repeat the same mistakes.

Fix: in evolve.md Step 1, require the status report to include a "Learnings applied" section that quotes at least one specific learning from docs/learnings/ and explains how it affects this session's approach. This creates an auditable trail.

## Acceptance Criteria
- [ ] evolve.md Step 1 status report includes "Learnings applied" section
- [ ] Agent must quote a specific learning file and explain relevance
- [ ] Pre-push checklist verifies learnings section exists in the handoff
