# Backfill vision_section on pending task files

The new `vision_section` frontmatter field (added in #0031) enables vision-alignment checking when creating tasks. Existing pending tasks lack this field, making the alignment check incomplete until they are backfilled.

Go through all pending task files in `docs/tasks/` and add the `vision_section` field based on each task's content. Valid values: `loop1`, `loop2`, `self-maintaining`, `meta-prompt`, `none`.

## Acceptance Criteria
- [ ] All pending task files have a `vision_section` field in frontmatter
- [ ] Values accurately reflect which tracker section the task advances
- [ ] Tasks that are pure internal cleanup use `none`
