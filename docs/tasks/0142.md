# Fix shell injection in run_evaluation(): use env var for after_task

## Vulnerability

`lib-agent.sh:525` interpolates `$after_task` directly into a Python `-c` string:

```bash
result=$(PYTHONPATH="$REPO_DIR" python3 -c "
...
    after_task='$after_task',
..." 2>/dev/null)
```

`$after_task` is extracted from the agent's own session output by parsing
`Built: <name>` lines (daemon.sh:317-320), truncated to 50 chars. An agent
that outputs `Built: foo'; os.system("touch /tmp/x"); x='` would inject and
execute arbitrary Python in the daemon shell context. Single-quote is the
injection character (not double-quote as the pentest report stated, but the
exploit is equivalent).

This finding was reported by the pentest preflight agent on 2026-04-06 and
confirmed by source inspection. Related to task `#0045` (same pattern in
cleanup functions, but REPO_DIR-sourced = lower risk). This instance is
higher severity because the value is agent-controlled text.

## Fix

Pass `after_task` via environment variable instead of string interpolation:

```bash
result=$(PYTHONPATH="$REPO_DIR" _AFTER_TASK="$after_task" python3 -c "
import os, json
from pathlib import Path
from nightshift.evaluation import evaluate
from nightshift.config import merge_config

cfg = merge_config(Path('${REPO_DIR:-.}'))
r = evaluate(
    target_repo=cfg['eval_target_repo'],
    agent='$agent',
    nightshift_dir=Path('${REPO_DIR:-.}'),
    eval_dir=Path('${REPO_DIR:-.}/docs/evaluations'),
    task_dir=Path('${REPO_DIR:-.}/docs/tasks'),
    after_task=os.environ.get('_AFTER_TASK', ''),
)
..." 2>/dev/null)
```

## Acceptance Criteria

- [ ] `run_evaluation()` in `lib-agent.sh` passes `after_task` via `_AFTER_TASK` env var, not string interpolation
- [ ] `bash -n scripts/lib-agent.sh` passes
