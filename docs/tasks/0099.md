# Evaluation #0001: add a real verification command for Phractal evals

The first real evaluation ran with `verify_command: null`, so baseline verification was skipped and no repo verification ran after either cycle. That makes the evaluation score and the acceptance signal much weaker than intended.

- Re-running the Phractal evaluation records a non-skipped baseline or cycle verification step
- The standard evaluation setup is documented in `docs/evaluations/README.md` or a target metadata file it references
