# Structured CI report artifact from GitHub Actions

Blocked epic `#0103` bundles several CI/CD capabilities together. The first missing slice is still untracked: every CI run should emit one machine-readable summary artifact the daemon can inspect next session instead of scraping raw logs.

## Acceptance Criteria
- [ ] CI writes a `ci-report.json` artifact on every run, including failures
- [ ] The report captures at least: checks run, checks passed, checks failed, failure details, test count, and total duration
- [ ] `.github/workflows/ci.yml` uploads the artifact in a predictable location/name
- [ ] `docs/ops/OPERATIONS.md` documents the artifact contract for the agent
