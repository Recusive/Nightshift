# Sub-agent coordination module -- shared context and conflict detection

## Context

Loop 2 Feature Builder spawns multiple sub-agents in parallel waves via `nightshift/subagent.py`. Currently each sub-agent works independently with no awareness of what other sub-agents in the same wave are doing. The "Sub-agent coordination" tracker item is at 0%.

## What to build

A coordination module (`nightshift/coordinate.py`) that:

1. **Shared context injection**: Before spawning wave N, collect summaries of completed waves (0..N-1) and inject them into each sub-agent's prompt so they know what code already exists.
2. **Conflict detection**: After a wave completes, compare file modification lists across sub-agents in that wave. Flag files modified by 2+ sub-agents as merge conflicts. The integrator already handles merging, but early detection lets it prioritize conflict resolution.
3. **TypedDict types**: `WaveContext`, `ConflictReport` in `nightshift/types.py`.

## Acceptance criteria

- [ ] `nightshift/coordinate.py` module with `build_wave_context()` and `detect_conflicts()` functions
- [ ] Types added to `nightshift/types.py`
- [ ] `build_wave_context()` takes completed wave results and returns a prompt section summarizing what was built
- [ ] `detect_conflicts()` takes a list of sub-agent results and returns files touched by 2+ agents
- [ ] Tests for both functions (at least 8 tests)
- [ ] `make check` passes
- [ ] Module added to `__init__.py` re-exports and `scripts/install.sh`
