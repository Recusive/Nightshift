# Add .nightshift.json to PROMPT_GUARD_FILES when webhook feature is added

Pentest finding (watch, 2026-04-06): `.nightshift.json` is not in
`PROMPT_GUARD_FILES`. Currently the file only contains `verify_command`. But
if a notification webhook feature is ever added (so `notify_human` calls can
route to a configurable URL), a builder agent could add a
`notification_webhook` key pointing to an attacker URL, intercepting all
cost reports, budget alerts, and human-escalation content.

Risk is low today because there is no webhook feature. But the risk grows
as webhook capability is advertised in `docs/ops/OPERATIONS.md`.

## When to action

Action this task when any of the following happens:
- A `notification_webhook` or similar config key is added to `.nightshift.json`
- `notify_human` or equivalent is made configurable via `.nightshift.json`
- The task queue drops below 20 pending tasks (general cleanup sweep)

## Acceptance Criteria

- [ ] `.nightshift.json` and `.nightshift.json.example` are in `PROMPT_GUARD_FILES`
  in `scripts/lib-agent.sh`
- [ ] `save_prompt_snapshots` copies them correctly (they are at repo root)
- [ ] Test added: guard detects modification of `.nightshift.json` on origin/main
- [ ] `make check` passes
