# Human escalation channel — gh issue create + optional webhook

The system can block tasks and log "needs human" but has no way to actually notify the human when they're away. The human only sees the problem if they actively check the terminal or task queue.

## Implementation

Add `notify_human` function to `scripts/lib-agent.sh`:

```bash
notify_human() {
    local title="$1"
    local body="$2"
    # Always: create GitHub issue
    gh issue create --title "$title" --label "needs-human" --body "$body" 2>/dev/null || true
    # Optional: webhook (Slack/Discord/etc)
    local webhook
    webhook=$(python3 -c "import json; print(json.load(open('.nightshift.json')).get('notification_webhook',''))" 2>/dev/null || true)
    if [ -n "$webhook" ]; then
        curl -s -X POST -H 'Content-Type: application/json' -d "{\"text\":\"$title\"}" "$webhook" >/dev/null 2>&1 || true
    fi
}
```

Wire into:
- Healer (task #0046): calls `notify_human` when it spots critical patterns
- Builder: calls when a task fails 3 consecutive times
- Overseer: calls when it finds systemic issues requiring human decision

Add `notification_webhook` to `.nightshift.json.example` and `NightshiftConfig` type.

Create `needs-human` label on the GitHub repo.

## Verification

1. Run: `source scripts/lib-agent.sh && notify_human "Test notification" "This is a test from Nightshift"`
2. Check: `gh issue list --label needs-human` — should show the test issue
3. Close it: `gh issue close <number>`
4. Test webhook (if configured): check Slack/Discord for the message
5. Test without webhook: verify it doesn't error when `notification_webhook` is absent
