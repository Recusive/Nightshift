# Clean up non-ASCII box-drawing characters in shell scripts

CLAUDE.md rule: "No emojis, Unicode, or non-ASCII in `.py`, `.sh`, `.toml` files." All shell scripts in `scripts/` use U+2500 BOX DRAWINGS LIGHT HORIZONTAL (`---`) for section headers. This is a pre-existing pattern that predates the rule and has been extended by subsequent changes.

Flagged by code review on PR #34.

## Acceptance Criteria
- [ ] Replace all non-ASCII section dividers in `scripts/*.sh` with ASCII equivalents (e.g., `# ----------`)
- [ ] Verify no non-ASCII characters remain in any `.sh` file: `grep -P '[^\x00-\x7F]' scripts/*.sh`
- [ ] All bash -n checks still pass
