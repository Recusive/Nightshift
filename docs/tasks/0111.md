# Module map should distinguish late imports from hard dependencies

The current AST scan treats any `nightshift.*` import anywhere in a module as a
hard dependency. If function-local late imports become common, the published
dependency order could drift away from the true import-time graph.

## Acceptance Criteria
- [ ] Module-map dependency analysis either ignores function-local imports or labels them separately from import-time dependencies
- [ ] The published dependency order matches import-time dependencies for representative late-import cases
- [ ] Regression tests cover a module that imports `nightshift.*` inside a function body
