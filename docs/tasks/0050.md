# Prompt self-refinement — analyze which instructions produce good sessions

The evolve prompt is 470+ lines and growing. Some instructions drive high-quality sessions, some get ignored, some cause confusion. The system has no way to measure prompt effectiveness or propose refinements.

## Design

The Strategist daemon (runs once, analyzes everything) gets a new analysis:

1. Read the last 10 session logs
2. For each session: what was the tracker delta? cost? time? did it skip tasks? did CI fail?
3. Correlate with which prompt sections were relevant (e.g., sessions that touched shell scripts — was the "shell portability" instruction followed?)
4. Identify: which instructions consistently produce good outcomes? which get ignored?
5. Propose specific prompt edits (add, remove, reword) with evidence

Output: a section in the strategist report under "Prompt Health" with specific recommendations. The human approves or rejects each one. Approved changes become tasks for the builder.

## Verification

1. Run the strategist: `make strategist`
2. Check the report includes a "Prompt Health" section
3. Verify recommendations reference specific prompt lines and session evidence
4. Verify recommendations are actionable (not "improve the prompt" but "remove lines 45-50 of evolve-auto.md because no session in the last 10 followed this instruction")

## Acceptance Criteria

- [ ] Strategist prompt updated with prompt health analysis
- [ ] Analyzes last N sessions for prompt effectiveness
- [ ] Recommendations reference specific lines with evidence
- [ ] Human approves/rejects via task queue (not auto-applied)
- [ ] Documented in DAEMON.md under Strategist section
