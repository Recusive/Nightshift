# Add path traversal guard to readiness scanner

Safety reviewer advisory: `readiness.py` trusts that `rel_path` values from `FeatureState` are relative paths within `repo_dir`. A malicious or buggy state file could contain paths like `../../etc/passwd` that escape the repo directory.

Add a guard in `check_secrets()`, `check_debug_prints()`, and `check_test_coverage()` that verifies `abs_path.resolve().is_relative_to(repo_dir.resolve())` before reading the file.

## Acceptance Criteria
- [ ] All file-reading paths in `readiness.py` verify the resolved path is inside `repo_dir`
- [ ] Tests cover path traversal attempts (e.g., `../../../etc/passwd`)
- [ ] `make check` passes
