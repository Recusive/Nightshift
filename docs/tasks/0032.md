# Add monitoring/alerting webhook

No one knows if a shift succeeded or failed without manually checking logs. No notifications.

Fix: add a notify() function that fires a webhook (configurable URL in .nightshift.json) at session end with a JSON payload: session ID, exit code, duration, feature built, PR URL. Support Slack, Discord, ntfy, or any generic webhook.

## Acceptance Criteria
- [ ] notify_webhook key in .nightshift.json.example
- [ ] nightshift/notify.py module with send_notification()
- [ ] Called at daemon session end (success and failure)
- [ ] Works with Slack incoming webhooks

session-end notification behavior rather than generic webhook support.
