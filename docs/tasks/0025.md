# Protect against self-modification of prompts

The agent can edit evolve.md, evolve-auto.md, and CLAUDE.md — the files that control its own behavior. One bad session can permanently corrupt all future sessions. The daemon re-reads these files each cycle.

Fix: add a checksum step in daemon.sh. Before each cycle, hash all prompt files. After the cycle, compare. If prompts changed, log a warning and require the NEXT session to verify the changes are benign before proceeding. Optionally: make prompt files read-only during daemon execution.

## Acceptance Criteria
- [ ] daemon.sh checksums prompt files before and after each cycle
- [ ] Changed prompts are logged with a diff
- [ ] Next session reviews prompt changes before building
