# Sub-agent coordination module for Loop 2

Loop 2 "Sub-agent coordination" is at 0% in the tracker. Currently, sub-agents in the same wave run independently with no awareness of each other. When parallel tasks touch overlapping files, the integrator discovers conflicts only after all agents finish.

Build a `nightshift/coordination.py` module that:

1. Before spawning a wave, analyzes estimated_files and acceptance_criteria to detect potential overlap
2. Generates coordination hints for each sub-agent (e.g., "Task 3 is also modifying src/api/routes.py -- coordinate your changes")
3. Injects these hints into the work order prompt via the decomposer
4. After wave completion, checks for file conflicts between completed tasks before integration

This does NOT require inter-agent communication. It is a pre-spawn analysis + post-spawn validation step.

## Acceptance Criteria
- [ ] `nightshift/coordination.py` exists with `detect_overlaps()` and `generate_coordination_hints()` functions
- [ ] Hints are injected into work order prompts for overlapping tasks
- [ ] Post-wave file conflict detection before integration
- [ ] Tests cover overlap detection, hint generation, and conflict detection
- [ ] `make check` passes
