# E2E test runner module for Loop 2

Loop 2 "E2E test runner" is at 0% in the tracker. After `nightshift build` completes a feature via sub-agents and wave integration, there is no automated end-to-end verification that the built feature actually works as a cohesive whole.

Build a `nightshift/e2e.py` module that runs end-to-end tests after all waves complete but before the feature is marked as done. The E2E runner should:
1. Detect the project's test command (reuse `infer_lint_command` pattern from feature.py, but for tests)
2. Run the full test suite and verify it passes
3. Optionally run a smoke test script if one exists (e.g., `scripts/smoke-test.sh`)
4. Report a structured pass/fail result with details

Wire it into `build_feature()` in `nightshift/feature.py` between wave execution and the final verification step.

## Acceptance Criteria

- [ ] `nightshift/e2e.py` module exists with `run_e2e_tests()` function
- [ ] Detects test command from project config (pytest, npm test, make test, etc.)
- [ ] Runs detected tests and returns structured result (pass/fail, output, duration)
- [ ] Integrated into `build_feature()` pipeline
- [ ] Tests cover: test command detection, pass/fail scenarios, missing test command
- [ ] `make check` passes
