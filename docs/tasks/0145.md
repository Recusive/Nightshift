# Train lightweight ML model on daemon session history for adaptive decision-making

## Vision

Train a small model on the daemon's own operational history to replace static scoring thresholds with learned predictions. The repo IS the training data — every PR, handoff, eval report, and session log is structured, timestamped, and linked.

## Training Data Sources

### Code quality
- File change co-occurrence (dependency patterns)
- CI failure rates per module (fragile paths)
- Module growth rates (complexity debt)
- Test coverage per module

### Process
- Task completion: 1 session vs 3+ attempts
- Handoff recommendation follow-through
- Learnings applied vs ignored
- Pentest finding accuracy (real vs false positive)

### Agent behavior
- Token usage per file read (efficiency)
- make check retry count per session (iteration quality)
- Prompt adherence by role
- Review comment → fix rate

### Decision outcomes
- Role picked vs hindsight-optimal role
- Task picked vs eval-score-maximizing task
- Session cost vs outcome value

## Architecture

- Small classifier/regressor, not an LLM
- Input: system state vector (eval score, pending count, consecutive builds, healer status, etc.)
- Output: predicted outcome score per role
- Replace pick-role.py static thresholds with model predictions
- Retrain weekly on accumulated session data

## Prerequisites

- 100+ completed sessions (need enough data for train/test split)
- Structured extraction of all signals into a training dataset
- A/B testing framework to compare model vs static scoring

## Acceptance Criteria

- [ ] Training data extraction script produces a clean dataset from repo history
- [ ] Model trained and evaluated on held-out sessions
- [ ] Model predictions outperform static pick-role.py on backtesting
- [ ] Integration into pick-role.py with fallback to static rules
- [ ] Weekly retraining pipeline

## When

After 100+ sessions. Don't build this prematurely — the static scoring works and we need data first.
