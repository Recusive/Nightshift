# Codebase world model — persistent MODULE_MAP.md for instant orientation

Each session re-reads the codebase from scratch to understand what exists. At 21 modules this takes ~20 tool calls. At 50+ modules it'll eat significant context. A persistent map that the builder updates after each session would give instant orientation.

## Design

Create `docs/architecture/MODULE_MAP.md`:

```markdown
# Module Map
Last updated: 2026-04-05 by session 11

## Modules (21)
| Module | Lines | Purpose | Key functions | Last changed |
|--------|-------|---------|---------------|-------------|
| types.py | 280 | TypedDicts | NightshiftConfig, ShiftState, ... | PR #41 |
| constants.py | 650 | Config defaults | DEFAULT_CONFIG, MODEL_PRICING | PR #41 |
...

## Dependency chain
types -> constants -> errors -> shell -> config/state -> ...

## Recent changes (last 5 sessions)
- Session 11: added compact.py (handoff compaction)
- Session 10: added Codex pricing to costs.py
...
```

The builder updates this in Step 6 (docs phase) — add/modify rows for any modules it changed. The Healer checks it's not stale. New sessions read this INSTEAD of scanning every module file.

## Verification

1. Check: `docs/architecture/MODULE_MAP.md` exists with all 21+ modules listed
2. Run: `wc -l nightshift/*.py` and compare against the "Lines" column — should match
3. Run a builder session — verify it updates the map for files it changed
4. Check the dependency chain matches CLAUDE.md's documented flow

## Acceptance Criteria

- [ ] `docs/architecture/MODULE_MAP.md` exists with all modules
- [ ] Each module has: lines, purpose, key functions, last changed PR
- [ ] Dependency chain documented
- [ ] Recent changes section (last 5 sessions)
- [ ] Builder updates it in Step 6 (add to evolve.md checklist)
- [ ] evolve.md Step 1 reads MODULE_MAP.md for orientation
- [ ] Healer flags if MODULE_MAP.md is stale (>5 sessions without update)
