# Evaluation #0001: normalize shift-log paths on case-insensitive filesystems

The Phractal evaluation wrote real shift-log updates, but committed them under `Docs/Nightshift/...` instead of `docs/Nightshift/...`. On a case-insensitive filesystem that looked correct to the agent, but `verify_cycle()` still rejected both cycles with `No commit in this cycle includes the shift log update.`

## Acceptance Criteria
- [ ] Shift-log verification accepts the committed path when `docs/` and `Docs/` resolve to the same file on disk, or the runner canonicalizes the path before prompting/verifying
- [ ] Rejected-cycle reproduction from Evaluation #0001 passes once the fix lands
- [ ] Regression tests cover the case-insensitive `docs/` vs `Docs/` scenario
