# Cross-session cost intelligence — learn which tasks cost what

The system tracks costs per session but doesn't learn from the data. Which task types are expensive ($15 for new modules) vs cheap ($3 for doc updates)? Which model produces better results per dollar? This data exists in session logs and the cost ledger — it's just never analyzed.

## Design

Add a `cost_analysis` function to `nightshift/costs.py` that:

1. Reads the cost ledger and session index
2. Correlates: task type (feat/fix/docs/refactor) -> average cost, average duration
3. Correlates: model used -> cost-per-test-added, cost-per-tracker-delta
4. Identifies outliers: sessions that cost 2x+ the average for their task type
5. Returns a CostAnalysis TypedDict with recommendations

The Healer (task #0046) calls this between sessions and includes cost trends in its observations. The Strategist includes it in its report.

## Verification

1. Run: `python3 -c "from nightshift.costs import cost_analysis; print(cost_analysis('docs/sessions'))"` 
2. Verify it returns task-type breakdowns with average costs
3. Verify outlier detection flags sessions that were abnormally expensive
4. Run the healer — verify it mentions cost trends in its log
