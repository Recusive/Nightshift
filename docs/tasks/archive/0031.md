# Task queue vision-alignment check

The task queue can drift from the vision — all tasks might target Loop 1 while Loop 2 stays at 0%. The agent optimizes what it already does well.

Fix: when creating follow-up tasks, check docs/vision-tracker/TRACKER.md. If the last N tasks (configurable, default 5) all target the same section, force the next task to target a different section. Add this check to docs/tasks/GUIDE.md as a rule for task creation.

## Acceptance Criteria
- [ ] GUIDE.md includes vision-alignment rule
- [ ] Agent checks tracker before creating tasks
- [ ] Consecutive same-section tasks get flagged
