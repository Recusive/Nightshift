# Evaluation clean-state scoring should detect dirty clones

This session's fresh-clone Phractal run ended with `git status --short`
showing `?? Docs/Nightshift/`, but the automated evaluation scorer still
awarded Clean state `10/10` because it only looked at runner exit status and
state counters. That makes cleanup regressions easy to miss unless a human
manually inspects the clone.

## Acceptance Criteria

- [ ] The evaluation pipeline records post-run `git status --short` (or an
      equivalent dirty/clean signal) in its parsed artifacts
- [ ] `score_clean_state()` penalizes dirty clones even when the runner exits `0`
- [ ] Regression tests cover a dirty-clone artifact with a successful runner exit
- [ ] A re-run real evaluation reports a Clean state score that matches the
      clone's actual cleanliness
