# Add cost tracking and budget ceiling

No visibility into token spend. A runaway overnight session could burn hundreds of dollars. The daemon has no budget limit.

Fix: after each session, parse the session log for token counts (stream-json output includes usage data). Accumulate in docs/sessions/costs.json. Add a --budget flag to daemon.sh that halts when cumulative spend exceeds the limit. Log cost per session in the session index.

## Acceptance Criteria
- [ ] Token usage extracted from session logs
- [ ] Cumulative cost tracked in docs/sessions/costs.json
- [ ] --budget flag on daemon.sh (default: unlimited)
- [ ] Session index includes token count column
