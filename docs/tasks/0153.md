## Context

PR #142 review noted: `origin_tmp=$(mktemp) || return 0` in `check_origin_integrity`
silently swallows tmpfs exhaustion or permission failures. The comment documents the
intent (skip check to avoid false positives) but the daemon log shows nothing, making
it hard to diagnose in production when the guard silently stops working.

## Task

Add a stderr warning before the silent `return 0` so the daemon log records the failure:

```bash
origin_tmp=$(mktemp) || {
    echo "  WARNING: check_origin_integrity: mktemp failed -- origin guard skipped" >&2
    return 0
}
```

This keeps the existing behavior (no false positives on tmpfs failure) but adds
visibility for operations.

## Acceptance Criteria

- [ ] `check_origin_integrity` emits a warning to stderr when `mktemp` fails
- [ ] Existing behavior unchanged: function returns 0 when mktemp fails
- [ ] New test verifies the warning appears when mktemp is unavailable (mock it if needed)
- [ ] `make check` passes
