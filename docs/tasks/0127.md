# Keep task-frontmatter parsing aligned across queue scripts

The architecture review for task `#0058` passed, but it noted that
`scripts/list-tasks.sh` and `scripts/validate-tasks.sh` now each carry their
own frontmatter parser. That is acceptable today, but the rules can drift if
future task metadata changes only update one script.

## Acceptance Criteria

- [ ] Either extract a shared implementation path for task-frontmatter parsing
      or add mirrored regression coverage that proves both scripts agree on the
      same malformed-task cases
- [ ] The shared or mirrored coverage includes malformed frontmatter and
      missing required fields
- [ ] `make check` passes after the alignment work
