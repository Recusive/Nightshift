# Evaluation scoring should read rejected-cycle artifacts

The first real evaluation found a second gap after the human-readable output issues: the evaluation parser/scorers still undercount rejected runs. `parse_shift_artifacts()` looks for legacy `SHIFT-LOG*.md` files instead of dated Nightshift logs, and the discovery/fix-quality/usefulness scorers ignore nested `cycle_result` data from rejected cycles.

## Acceptance Criteria
- [ ] `parse_shift_artifacts()` recognizes the current dated Nightshift log filenames
- [ ] Evaluation scorers fall back to rejected-cycle `cycle_result` data when counters stay at zero
- [ ] Regression tests cover a rejected-cycle state file with nested fixes/issues
- [ ] Re-running the same Phractal evaluation improves report fidelity without inflating scores for empty runs
