# Add safety comment to notify_human payload python3 -c call

## Context

PR #176 code review advisory: the `notify_human` function in `lib-agent.sh` has a
remaining `python3 -c "..."` call for the webhook payload:

```bash
payload=$(python3 -c "
import json, sys
print(json.dumps({'text': '[Nightshift] ' + sys.argv[1]}))
" "$title" 2>/dev/null) || payload='{"text":"[Nightshift] notification"}'
```

This is **safe** — `$title` is passed as `sys.argv[1]` (separate shell argument), not
interpolated into the Python source string. However, it looks like an unconverted call
to a reviewer or automated scanner doing `grep 'python3 -c'`.

A one-line comment would prevent future false-positives in both human review and
pentest scans.

## Acceptance Criteria

- Add a comment before the `payload=$(python3 -c ...)` block explaining:
  `# safe: $title is passed as sys.argv[1], not interpolated into the -c string`
- `bash -n scripts/lib-agent.sh` passes
- No other changes needed
