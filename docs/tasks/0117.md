# E2E eval runs against Nightshift repo instead of Phractal clone

## Bug

The E2E evaluation clones Phractal to `/tmp/nightshift-eval` but then runs `nightshift test` without `--repo-dir` or `cd`ing into the clone. Since `nightshift test` defaults to `os.getcwd()`, it runs against the Nightshift repo itself, not Phractal.

## Root cause

The agent's eval command is:
```bash
git clone Phractal /tmp/nightshift-eval && PYTHONPATH=... python3 -m nightshift test --agent claude --cycles 2
```

Missing: `--repo-dir /tmp/nightshift-eval` or `cd /tmp/nightshift-eval &&` before the test command.

## Fix

Update `docs/prompt/evolve.md` Step 0 to be explicit about the `--repo-dir` flag:

```bash
git clone --depth 1 https://github.com/fazxes/Phractal.git /tmp/nightshift-eval
PYTHONPATH=$(pwd) python3 -m nightshift test --agent claude --cycles 2 --cycle-minutes 5 --repo-dir /tmp/nightshift-eval
```

This is a prompt-only fix — hot reloads next cycle, no daemon restart needed.
