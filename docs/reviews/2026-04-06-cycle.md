## Findings
1. Invalid UTF-8 repo instruction files could raise `UnicodeDecodeError` and abort cycle setup. Added defensive skipping with an explicit warning so malformed target-repo instructions do not crash the runner.
2. Total instruction-budget truncation could exceed `MAX_INSTRUCTION_TOTAL_BYTES` when the remaining budget was smaller than the warning footer. Added byte-budgeted truncation helpers so the injected instruction content now stays within the configured cap.
3. High-signal focus filtering only skipped exact hot prefixes, so a hot file nested under a candidate directory still surfaced that directory as a recommended focus area. Tightened overlap detection to exclude both parent and child path overlaps.
4. Structured `tests_run` validation missed shell-wrapped forbidden commands and over-flagged file-scoped package-manager commands. Normalized wrapped commands and limited matching to exact or note-suffixed repo-wide commands.

## Skipped
