# v0.0.2 — Control Plane

**Released**: TBD
**Tag**: `v0.0.2`
**Status**: Merged to main. Ready to tag and release.

Replaces bash orchestration with a Python control plane. Adds runner-enforced guard rails, structured state, and pluggable agent adapters. Nightshift can now use Codex or Claude — same pipeline, you just pick.

## Added

- `[feat]` **nightshift/ package** — Python orchestrator replacing the monolithic `nightshift.py`. 8 modules + types:
  - `types.py` — TypedDicts for all data structures (strict typing)
  - `constants.py` — `DATA_VERSION`, `DEFAULT_CONFIG`, `SHIFT_LOG_TEMPLATE`, utilities
  - `errors.py` — `NightshiftError`
  - `shell.py` — `run_command`, `git`, `command_exists`, `run_shell_string`
  - `config.py` — `merge_config`, `resolve_agent`, `infer_package_manager`, `infer_verify_command`
  - `state.py` — `read_state`, `write_json`, `load_json`, `append_cycle_state`, `top_path`
  - `worktree.py` — `ensure_worktree`, `ensure_shift_log`, `sync_shift_log`, `revert_cycle`, `cleanup_safe_artifacts`
  - `cycle.py` — `build_prompt`, `command_for_agent`, `verify_cycle`, `evaluate_baseline`, `extract_json`, `blocked_file`
  - `cli.py` — `run_nightshift`, `summarize`, `verify_cycle_cli`, `build_parser`, `main`
  - `__main__.py` — entry point for `python3 -m nightshift`
- `[feat]` **nightshift.schema.json** — JSON Schema for structured agent cycle output. Codex uses `--output-schema` to enforce it. Claude parses via `extract_json()`
- `[feat]` **.nightshift.json.example** — Per-repo config template with all tunable knobs
- `[feat]` **.gitignore** — Ignores worktree dirs, runner logs, state files
- `[feat]` **Interactive agent picker** — `prompt_for_agent()` asks "1) codex  2) claude" when no agent configured. `resolve_agent()` priority: CLI flag → config → interactive → error
- `[feat]` **Runner-enforced guard rails**:
  - Max 3 fixes per cycle
  - Max 5 files per fix
  - Max 12 files touched per cycle
  - Max 4 low-impact fixes per shift
  - Blocked paths (CI, deploy, infra, vendor, lockfiles)
  - Hot-file protection (recent git activity)
  - Category dominance check (>50% rejection)
  - Path bias detection (3 consecutive cycles in same dir)
  - Clean worktree requirement
  - Shift log in every commit
  - No file deletions
  - Halt after 2 failed verifications or 2 empty cycles
- `[feat]` **Baseline verification** — Runs repo tests before starting. Auto-detects `npm test`, `cargo test`, `go test`, `python3 -m pytest`. Enters log-only mode if baseline fails
- `[feat]` **Machine-readable state** — `docs/Nightshift/YYYY-MM-DD.state.json` tracks cycles, counters, violations, halt reasons
- `[feat]` **Cycle revert** — Failed verification → `git reset --hard` to pre-cycle HEAD
- `[feat]` **`--dry-run` flag** — Preview the prompt without spawning agents
- `[feat]` **CLI subcommands** — `run`, `test`, `summarize`, `verify-cycle`
- `[test]` **123-test suite** — `tests/test_nightshift.py` covering all pure functions, config, state, CLI, and dry-run integration
- `[docs]` **Vision documents**:
  - `docs/vision/00-overview.md` — Full vision: both loops, meta-prompt, architecture principles
  - `docs/vision/01-loop1-hardening.md` — Loop 1 roadmap with 6 improvement items
  - `docs/vision/02-loop2-feature-builder.md` — Loop 2 design: sub-agent architecture, phases, open questions
- `[docs]` **Self-improving prompt** — `docs/prompt/evolve.md`
- `[docs]` **Changelog system** — `docs/changelog/` with per-version files
- `[docs]` **Vision tracker** — `docs/vision-tracker/` with progress tracking
- `[meta]` **pyproject.toml** — mypy strict, ruff lint/format, pytest config
- `[meta]` **requirements-dev.txt** — Pinned dev tool versions
- `[meta]` **scripts/check.sh** — Local CI script
- `[meta]` **.github/workflows/ci.yml** — CI pipeline: lint → typecheck + test → integration + artifact validation

## First Autonomous Run

Nightshift ran against its own repo for the first time:
- 2 cycles with Claude adapter
- Baseline failed (tests/ not on main yet) → log-only mode
- Cycle 1: Found `merge_config()` shallow update replaces default blocked paths instead of extending (security)
- Cycle 2: Found `run_command()` timeout can't fire while `readline()` blocks on hung process (reliability)
- Both cycles accepted. Shift log: `docs/Nightshift/2026-04-03.md`

