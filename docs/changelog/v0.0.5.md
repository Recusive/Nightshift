# v0.0.5 -- Multi-Repo

**Released**: TBD
**Tag**: `v0.0.5`
**Status**: In progress

Multi-repo support: run hardening loops across multiple target repositories in a single shift.

## Added

- **[runner]** Multi-repo support: new `nightshift multi` subcommand runs hardening shifts across multiple target repos sequentially. Accepts a list of repo paths, validates each upfront, runs a full shift per repo, and prints an aggregate summary with per-repo status, cycle counts, fixes, and logged issues.
- **[runner]** New `nightshift/multi.py` module with `run_multi_shift()`, `validate_repos()`, `format_multi_summary()`, and `_read_repo_metrics()`.
- **[types]** `RepoShiftResult` TypedDict for per-repo shift outcomes.
