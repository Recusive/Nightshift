**Released**: TBD
**Tag**: `v0.0.6`

Loop 2 scaffolding begins: repo profiling, feature planning, and the foundation for sub-agent coordination.

## Added

- **[profiler]** Repo profiling module (`nightshift/profiler.py`): detects primary language by file extension census, frameworks by marker files and package.json dependencies, package manager, test runner, instruction files (CLAUDE.md, AGENTS.md, etc.), top-level directory structure, monorepo markers, and total file count. Returns a `RepoProfile` TypedDict consumed by all future Loop 2 components.
- **[types]** `RepoProfile` and `FrameworkInfo` TypedDicts for structured repo analysis.
- **[constants]** Profiling data: `LANGUAGE_EXTENSIONS`, `FRAMEWORK_MARKERS`, `FRAMEWORK_PACKAGES`, `INSTRUCTION_FILE_NAMES`, `MONOREPO_MARKERS`, `PROFILER_SKIP_DIRS`.

- **[planner]** Feature planner module (`nightshift/planner.py`): takes a `RepoProfile` + natural language feature description, builds a structured planning prompt, validates agent-generated plans, computes execution waves via topological sort, and renders plans as human-readable markdown. Includes scope checking to warn when features should be broken into phases.
- **[types]** `FeaturePlan`, `PlanTask`, `ArchitectureDoc`, `TestPlan` TypedDicts for structured feature planning.
- **[constants]** Planning data: `PLAN_PROMPT_TEMPLATE`, `PLAN_MAX_TASKS`, `PLAN_MAX_TOTAL_FILES`.
- **[cli]** `nightshift plan` subcommand: `--dry-run` prints the planning prompt, `--result-file` parses and displays an agent-generated plan.
- **[schema]** `schemas/feature.schema.json` -- JSON Schema for structured plan output from agents.
