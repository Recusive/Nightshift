# v0.0.3 — Intelligence

**Released**: TBD
**Tag**: `v0.0.3`
**Status**: In progress

Improvements to cycle intelligence: config security, timeout reliability, diff scoring.

## Fixed

- `[fix]` **merge_config() deep merge** — List fields (`blocked_paths`, `blocked_globs`) now union with defaults instead of replacing them. Previously, a user config with `"blocked_paths": ["my/path/"]` would silently drop all default blocked paths (`.github/`, `deploy/`, `infra/`, etc.), a security issue. Scalar fields still override normally.
- `[fix]` **run_command() timeout race** — Replaced blocking `readline()` loop with a daemon reader thread. Main thread now enforces the timeout via `thread.join()` with a deadline. Previously, `readline()` blocked indefinitely on hung processes, making `timeout_seconds` useless. Now the process is terminated/killed reliably when the deadline expires. Partial output captured before timeout.
- `[fix]` **discover_base_branch CI test** — Skips length assertion in CI where detached HEAD returns empty string.

## Added

- `[feat]` **Post-cycle diff scorer** — `score_diff()` in `cycle.py` scores each cycle's changes 1-10 based on production impact. Factors: fix category priority (Security=8, Error Handling=6, Tests=6, etc.), diff content analysis (regex patterns for security/error keywords), test file bonus (+1), category breadth bonus (+1 for multi-category cycles). Cycles scoring below `score_threshold` (default 3) are reverted with "agent should try harder". Integrated into the main cycle loop in `cli.py`.
- `[feat]` **`score_threshold` config option** — New `.nightshift.json` field (default: 3). Controls minimum diff score for cycle acceptance.
- `[type]` **`DiffScore` TypedDict** — New type in `types.py` for structured diff scoring results.
- `[feat]` **Cycle-to-cycle state injection** — `build_state_summary()` in `cycle.py` generates a human-readable summary of prior cycles (categories fixed, unexplored categories, paths visited, running totals) and injects it into the agent prompt as "Prior cycle intelligence". On cycle 1 the block is omitted. Prevents agents from re-exploring the same categories and paths across cycles.
