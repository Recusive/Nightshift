# v0.0.7 -- Security Hardening

**Released**: TBD
**Tag**: `v0.0.7`
**Status**: In progress

Security hardening for running against untrusted target repositories.

## Added

- **[security]** Prompt injection protection for target repos: the cycle prompt now pre-reads instruction files (CLAUDE.md, AGENTS.md, etc.) from the target repo and injects their content inside an untrusted context block with explicit behavioral warnings. The agent is told to treat these as coding-convention references only. Cycle directives always take precedence over target repo instructions. (`nightshift/cycle.py`: `read_repo_instructions()`, `wrap_repo_instructions()`; `nightshift/constants.py`: `UNTRUSTED_INSTRUCTIONS_PREAMBLE`, `UNTRUSTED_INSTRUCTIONS_SUFFIX`)
- **[docs]** SKILL.md now includes a "Prompt Injection Protection" section explaining the threat model and operator guidance.

## Changed

- **[prompt]** Cycle prompt no longer tells the agent to "Read these first: 1. The repo's AGENTS.md / CLAUDE.md / equivalent instructions." Instead, instruction content is pre-loaded by the runner and injected inline with an untrusted wrapper.
- **[security]** Prompt self-modification guard for all daemon scripts. Before each cycle, the daemon snapshots 7 prompt/control files (CLAUDE.md, evolve.md, evolve-auto.md, review.md, overseer.md, strategist.md, harden-daemon.md). After the cycle, it compares checksums. If any file was modified, the daemon logs a full diff, flags the session index with `[PROMPT MODIFIED]`, and writes an alert file that the next cycle injects into the agent's prompt for review. (`scripts/lib-agent.sh`: `save_prompt_snapshots()`, `check_prompt_integrity()`, `cleanup_prompt_snapshots()`; all 4 daemon scripts instrumented)
