# v0.0.7 -- Security Hardening

**Released**: TBD
**Tag**: `v0.0.7`
**Status**: In progress

Security hardening for running against untrusted target repositories.

## Added

- **[security]** Prompt injection protection for target repos: the cycle prompt now pre-reads instruction files (CLAUDE.md, AGENTS.md, etc.) from the target repo and injects their content inside an untrusted context block with explicit behavioral warnings. The agent is told to treat these as coding-convention references only. Cycle directives always take precedence over target repo instructions. (`nightshift/cycle.py`: `read_repo_instructions()`, `wrap_repo_instructions()`; `nightshift/constants.py`: `UNTRUSTED_INSTRUCTIONS_PREAMBLE`, `UNTRUSTED_INSTRUCTIONS_SUFFIX`)
- **[docs]** SKILL.md now includes a "Prompt Injection Protection" section explaining the threat model and operator guidance.

## Changed

- **[prompt]** Cycle prompt no longer tells the agent to "Read these first: 1. The repo's AGENTS.md / CLAUDE.md / equivalent instructions." Instead, instruction content is pre-loaded by the runner and injected inline with an untrusted wrapper.
- **[security]** Prompt self-modification guard for all daemon scripts. Before each cycle, the daemon snapshots 7 prompt/control files (CLAUDE.md, evolve.md, evolve-auto.md, review.md, overseer.md, strategist.md, harden-daemon.md). After the cycle, it compares checksums. If any file was modified, the daemon logs a full diff, flags the session index with `[PROMPT MODIFIED]`, and writes an alert file that the next cycle injects into the agent's prompt for review. (`scripts/lib-agent.sh`: `save_prompt_snapshots()`, `check_prompt_integrity()`, `cleanup_prompt_snapshots()`; all 4 daemon scripts instrumented)

- **[config]** Configurable model, effort, and thinking for agent commands. Claude commands now accept `--model` and `--effort` flags; Codex commands accept `--model` and `-c reasoning_effort`. All configurable via `.nightshift.json` keys (`claude_model`, `claude_effort`, `codex_model`, `codex_thinking`) or environment variables (`NIGHTSHIFT_CLAUDE_MODEL`, `NIGHTSHIFT_CODEX_MODEL`, `NIGHTSHIFT_CODEX_THINKING`). Config file > env var > default. Defaults match `lib-agent.sh`. Applies to `command_for_agent()`, `plan_command_for_agent()`, and `_build_subagent_command()`. (task #0035)
- **[config]** Codex CLI flags updated from old `-c 'approval_policy="never"' -s workspace-write` to `--dangerously-bypass-approvals-and-sandbox` across all command builders.

- **[feat]** Interactive daemon setup: running any daemon script with no arguments now prompts for agent choice (claude/codex) and duration (2h/4h/6h/8h/unlimited), with a confirmation step before starting. Duration converts to a max-sessions count (hours * 2, assuming 30-min sessions). Passing arguments directly (e.g., `daemon.sh claude 60 10`) skips prompts for backward compatibility. Shared functions `interactive_setup()` and `interactive_setup_strategist()` live in `lib-agent.sh`. All 4 daemon scripts updated. (task #0034)
- **[feat]** Cost tracking and budget ceiling for daemon sessions. After each session, the daemon parses token usage from the stream-json log (input, cache creation, cache read, output), calculates USD cost using model-specific pricing, and appends to a cumulative ledger (`docs/sessions/costs.json`). Session index now includes a cost column. Set `NIGHTSHIFT_BUDGET=50` to auto-stop the daemon when cumulative spend exceeds the limit. Interactive setup includes a budget prompt ($25/$50/$100/unlimited). Supports Claude Opus, Sonnet, and Haiku pricing. New Python module `nightshift/costs.py` with 32 tests. All 3 looping daemon scripts instrumented. (task #0026)
- **[feat]** Daemon log rotation and orphan branch pruning. At the start of each daemon cycle, old session logs (>.7 days) are automatically deleted and remote branches created by nightshift that have no open PR are pruned. Configurable via `NIGHTSHIFT_KEEP_LOGS` env var (default: 7 days). Only branches matching daemon prefixes (`feat/`, `fix/`, `docs/`, `refactor/`, `release/`, `test/`) are pruned; protected branches (`main`, `master`, `develop`) are never touched. New Python module `nightshift/cleanup.py` with 31 tests. All 3 looping daemon scripts instrumented. (task #0027)
