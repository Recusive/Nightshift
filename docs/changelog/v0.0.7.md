# v0.0.7 -- Security Hardening

**Released**: 2026-04-05
**Tag**: `v0.0.7`
**Status**: Released

Security hardening for running against untrusted target repositories.

## Added

- **[security]** Prompt injection protection for target repos: the cycle prompt now pre-reads instruction files (CLAUDE.md, AGENTS.md, etc.) from the target repo and injects their content inside an untrusted context block with explicit behavioral warnings. The agent is told to treat these as coding-convention references only. Cycle directives always take precedence over target repo instructions. (`nightshift/cycle.py`: `read_repo_instructions()`, `wrap_repo_instructions()`; `nightshift/constants.py`: `UNTRUSTED_INSTRUCTIONS_PREAMBLE`, `UNTRUSTED_INSTRUCTIONS_SUFFIX`)
- **[docs]** SKILL.md now includes a "Prompt Injection Protection" section explaining the threat model and operator guidance.

## Changed

- **[prompt]** Cycle prompt no longer tells the agent to "Read these first: 1. The repo's AGENTS.md / CLAUDE.md / equivalent instructions." Instead, instruction content is pre-loaded by the runner and injected inline with an untrusted wrapper.
- **[security]** Prompt self-modification guard for all daemon scripts. Before each cycle, the daemon snapshots 7 prompt/control files (CLAUDE.md, evolve.md, evolve-auto.md, review.md, overseer.md, strategist.md, harden-daemon.md). After the cycle, it compares checksums. If any file was modified, the daemon logs a full diff, flags the session index with `[PROMPT MODIFIED]`, and writes an alert file that the next cycle injects into the agent's prompt for review. (`scripts/lib-agent.sh`: `save_prompt_snapshots()`, `check_prompt_integrity()`, `cleanup_prompt_snapshots()`; all 4 daemon scripts instrumented)

- **[config]** Configurable model, effort, and thinking for agent commands. Claude commands now accept `--model` and `--effort` flags; Codex commands accept `--model` and `-c reasoning_effort`. All configurable via `.nightshift.json` keys (`claude_model`, `claude_effort`, `codex_model`, `codex_thinking`) or environment variables (`NIGHTSHIFT_CLAUDE_MODEL`, `NIGHTSHIFT_CODEX_MODEL`, `NIGHTSHIFT_CODEX_THINKING`). Config file > env var > default. Defaults match `lib-agent.sh`. Applies to `command_for_agent()`, `plan_command_for_agent()`, and `_build_subagent_command()`. (task #0035)
- **[config]** Codex CLI flags updated from old `-c 'approval_policy="never"' -s workspace-write` to `--dangerously-bypass-approvals-and-sandbox` across all command builders.

- **[feat]** Interactive daemon setup: running any daemon script with no arguments now prompts for agent choice (claude/codex) and duration (2h/4h/6h/8h/unlimited), with a confirmation step before starting. Duration converts to a max-sessions count (hours * 2, assuming 30-min sessions). Passing arguments directly (e.g., `daemon.sh claude 60 10`) skips prompts for backward compatibility. Shared functions `interactive_setup()` and `interactive_setup_strategist()` live in `lib-agent.sh`. All 4 daemon scripts updated. (task #0034)
- **[feat]** Cost tracking and budget ceiling for daemon sessions. After each session, the daemon parses token usage from the stream-json log (input, cache creation, cache read, output), calculates USD cost using model-specific pricing, and appends to a cumulative ledger (`docs/sessions/costs.json`). Session index now includes a cost column. Set `NIGHTSHIFT_BUDGET=50` to auto-stop the daemon when cumulative spend exceeds the limit. Interactive setup includes a budget prompt ($25/$50/$100/unlimited). Supports Claude Opus, Sonnet, and Haiku pricing. New Python module `nightshift/costs.py` with 32 tests. All 3 looping daemon scripts instrumented. (task #0026)
- **[feat]** Daemon log rotation and orphan branch pruning. At the start of each daemon cycle, old session logs (>.7 days) are automatically deleted and remote branches created by nightshift that have no open PR are pruned. Configurable via `NIGHTSHIFT_KEEP_LOGS` env var (default: 7 days). Only branches matching daemon prefixes (`feat/`, `fix/`, `docs/`, `refactor/`, `release/`, `test/`) are pruned; protected branches (`main`, `master`, `develop`) are never touched. New Python module `nightshift/cleanup.py` with 31 tests. All 3 looping daemon scripts instrumented. (task #0027)
- **[feat]** Automated handoff compaction in daemon. At the start of each daemon cycle, numbered handoff files (`docs/handoffs/NNNN.md`) are counted. When 7+ files exist, they are automatically compacted into a weekly summary in `docs/handoffs/weekly/` and the originals are deleted. Prevents context bloat from accumulating handoffs. New Python module `nightshift/compact.py` with 14 tests. All 3 looping daemon scripts instrumented. (task #0030)

- **[security]** Instruction file size cap: `read_repo_instructions()` now enforces a per-file limit (`MAX_INSTRUCTION_FILE_BYTES`, 10 KB) and a total limit across all files (`MAX_INSTRUCTION_TOTAL_BYTES`, 30 KB). Files exceeding the per-file limit are truncated; files that would exceed the total budget are either partially truncated or skipped entirely. Truncation warnings are injected into the output so the agent knows content was cut. Prevents malicious repos from flooding the agent's context window with oversized instruction files. (`nightshift/constants.py`, `nightshift/cycle.py`; task #0036)

- **[security]** Symlink rejection in instruction file reader: `read_repo_instructions()` now checks `is_symlink()` before reading any instruction file. Symlinked files are skipped with a security warning injected into the output. Prevents path-traversal attacks where a malicious repo symlinks CLAUDE.md to sensitive files like `/etc/passwd`. Even broken/dangling symlinks are detected and rejected. (`nightshift/cycle.py`; task #0037)

- **[security]** New-file detection in prompt self-modification guard: `save_prompt_snapshots()` now captures directory listings for watched directories (`docs/prompt/`). After a cycle, `check_prompt_integrity()` compares the listings and flags any new files the agent created. Detects both new files in existing directories and entirely new directories. New `PROMPT_GUARD_DIRS` array in `scripts/lib-agent.sh` makes it easy to add more watched directories. (task #0037)

- **[feat]** Healer -- meta-layer observer merged into builder session. The builder now runs an "Observe the System" step (Step 6n in evolve.md) before generating tasks. Reads the last 5 session entries, previous healer observations, task queue, and vision tracker to identify trends and upcoming problems. Writes observations to `docs/healer/log.md`. The original separate daemon.sh agent call never worked (shell expansion mangled markdown with backticks/$), so the healer was redesigned as a builder step that uses the builder's existing file access and commit flow. (`docs/prompt/evolve.md` Step 6n, `docs/prompt/healer.md` as reference; tasks #0046, #0061)

- **[feat]** Human escalation channel (`notify_human`). When the daemon hits a situation requiring human attention (circuit breaker tripped, budget limit reached, healer critical pattern), it creates a GitHub issue with the `needs-human` label and optionally fires a webhook. Configurable via `notification_webhook` in `.nightshift.json`. Wired into all 3 looping daemon circuit breakers, builder budget stop, and healer prompt. Fails silently -- never crashes the daemon. New `needs-human` label on repo. (`scripts/lib-agent.sh`: `notify_human()`; `scripts/daemon.sh`, `scripts/daemon-review.sh`, `scripts/daemon-overseer.sh`; `nightshift/types.py`, `nightshift/constants.py`, `nightshift/config.py`; task #0048)

- **[meta]** Generate Work step (Step 6n) in evolve.md. After each session, the agent now scans the system across 7 dimensions (meta pipeline, code quality, repo health, architecture, agent DX, vision progress, security) and creates 1-5 follow-up tasks. Prevents the agent from being a passive task runner -- it actively identifies work. Enforces dimension diversity, duplicate checking, specific acceptance criteria, and a per-session cap of 5 tasks. Moves Meta-Prompt "Priority engine" from 0% to 50%. (task #0053)

- **[meta]** Learnings verification in status reports. Agent sessions must now quote a specific learning from `docs/learnings/INDEX.md` in the status report and handoff, with an explanation of how it affects the session's approach. Creates an auditable trail that learnings are being read, not just written. Added to evolve.md Step 1 status report template, Step 6b handoff requirements, Step 12 session report, `PRE-PUSH-CHECKLIST.md` Part 3, and handoff README template. (task #0033)

- **[docs]** CONTRIBUTING.md -- agent-to-agent collaboration protocol. Written for two audiences: external contributing agents (how to produce clean PRs) and the resident daemon (how to handle external contributions). Covers quality gates, branch/commit/PR conventions, registration checklist, what to touch vs not touch, and machine-parseable PR format. Structured for LLM context windows. Does not duplicate CLAUDE.md. (task #0040)

- **[feat]** GitHub Issues to task file sync. Humans now create tasks as GitHub Issues with the `task` label instead of editing files directly. The daemon's housekeeping step (`sync_github_tasks` in `lib-agent.sh`) converts open issues to `docs/tasks/` files, maps labels to frontmatter (urgent/low -> priority, integration -> environment, loop1/loop2/self-maintaining/meta-prompt -> vision_section), closes the issue with "Converted to task #NNNN" comment, and commits the new files. Eliminates `.next-id` collision risk between human and daemon. Wired into all 3 looping daemons. 4 new GitHub labels created. 6 docs updated. (task #0070)

- **[meta]** Multi-agent PR review panel that scales with task complexity. Five specialized review agents replace the single monolithic code-reviewer: `code-reviewer.md` (narrowed to structure/types/tests), `safety-reviewer.md` (security, secrets, subprocess, destructive ops), `docs-reviewer.md` (changelog, CLAUDE.md, tracker, handoff consistency), `architecture-reviewer.md` (dependency flow, module boundaries, design), `meta-reviewer.md` (daemon scripts, prompt, autonomous pipeline integrity). Complexity heuristic in evolve.md Step 8 determines review depth: docs-only/small PRs get 1 reviewer, medium PRs get 3, complex PRs get 4-5 in parallel. All must pass to merge. (task #0047)

- **[feat]** Feature summary generation module for Loop 2. After `nightshift build` completes a feature, `generate_feature_summary()` in `nightshift/summary.py` extracts files created/modified, tests written, task completion counts, detected patterns (API endpoints, CLI commands, database changes, new modules, new test files), and a natural language description suitable for changelogs and handoffs. Summary stored in the feature state file under a `summary` key. Integrated into `build_feature()` as a post-verification step. Displayed in `format_feature_status()` output. 22 new tests. (task #0059)

- **[meta]** Vision-alignment check for task creation. New `vision_section` frontmatter field on tasks (`loop1`, `loop2`, `self-maintaining`, `meta-prompt`, `none`). Before creating tasks, agents must check the last 5 tasks -- if 3+ target the same vision section, new tasks must prioritize a different section. Prevents tunnel vision where all work clusters in one area while others stagnate. Rule added to `docs/tasks/GUIDE.md` and enforced in evolve.md Step 6n constraints. Moves Meta-Prompt "Priority engine" from 50% to 60%. (task #0031)

- **[feat]** Production-readiness checker module for Loop 2. After `nightshift build` completes final verification, `check_production_readiness()` in `nightshift/readiness.py` runs configurable checks on changed files: secret/credential detection (AWS keys, GitHub PATs, API keys, passwords), debug statement scanning (print, console.log, debugger, breakpoint, pdb), and test coverage verification (checks for test files matching production source files). Returns a structured `ReadinessReport` with per-check pass/fail and an overall verdict. Configurable via `readiness_checks` list in `.nightshift.json` (default: all three checks enabled). Integrated into `build_feature()` pipeline and displayed in `format_feature_status()` output. 40 new tests. Moves Loop 2 from 72% to 81%. (task #0068)

- **[feat]** E2E test runner module for Loop 2. After all waves complete but before final verification, `run_e2e_tests()` in `nightshift/e2e.py` runs the project's full test suite and optional smoke tests. Detects test commands from Makefile (make test), package.json (npm/pnpm/yarn/bun test), pyproject.toml/pytest.ini (python3 -m pytest), Cargo.toml (cargo test), and go.mod (go test ./...). Scans for smoke test scripts (scripts/smoke-test.sh, etc.) and runs them if found. Returns structured `E2EResult` with per-command exit codes and output. E2E failure is a gate -- prevents final verification from running. Integrated into `build_feature()` pipeline and displayed in `format_feature_status()` output. 42 new tests. Moves Loop 2 from 81% to 90%. (task #0065)

- **[feat]** Sub-agent coordination module for Loop 2. Before spawning a wave, `coordinate_wave()` in `nightshift/coordination.py` scans work order prompts and acceptance criteria for file-path references, detects overlapping files between parallel tasks, generates per-task coordination hints, and injects them into the work order prompts. After a wave completes, `log_conflicts()` checks for actual file conflicts (files touched by 2+ tasks) and logs warnings. Six public functions: `extract_file_references()`, `detect_overlaps()`, `generate_coordination_hints()`, `inject_hints()`, `detect_file_conflicts()`, `format_conflict_report()`. Pattern data in `constants.py` (`FILE_REFERENCE_PATTERN`, `COORDINATION_HINT_TEMPLATE`). Three new TypedDicts: `FileOverlap`, `FileConflict`, `ConflictReport`. Integrated into `build_feature()` wave loop. 47 new tests. Completes Loop 2 at 100%. (task #0083)

- **[feat]** Self-evaluation loop for quality scoring. After every N daemon sessions (configurable, default 5), `evaluate()` in `nightshift/evaluation.py` clones the test target repo, runs a test shift, parses the resulting state file and shift log, scores across 10 dimensions (Startup, Discovery, Fix quality, Shift log, State file, Verification, Guard rails, Clean state, Breadth, Usefulness), writes an evaluation report to `docs/evaluations/NNNN.md`, and creates follow-up tasks for any dimension scoring below threshold (default 6/10). All scorers are pure functions taking `ShiftArtifacts` -- no I/O. Configurable via `eval_frequency` and `eval_target_repo` in `.nightshift.json`. Integrated into `daemon.sh` via `should_evaluate()` and `run_evaluation()` in `lib-agent.sh`. Evaluation failure is non-blocking. Three new TypedDicts: `DimensionScore`, `EvaluationResult`, `ShiftArtifacts`. 66 new tests. (task #0049)
