### 1. merge_config shallow update silently drops default blocked paths (cycle 1)
- **Severity**: high
- **Category**: Security
- **Files**: `nightshift.py:259`
- **What I found**: `merge_config()` uses `config.update(loaded)` to apply user overrides from `.nightshift.json`. This is a shallow dict merge — if a user sets `"blocked_paths": ["custom/"]`, the entire default list is replaced rather than extended. The defaults include critical safety paths: `.github/`, `deploy/`, `dist/`, `infra/`, `k8s/`, `terraform/`, `vendor/`. The same issue applies to `blocked_globs`.
- **Production impact**: A user who adds a custom blocked path in `.nightshift.json` would unknowingly remove all default protections, allowing the agent to edit CI/CD pipelines, deployment configs, build artifacts, and vendored dependencies. This directly undermines the safety rails documented in SKILL.md.
- **Suggested fix**: Deep-merge list fields (`blocked_paths`, `blocked_globs`) by extending the defaults rather than replacing them. Something like: for each key in `loaded`, if both the default and override are lists, use `config[key] = list(set(config[key] + loaded[key]))` instead of a flat `update`. Alternatively, provide explicit `blocked_paths_add` / `blocked_paths_remove` keys for user overrides while preserving defaults.
- **Why not fixed tonight**: File has recent team activity; also requires a design decision on the merge strategy (extend vs. explicit add/remove keys).
