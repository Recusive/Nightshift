# v0.0.1 — Initial Beta

**Released**: 2026-04-03
**Tag**: `v0.0.1`
**GitHub Release**: [v0.0.1 — Initial Beta](https://github.com/Recusive/Nightshift/releases/tag/v0.0.1)

The first release of Nightshift. Claude Code only. Bash orchestration. Prompt-discipline safety.

## Added

- `[feat]` **nightshift/SKILL.md** — Core skill with discovery strategies, priority order, fix/log framework, safety rails, shift log template, and worktree isolation
- `[feat]` **scripts/run.sh** — Overnight runner: bash while-loop spawning fresh `claude -p` sessions in 30-min cycles inside a git worktree. Configurable duration and cycle length
- `[feat]` **scripts/test.sh** — Test runner: 4 short cycles for validation. Same architecture as scripts/run.sh
- `[feat]` **scripts/install.sh** — One-liner installer. Downloads nightshift/SKILL.md + scripts/run.sh to `~/.claude/skills/nightshift/`
- `[docs]` **README.md** — Project description, install instructions, usage guide, architecture diagram

## Architecture

- Single-agent: Claude Code only
- Safety: prompt-discipline only (no runner enforcement)
- Orchestration: pure bash (`while` loop, `claude -p`, `tee`, `cp`)
- State: shift log markdown is the only cross-cycle memory
- Isolation: git worktree at `docs/Nightshift/worktree-YYYY-MM-DD/`

## Known Limitations

- Claude Code only — no Codex, no Copilot CLI
- Agent gravitates toward easy a11y fixes over high-impact work
- Frontend tunnel vision — React components dominate, backend underexplored
- No runner enforcement — agent is trusted to self-police limits
- No machine-readable state — only the markdown shift log
- Test writing rarely happens despite being priority #3
