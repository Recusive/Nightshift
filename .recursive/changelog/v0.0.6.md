# v0.0.6 -- Loop 2 Foundation

**Released**: 2026-04-05
**Tag**: `v0.0.6`
**Status**: Released

Loop 2 scaffolding begins: repo profiling, feature planning, and the foundation for sub-agent coordination.

## Added

- **[profiler]** Repo profiling module (`nightshift/profiler.py`): detects primary language by file extension census, frameworks by marker files and package.json dependencies, package manager, test runner, instruction files (CLAUDE.md, AGENTS.md, etc.), top-level directory structure, monorepo markers, and total file count. Returns a `RepoProfile` TypedDict consumed by all future Loop 2 components.
- **[types]** `RepoProfile` and `FrameworkInfo` TypedDicts for structured repo analysis.
- **[constants]** Profiling data: `LANGUAGE_EXTENSIONS`, `FRAMEWORK_MARKERS`, `FRAMEWORK_PACKAGES`, `INSTRUCTION_FILE_NAMES`, `MONOREPO_MARKERS`, `PROFILER_SKIP_DIRS`.

- **[planner]** Feature planner module (`nightshift/planner.py`): takes a `RepoProfile` + natural language feature description, builds a structured planning prompt, validates agent-generated plans, computes execution waves via topological sort, and renders plans as human-readable markdown. Includes scope checking to warn when features should be broken into phases.
- **[types]** `FeaturePlan`, `PlanTask`, `ArchitectureDoc`, `TestPlan` TypedDicts for structured feature planning.
- **[constants]** Planning data: `PLAN_PROMPT_TEMPLATE`, `PLAN_MAX_TASKS`, `PLAN_MAX_TOTAL_FILES`.
- **[cli]** `nightshift plan` subcommand: `--dry-run` prints the planning prompt, `--result-file` parses and displays an agent-generated plan. `--agent claude` or `--agent codex` invokes the agent to generate a plan and displays the formatted result with scope warnings.
- **[planner]** Agent integration for plan generation: `plan_command_for_agent()` builds CLI commands, `run_plan_agent()` handles the full invoke-parse-validate pipeline.
- **[constants]** Plan agent data: `PLAN_AGENT_MAX_TURNS` (10), `PLAN_AGENT_TIMEOUT` (300s).
- **[schema]** `schemas/feature.schema.json` -- JSON Schema for structured plan output from agents.

- **[decomposer]** Task decomposer module (`nightshift/decomposer.py`): converts a `FeaturePlan` + `RepoProfile` into sub-agent work orders grouped by execution wave. Each work order contains a complete sub-agent prompt with repo context, feature context, task-specific instructions, acceptance criteria, dependency context, and the expected JSON output schema. Includes `format_work_orders()` for human-readable rendering.
- **[types]** `WorkOrder` and `DecomposerResult` TypedDicts for sub-agent work order structures.
- **[constants]** Decomposer data: `WORK_ORDER_PROMPT_TEMPLATE`, `TASK_SCHEMA_PATH`, `DECOMPOSER_MAX_RETRIES`.
- **[schema]** `schemas/task.schema.json` -- JSON Schema for structured sub-agent task completion output.

- **[subagent]** Sub-agent spawner module (`nightshift/subagent.py`): takes a wave of `WorkOrder`s from the decomposer, spawns codex or claude CLI agents to execute each one, parses their structured JSON output into `TaskCompletion` results, retries on agent failures (up to `DECOMPOSER_MAX_RETRIES`), and reports wave-level results via `WaveResult`. Includes `format_wave_result()` for human-readable output.
- **[types]** `TaskCompletion` and `WaveResult` TypedDicts for sub-agent execution results.
- **[constants]** Spawner data: `SUBAGENT_DEFAULT_TIMEOUT` (600s), `SUBAGENT_MAX_TURNS` (50).

- **[integrator]** Wave integrator module (`nightshift/integrator.py`): merges sub-agent work after each wave by collecting files, staging via git, running the full test suite, diagnosing failures by matching test output to task files, and spawning fix agents (up to `INTEGRATOR_MAX_FIX_ATTEMPTS`). Returns structured `IntegrationResult` with pass/fail status, staged files, fix attempt history, and failure diagnosis. Includes `format_integration_result()` for human-readable markdown output.
- **[types]** `IntegrationResult` and `FixAttempt` TypedDicts for wave integration results.
- **[constants]** Integrator data: `INTEGRATOR_MAX_FIX_ATTEMPTS` (3), `INTEGRATOR_TEST_TIMEOUT` (300s).

- **[feature]** Feature build orchestrator module (`nightshift/feature.py`): ties profiler -> planner -> decomposer -> spawner -> integrator into `nightshift build "description"`. Supports `--status` to check progress, `--resume` to continue interrupted builds, `--yes` to skip confirmation. Persists `FeatureState` to disk for crash recovery. Runs final verification (tests + lint) after all waves pass.
- **[types]** `FeatureState`, `FeatureWaveState`, `FinalVerificationResult` TypedDicts for feature build state tracking.
- **[cli]** `nightshift build` subcommand with `--status`, `--resume`, `--yes`, `--agent`, `--repo-dir` flags.
