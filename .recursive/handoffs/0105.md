### Red team audit of daemon post-agent processing

Focused on the trust boundary between agent execution and daemon processing.
The daemon's python3 -c calls that run AFTER the agent exits are the critical
attack surface because agent-created files are present in the working directory.

**New CONFIRMED finding -- PYTHONPATH shadow module bypass (#0198):**

The daemon's post-agent `python3 -c` blocks use `PYTHONPATH="$RECURSIVE_DIR/lib:$REPO_DIR"`.
Python's `-c` flag prepends `''` (current working directory) to `sys.path` at position 0.
Since cwd is the repo root, a shadow module at repo root is imported BEFORE
`Recursive/lib/costs.py`. An agent can exploit this to:

1. Zero out all cost tracking (budget limiter never triggers)
2. Execute arbitrary code in daemon context via shadow modules

Verified via: `PYTHONPATH="Recursive/lib:." python3 -c "import sys; print(sys.path[:3])"`
Output: `['', 'Recursive/lib', 'repo_root']` -- `''` (cwd) is at position 0.

Even a broken shadow module (raises ImportError) works because the `|| echo "0.0000"`
fallback at daemon.sh:386 silently defaults cost to 0.
