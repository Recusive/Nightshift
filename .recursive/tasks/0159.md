# Document PROMPT_GUARD_DIRS non-recursive scan limitation

`save_prompt_snapshots` and `check_prompt_integrity` use `ls -1` (non-recursive) when scanning `PROMPT_GUARD_DIRS`. For `.github/workflows`, this means files added in subdirectories under `.github/workflows/` are not scanned. GitHub Actions does not load workflow YAML from subdirectories, so this is low-risk in practice, but it is an undocumented boundary.

## Acceptance Criteria
- [ ] OPERATIONS.md or lib-agent.sh has a comment documenting the non-recursive limitation for PROMPT_GUARD_DIRS
- [ ] OR: switch from `ls -1` to `find -maxdepth 1` (same behavior, explicit about depth) with a task to consider deeper scans
- [ ] Existing behavior is unchanged -- this is documentation/clarity only unless deep scan is implemented
