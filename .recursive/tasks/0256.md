# Add regression test for daemon.sh role extractor

## Problem

PR #254 hardened the role extraction pipeline in `daemon.sh` against sed metacharacter injection. The fix was verified manually but has no automated regression test. A test would prevent future regressions if the extraction code is modified.

## What To Do

1. Add a test in `.recursive/tests/` that verifies the role extraction logic:
   - Valid roles (build, review, security-check, brain) are extracted correctly
   - Injection attempts (sed metacharacters, shell metacharacters) result in `unknown`
   - Empty/blank ROLE DECISION lines result in `unknown`
   - Missing ROLE DECISION lines leave SESSION_ROLE at default `brain`
2. The test should exercise the actual awk/tr/case pipeline from daemon.sh, not a reimplementation

## Acceptance Criteria

- At least 4 test cases covering valid roles, injection, empty, and missing
- Tests pass in `make check`
- No changes to daemon.sh itself
