# Include failed tasks in detect_file_conflicts scan

PR #72 code review noted that `detect_file_conflicts()` in `coordination.py` only iterates `wave_result["completed"]`. Failed/blocked tasks that partially wrote files before reporting failure are excluded from conflict detection. The docstring says "Check completed tasks" which is accurate but may mislead callers.

Either extend the scan to include `wave_result["failed"]` task completions (with a comment explaining why), or update the docstring to explicitly note that failed tasks are excluded and explain the rationale.

## Acceptance Criteria
- [ ] `detect_file_conflicts()` either scans failed tasks or documents the exclusion
- [ ] Tests cover the chosen behavior
