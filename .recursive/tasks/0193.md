# Harden remaining git commit calls in tests with CI identity flags

## Context

PR #180 fixed `TestGitStatusShort::test_clean_repo_returns_empty_output` by adding
`-c user.email=ci@test -c user.name=CI` to the git commit call. Code review flagged
that roughly 25 other `git commit` calls in `nightshift/tests/test_nightshift.py` carry the
same CI fragility (no identity flags), including those at lines 1030, 1043, 1067,
2392-2466, 3211, and others.

CI runners without a global git config will fail these tests with exit code 128.

## Fix

Audit all `git commit` calls in `tests/test_nightshift.py` and add
`-c user.email=ci@test -c user.name=CI` (or equivalent) to any call that does not
already carry identity flags.

## Acceptance Criteria

- [ ] All `git commit` calls in the test file carry identity flags or use an env-level
  git config fixture
- [ ] `make check` passes
- [ ] CI green after merge
