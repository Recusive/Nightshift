# Auto-release version tagging

Self-Maintaining "Auto-release" is at 0% in the tracker. Currently, releases are tagged manually during sessions (Step 6m + Step 11 in evolve.md). The agent checks if a version milestone is complete and manually runs `gh release create`. This has proven unreliable -- v0.0.6 has been ready for multiple sessions but never tagged.

Build a `nightshift/release.py` module that automates the release decision and execution:
1. Read version milestones from `Recursive/ops/OPERATIONS.md`
2. Check whether all tasks for a version are `status: done`
3. Read the changelog file for the version
4. If ready: create git tag, push tag, create GitHub release with changelog content
5. Return a structured result (released: bool, version, tag, release_url)

Wire it into the evolve.md Step 11 as a callable function, or into daemon.sh as a post-merge step.
