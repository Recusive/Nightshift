# Harden sync_github_tasks against YAML-injection in issue titles/bodies

The `sync_github_tasks` function in `scripts/lib-agent.sh` writes GitHub issue
titles and bodies into task files without sanitizing them for YAML frontmatter
injection.  A GitHub issue titled `\nstatus: done\n---\n` could produce a task
file with a corrupted frontmatter block that confuses the task parser.  While
the Python code that builds the task file correctly quotes the title inside
`# $title` (as a Markdown heading, not frontmatter), the body is written
verbatim and could contain a `---` sequence that closes the frontmatter block
and injects arbitrary content above it.

Severity: low (requires a GitHub issue with a malicious body; a human must
first create the issue and label it `task`; the daemon then converts it).

## Acceptance Criteria

- [ ] `sync_github_tasks` sanitizes or escapes `---` sequences in issue bodies
      before writing them into task files, OR documents why the current Python
      templating is safe and adds a regression test that verifies a body
      containing `---` cannot corrupt the frontmatter of the generated file
- [ ] A test in `TestSyncGithubTasks` (or equivalent) covers a body with `---`
      and verifies the resulting task file has exactly one frontmatter block
- [ ] Test suite remains green after the addition
