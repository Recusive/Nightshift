# Replace session index cost recording with multi-model aware recording in v2

## Problem

The v2 daemon.sh records session cost using `record_multi_model_session` from
costs.py, which is correct. However the session index (index.md) shows $0.0000
for many sessions in the cross-session data, and costs.json only has 2 entries
despite 76+ sessions.

The costs.json only started recording when v2 launched (2026-04-07). The pre-v2
sessions used a different cost recording mechanism. This is acceptable.

However, the index.md shows costs for individual sessions going up to $34 (a 
single session costing $34 is a significant outlier). Looking at the session 
history, there are many $27-$34 sessions -- these are expensive and need analysis.

Additionally, the "Feature" and "PR" columns in index.md are always empty for
brain sessions (the brain delegates to sub-agents, so the feature summary and PR
URL are in the sub-agent logs, not the brain log). The session index parser in
daemon.sh may need updating to extract sub-agent PR URLs.

## Acceptance Criteria
- [ ] Session index shows PR URLs for brain sessions (extracted from sub-agent outputs)
- [ ] Sessions with cost > $15 are flagged in the dashboard or healer log for review
- [ ] costs.json accurately accumulates all v2 session costs
