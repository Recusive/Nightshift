# Add missing test for sessions_since_eval date-only fallback path

## Problem

PR #244 code review identified that the `sessions_since_eval` function in `signals.py` has a code path for when the eval file contains no parseable date (returns `len(sessions_index)`), but no test covers this edge case.

## Fix

Add a test in `test_signals.py::TestSessionsSinceEval` that creates an eval file with no `**Date**: YYYY-MM-DD` line and verifies that `sessions_since_eval` returns `len(sessions_index)`.

## Acceptance Criteria

- New test for the no-date fallback path exists
- Test verifies `sessions_since_eval` returns `len(sessions_index)` for malformed eval files
- All tests still pass
