## Context

In `Recursive/engine/daemon.sh` the security-abort early-exit path writes a session index
row using `${FEATURE:--}`. At that point in the code FEATURE has not yet been
extracted from the current session log, so it carries the previous cycle's value
(or is empty on the first cycle). The previous cycle's value was already sanitized,
so this is not exploitable -- but it is a latent inconsistency flagged in the PR
#156 review.

## Acceptance Criteria

- The security-abort index write uses a clearly scoped value (e.g., a literal `-`
  or a local variable set explicitly to `-` before any extraction logic runs).
- A comment explains why FEATURE is not yet available at that point in the code.
- `make check` passes.
