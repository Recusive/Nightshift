# Pin Phractal to a known-good commit

The eval system clones Phractal at HEAD. If upstream changes break Phractal's own build, every evaluation fails and the agent wastes sessions diagnosing an external repo.

Fix: pin the Phractal clone to a specific commit SHA in a config file (.recursive/evaluations/targets.json). The eval step uses this SHA instead of HEAD. Update the SHA manually when you want to test against newer code.

## Acceptance Criteria
- [ ] .recursive/evaluations/targets.json exists with Phractal repo + commit SHA
- [ ] Eval step in evolve.md uses the pinned SHA
- [ ] smoke-test.sh also uses the pinned SHA

## Overseer Decomposition

The code-only part of this work now has explicit internal queue coverage:

- `#0132` tracks richer evaluation-target metadata so pinned revisions can live
  in structured data instead of ad-hoc branches.
- `#0136` tracks wiring Step 0 and `scripts/smoke-test.sh` to honor an optional
  pinned target revision once one is configured.

This parent stays blocked because choosing and periodically refreshing a real
Phractal commit SHA still depends on external repo state, which the internal
builder queue cannot prove stable on its own.
