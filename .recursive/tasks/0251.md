# Harden daemon.sh role extractor against sed metacharacters

## Problem

The role extractor in `daemon.sh` (line ~196) pipes agent-controlled log content through `sed` and `tr`:

```bash
sed 's/ROLE DECISION: *//' | tr -d '[:space:]' | head -c 20
```

A crafted `ROLE DECISION:` line containing sed metacharacters can corrupt the `SESSION_ROLE` variable. Evidence: session `20260409-020609` had role `.*'"$LOG_FILE"2>/d` in the session index (fixed in PR #248).

## What To Do

1. Restrict the extracted role to `[a-z-]` character class only
2. Add a validation check after extraction: if the role doesn't match a known role list (`build|review|oversee|strategize|achieve|security-check|evolve|audit|brain`), default to `unknown`
3. Add a test case if possible

## Acceptance Criteria

- Role extractor only accepts `[a-z-]` characters
- Unknown/malformed roles default to a safe value
- No shell injection possible through crafted ROLE DECISION lines
