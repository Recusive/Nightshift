# Add URL validation to clone_repo() in worktree.py (T-1)

## Problem

`clone_repo(url, dest)` in `nightshift/infra/worktree.py` passes the `url` argument directly to `subprocess.run(["git", "clone", url, str(dest)])`. There is no URL scheme or content validation. The function is a public API exported by `nightshift/__init__.py`.

Currently all call sites use hardcoded or internally-derived URLs (PHRACTAL_URL from eval_targets.py), so no real attack path exists today. However, if a future call site passes user-supplied input, the same risks as Finding C-1 apply (local path clone, `--upload-pack` flag injection).

The fix for C-1 (task #0268) will add URL validation to `eval_target_repo` in config and `run_eval_full()`. This task closes the remaining surface by adding validation inside `clone_repo()` itself, making it safe by default regardless of the call site.

## Fix

Add a validation guard at the start of `clone_repo()`:

```python
def clone_repo(url: str, dest: Path) -> None:
    if not (url.startswith("https://") or url.startswith("git@")):
        raise NightshiftError(
            f"clone_repo: unsafe URL rejected. Only https:// and git@ are allowed. Got: {url!r}"
        )
    ...
```

The validation should mirror whatever is implemented for task #0268.

- [ ] `clone_repo()` rejects values starting with `--` or `/`
