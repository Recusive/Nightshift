# Wrap os.unlink in try/except OSError in _create_github_release finally block

## Problem

In `_create_github_release()` in `nightshift/infra/release.py`, the `finally` block calls `os.unlink(notes_file)`. If the tempfile was already deleted by an external actor between `run_capture` returning and the `finally` executing, a `FileNotFoundError` would propagate and mask the original `NightshiftError` from a failed `gh release create`.

## Fix

Wrap the `os.unlink` call:
```python
finally:
    try:
        os.unlink(notes_file)
    except OSError:
        pass
```
