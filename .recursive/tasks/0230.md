# Keep _DELEGATION_ROLE_MAP in sync with available sub-agents

Meta-reviewer advisory note on PR #230: if the brain ever delegates a new sub-agent type not in `_DELEGATION_ROLE_MAP` (in `.recursive/engine/signals.py`), the delegation is silently dropped from sessions-since counters. This is safe behavior but means new agent types would be invisible to the advisory system until the map is updated.

## Acceptance Criteria
- [ ] When a new sub-agent type is added to `.recursive/agents/`, `_DELEGATION_ROLE_MAP` is documented as a required update in the new-agent checklist or OPERATIONS.md
- [ ] Alternatively, add a test that verifies all agent definition files in `.recursive/agents/` have a corresponding entry in `_DELEGATION_ROLE_MAP`
- [ ] `make check` passes
