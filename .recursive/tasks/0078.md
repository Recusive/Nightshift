# Add live integration test gate to PR review pipeline

Task #0047's design included a "live integration test gate" that runs after all reviewers pass but before merge. This exercises the actual changed code paths with real invocations (not just reviewer opinions). It was not implemented in the multi-agent review panel PR.

Add to evolve.md Step 8 (between reviewer pass and merge): the builder writes a small throwaway test that exercises the exact code path the PR changed -- with real files, real CLI invocations, real behavior verification.

Source: code review advisory note on PR #63.

## Acceptance Criteria

- [ ] evolve.md Step 8 includes an integration test step between reviewer pass and merge
- [ ] Step provides examples of what "real test" means for different PR types (Python module, shell script, config change)
- [ ] The test is proportional to the PR -- small PRs get a simple smoke test, large PRs get more thorough verification
- [ ] The builder runs the test and includes the output in the PR body or merge commit
