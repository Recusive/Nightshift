# Backlog budget for Generate Work step

The first strategist `Prompt Health` report found that the builder is following the "create 1-5 tasks every session" instruction too literally. The queue grew to 50 unresolved active tasks while evaluation debt remained unfixed, so task generation is now amplifying backlog faster than the builder can retire it.

## Acceptance Criteria
- [ ] `Recursive/operators/build/SKILL.md` limits task generation to `0-2` per session instead of `1-5`
- [ ] Task generation step checks the current active-task count before generating new work
- [ ] Hard cap: if 50+ pending tasks exist, generate 0 new tasks (absorbs superseded task `#0129`)
- [ ] Above a documented backlog ceiling, the builder must prefer reprioritizing or tightening existing tasks over creating new ones
- [ ] `Recursive/ops/DAEMON.md` or `.recursive/tasks/GUIDE.md` documents the backlog-budget rule
