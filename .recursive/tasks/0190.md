# Batch minor documentation comments in daemon/lib-agent shell scripts

Consolidates three trivial "add a comment" tasks that can be done in one pass.
These are all low-risk, zero-behavior-change documentation additions.

## Items

### 1. Safety comment on notify_human python3 -c call (original #0190)

In `Recursive/engine/lib-agent.sh`, add a comment before the `payload=$(python3 -c ...)`
block in `notify_human` explaining:
`# safe: $title is passed as sys.argv[1], not interpolated into the -c string`

### 2. Document PROMPT_GUARD_DIRS non-recursive scan (absorbed from #0159)

In `Recursive/engine/lib-agent.sh` or `Recursive/ops/OPERATIONS.md`, add a comment
documenting that `save_prompt_snapshots` uses `ls -1` (non-recursive) when scanning
`PROMPT_GUARD_DIRS`, so files in subdirectories are not scanned.

### 3. Document pipe sanitization tab omission (absorbed from #0161)

In `Recursive/engine/daemon.sh`, add a brief inline comment on the `tr -d '|\n\r'`
lines noting that tabs are intentionally omitted (handled by Python-side `.strip()`
in `parse_session_index`).
