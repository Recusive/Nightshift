# Add docstring to _scan_file_for_patterns and fix is_file/is_symlink ordering in readiness.py

## Context

Code reviewer and safety reviewer advisory notes from PR #204:

1. `_scan_file_for_patterns()` receives a `file_path: Path` that callers have already
   validated via `_is_within_repo()`. The function itself has no guard. A future caller
   who skips the guard would create a silent vulnerability. Add a brief docstring note:
   "Callers must verify `file_path` is within `repo_dir` before calling."

2. In `check_test_coverage` candidates inner loop, `is_file()` appears before
   `is_symlink()`, which is inconsistent with the order in `check_secrets` and
   `check_debug_prints` (and the project learning about symlink-before-is-file).
   No security impact given no file reading occurs, but the inconsistency could
   mislead future maintainers. Reverse to `not .is_symlink() and .is_file()`.

## Acceptance Criteria

- [ ] `_scan_file_for_patterns` has docstring noting caller responsibility for path validation
- [ ] `check_test_coverage` candidates loop uses `is_symlink()` before `is_file()` ordering
- [ ] `make check` passes
