# Behavioral test for pick_session_role stdout isolation

## Context

PR #179 fixed #0184 (pick_session_role stdout+stderr merge) with two contract tests
that verify the fix exists at the source level (no `2>&1`, has `mktemp`). The code
review flagged that these are static analysis only -- a future refactor could satisfy
the string checks while reintroducing the semantic bug.

## Fix

Add a behavioral regression test using `subprocess.run(["bash", ...])` that:
1. Creates a mock `pick-role.py` that prints a stray stdout line BEFORE or AFTER
   the role name
2. Runs `pick_session_role` (or the equivalent logic extracted from daemon.sh)
3. Asserts that `SESSION_ROLE` equals the intended role, not the stray line

The test should use `bash -c 'source Recursive/engine/lib-agent.sh; source Recursive/engine/daemon.sh; ...'`
or extract the function body to a temp script for isolated testing.

## Acceptance Criteria

- [ ] Test creates a mock pick-role.py that emits unexpected stdout after the role name
- [ ] Test verifies SESSION_ROLE is still the correct role (not the stray line)
- [ ] Test is added to a logical location in test_nightshift.py
- [ ] `make check` passes
