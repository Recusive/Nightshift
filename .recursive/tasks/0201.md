# Harden lib-agent.sh PYTHONPATH against shadow module bypass

## Context

PR #190 fixed daemon.sh by replacing PYTHONPATH with sys.path.insert(0, lib_dir)
inside python3 -c blocks. Code review found 6 remaining `PYTHONPATH="$RECURSIVE_DIR/lib:$REPO_DIR"`
calls in `Recursive/engine/lib-agent.sh` that have the same vulnerability:

- Line 505: cleanup_old_logs
- Line 534: cleanup_healer_log
- Line 565: cleanup_orphan_branches
- Line 624: compact_handoffs
- Line 798: should_evaluate
- Line 824: run_evaluation

Most are housekeeping (run after git reset, lower risk). But `should_evaluate`
and `run_evaluation` run POST-AGENT where shadow modules can exist.

## Fix

Apply the same pattern as daemon.sh: replace `PYTHONPATH="$RECURSIVE_DIR/lib:$REPO_DIR"`
with `_NS_LIB="$RECURSIVE_DIR/lib"` env var prefix, and add
`sys.path.insert(0, os.environ['_NS_LIB'])` inside each python3 block.

## Acceptance Criteria

- [ ] All 6 PYTHONPATH calls in lib-agent.sh use sys.path.insert pattern
- [ ] No `$REPO_DIR` in any PYTHONPATH assignment in lib-agent.sh
- [ ] `bash -n Recursive/engine/lib-agent.sh` passes
- [ ] `make check` passes
