# Add legend to MODULE_MAP.md clarifying arrow direction

Meta-reviewer noted that MODULE_MAP.md's dependency order section uses arrows (`cli -> __main__`) without clarifying whether the arrow means "imports" or "topological precedence." The regenerated map changed `__main__ -> cli` to `cli -> __main__`, which is correct for topological order but could confuse agents reading the map.

Add a one-line legend to the "Dependency Order" section of MODULE_MAP.md template/generator clarifying: "A -> B means A must be loaded before B (A is a dependency of B)."
