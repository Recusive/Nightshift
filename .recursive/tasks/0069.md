# Auto-changelog module

Self-maintaining "Auto-changelog update" is at 0% in the tracker. Currently, agents manually add changelog entries to `docs/changelog/vX.X.X.md`. This is error-prone -- entries get missed, formatting drifts, and the agent burns context reading the file structure.

Build a `nightshift/autochangelog.py` module that can:

1. Parse existing changelog entries from `docs/changelog/` files
2. Generate a changelog entry from a git diff + commit message + PR description
3. Append the entry to the correct version file under the correct section (Added/Changed/Fixed/Removed/Internal)
4. Detect the current version from the latest changelog file

This is infrastructure for the self-maintaining loop. Agents still review the output, but the heavy lifting is automated.

## Acceptance Criteria
- [ ] `nightshift/autochangelog.py` exists with `generate_changelog_entry()` and `append_to_changelog()` functions
- [ ] Parses existing changelog structure correctly
- [ ] Categorizes entries by type (feat->Added, fix->Fixed, refactor->Changed, etc.)
- [ ] Tests cover parsing, generation, and appending
- [ ] `make check` passes
