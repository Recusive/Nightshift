# Separate eval-target overrides from generic verify-command inference

PR #106 added repo-URL overrides inside `infer_verify_command()`. The
architecture review passed the change, but noted that generic package-based
inference and evaluation-target overrides now share one path, which can blur
expectations for future non-eval callers.

## Acceptance Criteria
- [ ] Split package-based verify inference from repo-specific eval-target
      overrides into clearly named helpers
- [ ] Keep current CLI/evaluation behavior unchanged for Phractal and ordinary
      package-based repos
- [ ] Add tests covering both the generic inference path and the
      override-aware path
