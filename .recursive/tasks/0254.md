# Consider moving _SUBPACKAGE_DIRS to core/constants.py

## Problem

`_SUBPACKAGE_DIRS = ("core", "settings", "owl", "raven", "infra")` is defined inline in `nightshift/infra/module_map.py`. CLAUDE.md says "No hardcoded data in logic files" -- regex patterns, score maps, thresholds go in `core/constants.py`.

## Nuance

This is borderline -- it's module-private structural config, not a threshold or weight. The constant names the subpackage directories and is only used by module_map.py. Moving it to constants.py would make it more discoverable but also creates a cross-subpackage dependency for a single consumer.
