# Surface healer rotation subprocess failures in housekeeping

The shared `cleanup_healer_log()` shell helper currently suppresses stderr and
uses `|| true`, so an unexpected Python import/runtime failure can disable
healer retention silently.

## Acceptance Criteria
- [ ] `cleanup_healer_log()` prints a one-line failure notice when the Python subprocess exits nonzero unexpectedly
- [ ] The helper remains best-effort and does not fail the daemon loop
- [ ] Regression coverage exercises the shell helper on a forced Python failure path
