# Sanitize category_counts on state file load

## Problem

PR #263 and PR #265 added category string allowlist validation at both write paths (state.py and cycle.py). However, `_build_state` at state.py line 72 reads `category_counts` raw from disk without filtering against `_VALID_CATEGORIES`. If a state file contains unknown category keys from a prior session (before the allowlist fix), those keys survive reload and could influence the dominance guard check.

## Fix

In `_build_state` (state.py), filter `category_counts` keys against `_VALID_CATEGORIES` when loading from JSON. Unknown keys should be silently dropped.

## Acceptance Criteria

- [ ] `_build_state` filters category_counts keys against _VALID_CATEGORIES
- [ ] Test for loading a state file with unknown category keys
- [ ] `make check` passes
