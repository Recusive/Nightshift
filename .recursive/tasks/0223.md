# Worktree cleanup incomplete after daemon sessions

## Problem

After daemon sessions, `.claude/worktrees/agent-*` directories are left behind. The daemon calls `cleanup_worktrees` but it doesn't catch all of them. We had to manually `git worktree remove` after sessions.

## Fix

1. The daemon's post-session cleanup should explicitly remove all `.claude/worktrees/agent-*` worktrees
2. Add `git worktree prune` after removal
3. The housekeeping step at cycle start should also clean up stale worktrees from crashed sessions

## Acceptance Criteria

- No leftover worktrees after a clean daemon session
- Stale worktrees from crashed sessions are cleaned at next cycle start
