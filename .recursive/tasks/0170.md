# Deduplicate stream-json dual-format parsing logic across lib-agent.sh extractors

## Problem

`extract_result_summary`, `extract_feature_from_log`, and `extract_pr_url_from_log`
in `Recursive/engine/lib-agent.sh` each contain an identical 20-line Python inner loop that
handles both Claude (`type:result`) and Codex (`item.completed/agent_message`) formats.
The logic was triplicated when the Codex fix landed in PR #164 (task #0169).

If the Codex event schema changes (e.g., a new field name), the same fix must be applied
to all three functions independently. This is a maintenance burden.

## Solution

Extract a single shared Python helper (e.g., `_extract_last_result_text(log_path)`)
as a standalone script or a shared heredoc function in lib-agent.sh that all three
extractors call. Each extractor then only contains the domain-specific search logic
(searching for "Built:", "PR:", or returning the full text).

## Acceptance Criteria

- [ ] Common dual-format parsing loop appears once in `Recursive/engine/lib-agent.sh` (not three times)
- [ ] All three extractors produce identical output to the current implementation
- [ ] Existing tests (9 tests from PR #164) still pass unchanged
- [ ] `make check` passes
