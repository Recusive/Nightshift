# Fix latent IndexError in readiness display formatting

Code reviewer advisory: `feature.py:450` uses `check['details'].splitlines()[0]` which raises `IndexError` if `details` is an empty string (since `"".splitlines()` returns `[]`). The `_build_readiness_check` deserializer can produce `details=""` from malformed persisted state.

Fix with defensive access: `(check['details'].splitlines() or [""])[0]`.
