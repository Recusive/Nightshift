# Replace manual NightshiftConfig construction in profiler.py with DEFAULT_CONFIG copy

`nightshift/profiler.py` line 162-185 manually constructs a NightshiftConfig by listing every key from DEFAULT_CONFIG. Every time a new config key is added, this construction must be manually updated or mypy fails. This is fragile and has already caused a build break (readiness_checks addition required a fix).

Replace with a copy of DEFAULT_CONFIG, overriding only the fields that need to differ (currently just `verify_command=None`):

```python
import json
config = NightshiftConfig(**json.loads(json.dumps(DEFAULT_CONFIG)))
config["verify_command"] = None
```

Or better, extract a factory function in config.py that does this.

## Acceptance Criteria
- [ ] profiler.py no longer manually lists all NightshiftConfig keys
- [ ] Adding a new config key does NOT require updating profiler.py
- [ ] `make check` passes
