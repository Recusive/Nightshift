# Brain never re-runs eval after building nightshift

## Problem

The brain ran `nightshift run` against Phractal once (eval #0016, score 53→86) in session #0108. Since then, 5+ sessions of nightshift code changes (security hardening, bug fixes, features) have shipped but the brain never re-evaluated to measure impact.

The build→measure→build feedback loop is broken. The brain builds without measuring.

## Fix

Add a rule to brain.md: after every 3 build sessions, the next session MUST include an eval rerun task. The dashboard should show "sessions since last eval" and alert when >= 3.

## Acceptance Criteria

- Dashboard shows `sessions_since_eval` signal
- Dashboard alerts when eval is 3+ sessions stale
- Brain includes eval rerun in its delegation when the alert fires
- Eval score trend is visible in the dashboard
