# Tighten validate_repo_url: reject embedded newlines and carriage returns

## Problem

`validate_repo_url("https://github.com/foo/bar\n--config=foo")` currently passes validation because the URL starts with `https://`. While the subprocess call is list-form (no shell injection possible), embedded newlines in URLs are never legitimate and could cause issues if the URL is ever logged or passed through a shell-based wrapper.

## Fix

Add a check in `validate_repo_url()` in `nightshift/core/shell.py`:
```python
if "\n" in stripped or "\r" in stripped:
    raise NightshiftError("eval_target_repo rejected: URL contains embedded newline/carriage return")
```

Place it after the `strip()` and empty check, before the prefix checks.
