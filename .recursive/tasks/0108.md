# Module map should survive per-file syntax errors

`nightshift.module_map.generate_module_map()` currently parses every
`nightshift/*.py` file eagerly. A single syntax error in one partially edited
module can abort the whole map refresh, which weakens the feature exactly when a
session is mid-edit and most needs orientation help.

## Acceptance Criteria
- [ ] `python3 -m nightshift module-map --write` still succeeds when one package module has invalid syntax
- [ ] The generated output records which module(s) failed to parse instead of silently dropping them
- [ ] Regression tests cover a repo with one valid module and one syntactically invalid module
