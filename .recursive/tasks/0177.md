# Re-run Step 0 evaluation to confirm score rise after #0139 fix

## Context

Evaluation #0015 scored 53/100. Three PRs that directly fix the scored
failures have since been merged:

- PR #165 (review session): hardened `evaluation.py` and fixed 2 pentest
  findings — improves Guard rails (was 4/10) and Fix quality (was 3/10)
- PR #167 (build session, task #0139): count-only Claude payloads no longer
  false-reject via "structured output implies 0" — fixes Guard rails (was
  4/10) and Shift log (was 0/10 due to false rejections)
- PR #158 (build session, tasks #0101/#0102): rejected-cycle findings and
  parser fidelity — improves State file and Fix quality

The BUILD EVAL GATE requires eval >= 80 before normal task work can proceed.
With a stale 53/100, this gate has been blocking normal BUILD tasks all day.
No mechanism currently triggers a Step 0 re-run automatically after relevant
fixes land. Handoff #0092 notes the daemon "will re-run eval automatically
on next qualifying cycle" — but no such mechanism exists.

This task gives the next BUILD session an explicit directive to re-run the
evaluation and record a new report before proceeding to other tasks.

## Fix

Run Step 0 evaluation against Phractal:

```bash
PYTHONPATH=$(pwd) python3 -m nightshift test \
  --agent claude --cycles 2 --cycle-minutes 5 \
  --repo-dir /tmp/nightshift-eval-$(date +%s)
```

Score the result across all 10 dimensions and write a new evaluation report
to `.recursive/evaluations/0016.md`. If the score is >= 80, the eval gate
unblocks and normal BUILD tasks resume. If the score is still below 80,
identify which dimensions still fail and create targeted tasks.

## Acceptance Criteria

- [ ] A new evaluation report `.recursive/evaluations/0016.md` is committed
- [ ] The report scores all 10 dimensions with raw evidence
- [ ] If total score >= 80: handoff documents that eval gate is clear
- [ ] If total score < 80: at least one new task targets each dimension
      still below 6/10 (or references an existing pending task)
- [ ] `make check` passes (no new regressions introduced by the eval run)
