# Fix inconsistent SNAP_DIR quoting at daemon.sh inline cleanup

## Problem

PR #259 safety reviewer noted: the inline `rm -rf "$SNAP_DIR"` at line 261 of daemon.sh uses `"$SNAP_DIR"` without `:-` default, while `_daemon_cleanup` uses `"${SNAP_DIR:-}"`. Under `set -u`, if SNAP_DIR is unset when line 261 executes, bash would fail with "unbound variable".

This is pre-existing (before PR #259) and low severity since line 261 only runs after SNAP_DIR has been assigned. But for consistency and defense-in-depth, it should match the cleanup function pattern.

## Fix

Change line 261's `rm -rf "$SNAP_DIR"` to `rm -rf "${SNAP_DIR:-}"` to match the pattern in `_daemon_cleanup`.
