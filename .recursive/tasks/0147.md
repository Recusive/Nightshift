# Add test: PROMPT_ALERT closing-tag sanitization in prompt_alert wrapper

Skipped — obsolete in v2 architecture. The `prompt_alert` mechanism (which wrapped alert content in `<prompt_alert>` tags and injected it into the agent prompt) existed only in `daemon-v1.sh`. The v2 `daemon.sh` does not use `PROMPT_ALERT` at all — security alerts are handled differently in v2 (notify_human + logging). The sed sanitization being tested is in v1 dead code. Closing as done.

## Context

The `<prompt_alert>` wrapper logic in `Recursive/engine/daemon.sh`
sanitizes embedded `</prompt_alert>` closing tags via `sed`. This is
security-critical code but has no automated test coverage.

## Acceptance Criteria

- A test writes a crafted alert file containing `</prompt_alert>` within a
  simulated diff body.
- The test runs the shell logic (or calls the daemon function) that constructs
  the wrapped `PROMPT` variable.
- The test verifies the output contains `[/prompt_alert]` (escaped) rather
  than the raw closing tag.
- The test verifies the structural `<prompt_alert>` / `</prompt_alert>` wrapper
  is still present and intact in the output.
- `make check` continues to pass.
