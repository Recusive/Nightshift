# Tighten CODEX_THINKING validation to an explicit allowlist

## Context

PR #176 safety review advisory: `NIGHTSHIFT_CODEX_THINKING` is validated with
`^[a-z_]+$` which accepts values like `_`, `foo_bar_baz`, etc. that are not real
`reasoning_effort` values. This is not a security bypass (the characters blocked are
sufficient to prevent shell injection), but an allowlist would catch typos and provide
better UX.

Current valid values for Codex `reasoning_effort`: `low`, `medium`, `high`, `extra_high`.

## Acceptance Criteria

- Change the validation regex from `^[a-z_]+$` to `^(low|medium|high|extra_high)$`
- Update the error message to list valid values explicitly
- Update `TestCodexThinkingValidation`: existing tests for valid/invalid values still
  pass; optionally add a test for a malformed-but-lowercase value like `foo_bar`
- `make check` passes
