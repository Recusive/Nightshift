# Add CRLF regression test for _is_valid_eval_file()

## Context

PR #170 code review (session #0094) noted that `_is_valid_eval_file()` has
no test for CRLF-formatted eval files. Python's `re.MULTILINE` handles CRLF
correctly, but the coverage gap mirrors the exact class of bug (CRLF handling)
that triggered a separate regression test class (`TestHasUrgentTasksCRLF`) in
the same test file.

## Fix

In `tests/test_pick_role.py`, add a test to `TestIsValidEvalFile`:

```python
def test_crlf_eval_file_accepted(self) -> None:
    text = (
        "**Date**: 2026-04-06\r\n"
        "| Startup | 8/10 | OK |\r\n"
        "| Discovery | 6/10 | OK |\r\n"
        "| Fix quality | 5/10 | OK |\r\n"
        "| **Total** | **70/100** | |\r\n"
    )
    assert _is_valid_eval_file(text) is True
```
