# Guard should_evaluate() against non-numeric eval_frequency crashing bash arithmetic

## Context

Pentest-2026-04-06 (watch item): `should_evaluate()` in `Recursive/engine/lib-agent.sh` reads
`eval_frequency` from `.recursive.json` via a `python3 -c` call and then uses it
in bash arithmetic:

```bash
$(( session_count % freq ))
```

If `.nightshift.json` contains a non-numeric `eval_frequency` value and Python prints
it literally without a ValueError fallback, bash arithmetic under `set -uo pipefail`
would terminate the daemon. `.nightshift.json` is in `PROMPT_GUARD_FILES`, so agent
mutation is detected -- but a malformed config (human typo or partial write) would
cause a hard crash.

## Fix

Add a numeric validation in the Python extraction:
```python
try:
    val = int(config.get('eval_frequency', 5))
    print(val)
except (ValueError, TypeError):
    print(5)  # safe default
```

Or add a bash-side guard before the arithmetic:
```bash
if ! [[ "$freq" =~ ^[0-9]+$ ]]; then
    freq=5
fi
```

## Acceptance Criteria

- [ ] A non-numeric `eval_frequency` in `.nightshift.json` does NOT crash the daemon
- [ ] The function falls back to the default frequency (5) and logs a warning when
      the config value is invalid
- [ ] Existing behavior is unchanged when `eval_frequency` is a valid integer
- [ ] `bash -n Recursive/engine/lib-agent.sh` passes
