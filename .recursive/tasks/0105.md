# GitHub task sync should fill default target and vision_section fields

GitHub-synced tasks `#0093`, `#0094`, and `#0103` arrived without `target` or `vision_section`, which weakens release accounting and the vision-alignment check. The sync path should emit fully formed task frontmatter even when the issue labels are minimal.

## Acceptance Criteria
- [ ] Synced tasks default `target` to the current in-progress version when no explicit version label exists
- [ ] Synced tasks default `vision_section` to `none` when no vision label exists
- [ ] Sync tests cover unlabeled issues so queue parsing and tracker alignment stay stable
- [ ] `.recursive/tasks/GUIDE.md` or the sync docs describe the defaulting behavior
