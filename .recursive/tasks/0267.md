# Deduplicate _VALID_CATEGORIES frozenset between state.py and cycle.py

## Problem

Both `nightshift/core/state.py` and `nightshift/owl/cycle.py` independently define `_VALID_CATEGORIES: frozenset[str] = frozenset(CATEGORY_ORDER)`. Both derive from the same constant, so they're always in sync today. However, if a future change modifies the import in one file and not the other, they could silently diverge.

## Fix

Define `VALID_CATEGORIES` once in `nightshift/core/constants.py` (alongside CATEGORY_ORDER) and import it in both state.py and cycle.py. Remove the per-module _VALID_CATEGORIES definitions.

## Acceptance Criteria

- [ ] Single VALID_CATEGORIES definition in constants.py
- [ ] state.py and cycle.py import from constants instead of defining locally
- [ ] `make check` passes
