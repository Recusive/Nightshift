# Generalize evaluation target metadata beyond a single verify-command map

`nightshift/eval_targets.py` currently hardcodes a single target verifier as a
flat `dict[str, str]`. That is acceptable for Phractal, but the architecture
review noted that the shape will become brittle once Nightshift supports more
than one or two known evaluation targets.

## Acceptance Criteria
- [ ] Replace the flat verify-command map with a structured metadata shape that
      can carry additional per-target settings without changing control flow
- [ ] Keep the shipped Phractal verifier behavior unchanged
- [ ] Add regression coverage proving a second target can be expressed as data,
      not a new branch in the lookup logic
