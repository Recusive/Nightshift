# Tighten source: pentest substring match to regex in signals.py

## Context

Safety-reviewer advisory note (PR #199): `count_pending_pentest_framework_tasks()`
uses `"source: pentest" not in fm` which is a permissive substring match. It would
match `source: pentest-extended-v2` or similar compound values.

While this is an advisory signal (not a security gate), tightening to a regex like
`re.search(r"^source:\s*pentest\s*$", fm, re.MULTILINE)` would be more consistent
with how `status:` and `priority:` fields are matched elsewhere in the same file.

## Acceptance Criteria

- [ ] Substring checks in `count_pending_pentest_framework_tasks()` replaced with
  anchored regex patterns for both `source:` and `target:` fields
- [ ] Consistent with matching style used in other signal functions
- [ ] `make check` passes
