# Add explicit sed sanitization for open_pr_data closing tag

The `OPEN_PR` string injected into both the pentest and builder prompts is wrapped in `<open_pr_data>` tags, but unlike `PENTEST_REPORT` and `ALERT_CONTENT`, the `OPEN_PR` content does not have an explicit `sed` pass to escape the `</open_pr_data>` closing tag.

Currently the `tr` character-stripping on `PR_TITLE` (line 187) and `PR_BRANCH` (line 188) removes `<`, `>`, `/` which prevents the constructed string from containing the closing tag. However, this defense relies on two separate sanitization steps being consistent.

An explicit `sed 's|</ *open_pr_data *>|[/open_pr_data]|g'` on `OPEN_PR` before injection would make the defense self-contained, consistent with the existing `pentest_data` and `prompt_alert` patterns, and easier to audit.

## Acceptance Criteria

- [ ] `OPEN_PR` content is sanitized with sed to escape `</open_pr_data>` closing tag variants before injection into prompts
- [ ] Pattern is consistent with existing `pentest_data` and `prompt_alert` escaping
- [ ] `bash -n Recursive/engine/daemon.sh` passes
