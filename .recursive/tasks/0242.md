# Add sessions_since_eval signal and brain rule for eval rerun cadence

## Problem

Eval #0016 (86/100) was run in session #0108. As of session #0123, the eval is 14 sessions stale. 14 sessions of nightshift code changes (security hardening, feature builds, framework fixes) have shipped since the last measurement. The build-measure-build feedback loop is broken: the brain builds without ever measuring the product's health against Phractal. Human filed issue #0228 explicitly about this.

There is no signal in the dashboard that tells the brain "eval is stale." The brain cannot know eval cadence without reading the raw handoff.

## Fix

### Part 1: Add signal (project zone, build agent)

In `nightshift/core/signals.py` or equivalent, add a `sessions_since_eval` signal that:
1. Reads the latest eval file from `.recursive/evaluations/` (highest-numbered `NNNN.md`)
2. Reads the current session count from `.recursive/sessions/index.md`
3. Returns `sessions_since_eval = current_session - eval_session_number`

In `dashboard.py`, surface this signal and emit an alert when `sessions_since_eval >= 5`.

### Part 2: Add brain rule (framework zone, evolve agent)

In `.recursive/agents/brain.md`, add a rule to the Delegation Protocol:

"If `sessions_since_eval >= 5`, the next session MUST include a Phractal E2E eval run as one of the two parallel delegations. Use the build agent to run `nightshift test --agent claude --cycles 2 --cycle-minutes 5 --repo-dir /tmp/nightshift-eval-NNNN` and write results to `.recursive/evaluations/NNNN.md`."

## Acceptance Criteria

- `sessions_since_eval` signal appears in dashboard output
- Dashboard shows alert when eval is 5+ sessions stale
- brain.md contains the eval cadence rule
- At least 1 unit test for the sessions_since_eval calculation
