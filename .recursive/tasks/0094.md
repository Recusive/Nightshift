# E2E validation: run nightshift against Phractal every N sessions

## Problem

The agent builds features and ships PRs but never actually runs nightshift against a real repo (Phractal) to verify the product works end-to-end. Every handoff says "Eval skipped." The evaluation module (evaluation.py) was built but never invoked. We are building in the dark.

## What needs to happen

1. **Wire E2E into the daemon loop.** Every 3-5 sessions (configurable via `eval_frequency` in `.recursive.json`), the daemon should:
   - Clone `https://github.com/fazxes/Phractal.git` to `/tmp/nightshift-eval`
   - Run `nightshift test --agent claude --cycles 2 --cycle-minutes 5` against it
   - Score the results using `Recursive/lib/evaluation.py`
   - Write the score to `.recursive/evaluations/NNNN.md`
   - If score < 40/100, create a `priority: urgent` task and flag in the handoff
   - Clean up the clone

2. **Update `Recursive/operators/build/SKILL.md`** — make evaluation non-skippable when `eval_frequency` triggers.

3. **Update `Recursive/engine/daemon.sh` / `Recursive/engine/lib-agent.sh`** — add a cycle counter or check against the session index to determine if it's time to evaluate.

4. **Update `Recursive/ops/OPERATIONS.md`** — document the E2E validation system.

5. **Update `CLAUDE.md`** — mention the evaluation cadence.

6. **Verify it works** — after wiring it in, the next daemon run MUST actually produce an evaluation report in `.recursive/evaluations/`. If it doesn't, the task is not done.

## Acceptance Criteria

- [ ] Daemon runs E2E evaluation against Phractal every N sessions (configurable)
- [ ] Evaluation report written to `.recursive/evaluations/NNNN.md` with scores
- [ ] Low scores auto-create urgent tasks
- [ ] At least one real evaluation report exists after implementation
- [ ] All relevant docs updated (evolve.md, evolve-auto.md, OPERATIONS.md, CLAUDE.md, DAEMON.md)
