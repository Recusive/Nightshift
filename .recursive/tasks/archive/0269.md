# Replace fixed /tmp/nightshift-eval clone dest with mkdtemp (C-2)

## Problem

`run_eval_full()` in `nightshift/owl/eval_runner.py` uses a fixed path `EVALUATION_CLONE_DEST = "/tmp/nightshift-eval"` as the git clone destination. Before cloning, `_safe_rmtree(clone_dest)` removes it, rejecting symlinks at that path.

However, there is a TOCTOU (time-of-check/time-of-use) race between the rmtree and the git clone:

1. `_safe_rmtree("/tmp/nightshift-eval")` deletes the path -- symlink check passes, path removed.
2. An attacker (or concurrent CI job) creates `ln -s /home/user/.ssh /tmp/nightshift-eval`.
3. `git clone ... /tmp/nightshift-eval` follows the symlink and writes repo content into `/home/user/.ssh/`.

Additionally, the fixed path causes collision when two nightshift processes run concurrently (e.g., parallel CI jobs): one job's cleanup deletes the other's in-progress clone.

## Fix

Replace the fixed `EVALUATION_CLONE_DEST` path with `tempfile.mkdtemp()` for the clone destination, matching the pattern already used for `runtime_dir` in the same function:

```python
clone_dest = Path(tempfile.mkdtemp(prefix="nightshift-eval-"))
```

The `EVALUATION_CLONE_DEST` constant can be removed from `core/constants.py` and from all import sites, or kept as a deprecated fallback that is no longer the default.

Ensure the `finally` block still calls `_safe_rmtree(clone_dest, ignore_errors=True)` to clean up the temp directory.

## Acceptance Criteria

- [ ] `run_eval_full()` uses `tempfile.mkdtemp()` for the clone destination instead of the fixed `/tmp/nightshift-eval` path
- [ ] The temp clone directory is cleaned up in the `finally` block regardless of success or failure
- [ ] `EVALUATION_CLONE_DEST` constant is removed or clearly deprecated with no production code path that still uses it
- [ ] Two concurrent calls to `run_eval_full()` do not collide on the clone path
- [ ] `make check` passes
