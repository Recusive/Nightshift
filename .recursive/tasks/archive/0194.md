# Fix budget limiter triple-failure in daemon cost tracking

## Context

Pentest scan (2026-04-07) confirmed three chained bugs in `Recursive/engine/daemon.sh`
lines 459-472 that completely disable the budget limiter:

1. **TypeError**: `record_session_bundle()` is called with `part_agents=[...]` which
   is not a valid parameter. The correct kwarg is `pentest_agent` (positional arg 5).

2. **KeyError**: `entry['total_cost_usd']` references a key that doesn't exist.
   `record_session_bundle()` returns `cost_usd`, not `total_cost_usd`.

3. **Format mismatch**: `costs.json` is in legacy dict format
   `{"total_cost_usd": N, "sessions": [...]}` but `_read_ledger()` in
   `Recursive/lib/costs.py` returns `[]` for non-list data (line 119).
   This means `total_cost()` always returns 0.0.

All three errors are suppressed by `2>/dev/null || echo "0.0000"` at line 473.

**Impact**: `RECURSIVE_BUDGET` is never enforced. The daemon runs unlimited sessions
regardless of the configured budget. Cost tracking is silently broken.

## Reproduction

```bash
python3 -c "
import sys; sys.path.insert(0,'Recursive/lib')
from costs import record_session_bundle
record_session_bundle(['a.log'], 'c.json', 'sid', 'claude',
                      part_agents=['claude','claude'])"
# -> TypeError: got unexpected keyword argument 'part_agents'
```

## Fix

1. Change `part_agents=[...]` to `pentest_agent=os.environ['_NS_PAGENT']`
2. Change `entry['total_cost_usd']` to `entry['cost_usd']`
3. Add back-compat to `_read_ledger()`: if data is a dict with a `sessions` key,
   extract the list from `data['sessions']`
