# Add trailing anchor to status: pending regex in signals.py

## Context

Meta-reviewer advisory note (PR #205): The existing `r"^status:\s*pending"` pattern in
`.recursive/engine/signals.py` (line 229, pre-existing) has no trailing anchor, meaning
`status: pending-review` would also match. While no known `status:` values have `pending`
as a prefix, adding `\s*$` for consistency with the newly tightened `source:` and `target:`
patterns would prevent future false positives.
