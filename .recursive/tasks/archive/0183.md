# Fix shell-variable interpolation in daemon SESSION_COST python3 -c blocks

## Context

Pentest-2026-04-06 and pentest-2026-04-07 identified direct shell-variable interpolation
into `python3 -c` strings in three daemons. The original task scope was narrow
($PENTEST_AGENT/$AGENT only); the actual fix target was lines 376-428 of daemon.sh
and all 4 unsafe calls each in daemon-review.sh and daemon-overseer.sh.

Variables affected (per script):

**daemon.sh SESSION_COST (376-389):** $PENTEST_LOG_FILE, $LOG_FILE, $COST_FILE,
$SESSION_ID, $AGENT, $PENTEST_AGENT

**daemon.sh CUMULATIVE (424-427):** $COST_FILE

**daemon.sh OVER_BUDGET (428):** $CUMULATIVE, $BUDGET (NIGHTSHIFT_BUDGET env)

**daemon-review.sh (181-223):** $LOG_FILE, $COST_FILE, $SESSION_ID, $AGENT (SESSION_COST);
$LOG_FILE (log extraction); $COST_FILE (CUMULATIVE); $CUMULATIVE/$BUDGET (OVER_BUDGET)

**daemon-overseer.sh (181-223):** same as daemon-review.sh

## Fix

All python3 -c blocks converted to env-var pattern (matching lib-agent.sh established
convention from task #0045):
- Shell vars passed via `_NS_*=value python3 -c "... os.environ['_NS_*'] ..."`
- OVER_BUDGET float comparison moved to `awk -v c="$CUMULATIVE" -v b="$BUDGET"`,
  eliminating the python3 call entirely

## Acceptance Criteria

- [x] daemon.sh:376-389 no longer interpolates shell vars into Python source
- [x] daemon.sh:424-427 CUMULATIVE block uses env-var pattern
- [x] daemon.sh:428 OVER_BUDGET uses awk instead of python3 interpolation
- [x] daemon-review.sh: all 4 unsafe calls converted (SESSION_COST, log-extract,
      CUMULATIVE, OVER_BUDGET)
- [x] daemon-overseer.sh: all 4 unsafe calls converted (SESSION_COST, log-extract,
      CUMULATIVE, OVER_BUDGET)
- [x] `bash -n` passes on all three scripts
- [x] Existing cost tracking behavior is unchanged
