# Fix CODEX_THINKING shell injection in codex exec CLI arg

## Context

Pentest-2026-04-06 (fix-now): `NIGHTSHIFT_CODEX_THINKING` was used directly in a
double-quoted CLI arg at lib-agent.sh (the codex exec call):

```bash
-c "reasoning_effort=\"$CODEX_THINKING\""
```

If `NIGHTSHIFT_CODEX_THINKING` contains a `"` character, the outer double-quoted string
closes prematurely, causing the shell to misparse the argument list. Codex exits non-zero,
the circuit breaker handles it, but the failure is opaque with no indication that the env
var is malformed.

## Fix (applied in PR #176)

Added startup validation immediately after `CODEX_THINKING` is set (near CLAUDE_MODEL):

```bash
if ! printf '%s' "$CODEX_THINKING" | grep -qE '^[a-z_]+$'; then
    echo "ERROR: NIGHTSHIFT_CODEX_THINKING must match ^[a-z_]+$ ..." >&2
    exit 1
fi
```

Valid values are lowercase letters and underscores (e.g. `extra_high`, `high`,
`medium`, `low`). Any other value fails fast at daemon startup with a clear error
instead of silently breaking the codex exec call.
