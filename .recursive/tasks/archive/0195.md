# Fix remaining python3 -c shell variable interpolation in daemon.sh

## Context

Pentest scan (2026-04-07) found two `python3 -c` calls at
`Recursive/engine/daemon.sh` lines 136-137 that use shell variable expansion
inside Python string literals:

```bash
python3 -c "import json; c=json.load(open('$config_file')); ..."
```

If `$config_file` contains a single quote (e.g., `/tmp/test'repo/.recursive.json`),
the Python string boundary breaks -> SyntaxError.

These two calls were missed in the comprehensive python3 -c audit (PR #176).
The mandated pattern (per learnings) is to pass values via environment variables.

## Reproduction

```bash
config_file="/tmp/test'repo/.recursive.json"
python3 -c "import json; c=json.load(open('$config_file'))"
# -> SyntaxError: unterminated string literal
```

## Fix

Pass `$config_file` via env var:

```bash
_RECURSIVE_CONFIG="$config_file" python3 -c "
import json, os
c = json.load(open(os.environ['_RECURSIVE_CONFIG']))
print(c.get('project',{}).get('name',''))
" 2>/dev/null || basename "$REPO_DIR"
```

Apply the same pattern to both calls at lines 136 and 137.
