# Wire feature summary into CLI output and shift log

The feature summary module (#0059) generates structured summaries, but the data is only stored in the state file. It should also be:

1. Printed to stdout when `nightshift build` completes (already partially done via `format_feature_status`)
2. Available as a standalone command: `nightshift build --status` already shows the summary section
3. Included in the feature build's log directory as `summary.md` for easy human review

## Acceptance Criteria
- [ ] `nightshift build --status` includes the summary section in output
- [ ] A `summary.md` file is written to the feature log directory after build completion
- [ ] The markdown summary is suitable for pasting into changelogs or handoffs
- [ ] Tests verify summary.md is written and contains expected sections
- [ ] `make check` passes
