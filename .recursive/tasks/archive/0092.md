# Score calibration against known-good and known-bad shifts

The 10 evaluation dimension scorers use heuristic thresholds (e.g., "3+ issues found = 8/10 for Discovery"). These thresholds were set by intuition, not calibrated against real data. Without calibration, the system may flag healthy shifts or miss weak ones.

## Design

1. Collect 2-3 "known-good" shift artifacts (from successful Phractal runs) and 2-3 "known-bad" artifacts (synthetic or from failed runs)
2. Score all artifacts with `score_all_dimensions()`
3. Verify: good artifacts score above threshold (6/10 per dimension), bad ones score below
4. Adjust scorer logic or thresholds where calibration reveals misalignment
5. Store calibration artifacts in `tests/fixtures/evaluation/` for regression testing

## Verification

1. Known-good artifacts score >= 6 on all dimensions
2. Known-bad artifacts score < 6 on at least 3 dimensions
3. Fixture-based tests prevent threshold regression

## Acceptance Criteria

- [ ] At least 2 known-good and 2 known-bad artifact fixtures exist
- [ ] All fixtures are scored by CI tests
- [ ] Any scorer adjustments are documented in the evaluation module
- [ ] Thresholds in `constants.py` are updated if calibration reveals misalignment
