# Module map should surface dependency cycles explicitly

`nightshift.module_map._dependency_order()` currently falls back to
alphabetically appending unresolved nodes when it hits a cycle. That keeps the
command running, but it hides the fact that the package dependency graph is no
longer healthy.

## Acceptance Criteria
- [ ] Generated module-map output explicitly reports when a dependency cycle was detected
- [ ] The fallback order remains deterministic even when a cycle exists
- [ ] Regression tests cover a synthetic package with an internal import cycle
