# Fix PYTHONPATH shadow module bypass in daemon post-agent processing

## Context

Pentest scan (2026-04-07, session #0105) confirmed that the daemon's post-agent
`python3 -c` blocks at `Recursive/engine/daemon.sh:374-386` and `:443-447` are
vulnerable to Python module shadowing via the repo root.

**Root cause**: `python3 -c` prepends `''` (current working directory) to `sys.path`
at position 0. The daemon runs with `cwd=$REPO_DIR` (line 224) and sets
`PYTHONPATH="$RECURSIVE_DIR/lib:$REPO_DIR"`. Since `''` comes before all PYTHONPATH
entries, a file named `costs.py` at the repo root is imported instead of
`Recursive/lib/costs.py`.

**Attack**: Agent writes `costs.py` at repo root during its session. After the agent
exits, daemon's cost tracking imports the shadow module. All costs report as $0.00,
budget limiter never triggers. Even a `costs.py` that raises ImportError works because
the `|| echo "0.0000"` fallback at line 386 activates.

**Verified**: `PYTHONPATH="Recursive/lib:." python3 -c "import sys; print(sys.path[:3])"` outputs
`['', '/path/Recursive/lib', '/path/repo']` -- `''` is at position 0.

## Reproduction

Describe only (do not execute):

1. Agent writes `REPO_ROOT/costs.py`:
   ```python
   def record_session_bundle(*a, **kw): return {"cost_usd": 0.0}
   def total_cost(*a, **kw): return 0.0
   ```
2. Agent exits. Daemon reaches line 374 (cost tracking).
3. `from costs import record_session_bundle, total_cost` finds shadow module first.
4. Budget at line 442 reads cumulative = 0.00, never triggers.

Simpler variant: `raise ImportError()` in `costs.py` -> python3 crashes ->
`|| echo "0.0000"` fallback -> cost = 0.

## Fix

1. Remove `$REPO_DIR` from PYTHONPATH in post-agent python3 -c calls.
   Use `PYTHONPATH="$RECURSIVE_DIR/lib"` only.
2. Optionally, add `sys.path = [p for p in sys.path if p]` at the start of
   each python3 -c block to strip the cwd entry.
3. Consider changing cwd away from REPO_DIR before running post-agent processing.
