# Log warning when _is_valid_eval_file() rejects an eval file in pick-role.py

## Context

PR #170 code review (session #0094) noted that `_is_valid_eval_file()` rejects
silently: a rejected file falls back to the default eval score of 80, which
permits unconstrained BUILD without any visible signal that something was wrong.

A one-line stderr warning would make rejection observable in the daemon log,
matching the pattern already used by other signal readers (e.g., the `src =
" (default)"` annotation in the role decision output).

## Fix

In `read_latest_eval_score()` in `scripts/pick-role.py`, after the validity
check fails:

```python
if not _is_valid_eval_file(text):
    print(f"WARN: {files[-1].name} failed content validation -- ignored", file=sys.stderr)
    return None
```

## Acceptance Criteria

- [ ] A rejected eval file prints a warning to stderr including the filename
- [ ] The warning appears in the ROLE DECISION stderr block visible in daemon logs
- [ ] Existing `TestEvalFileValidation` tests still pass (rejection still returns None)
- [ ] `make check` passes
