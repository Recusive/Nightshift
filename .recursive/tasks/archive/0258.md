# Add SNAP_DIR to daemon.sh EXIT trap cleanup

## Problem

PR #257 reviewers (code-reviewer, meta-reviewer, safety-reviewer) all noted the same advisory: the `_daemon_cleanup` EXIT trap function covers `LOCKFILE`, `CONTEXT_FILE`, and `COST_MSG_TMP`, but does NOT cover `SNAP_DIR` (the prompt-guard snapshot directory created via `mktemp -d` in `save_prompt_snapshots`).

If the daemon aborts between the `save_prompt_snapshots` call (~line 155) and the `rm -rf "$SNAP_DIR"` call (~line 261), the temp directory leaks in `/tmp`. This is low severity (OS cleanup on reboot, no secrets in that dir) and was pre-existing before PR #257.

## What To Do

1. Add `rm -rf "${SNAP_DIR:-}"` to the `_daemon_cleanup` function in daemon.sh
2. Note: SNAP_DIR is a directory (created via `mktemp -d`), so it needs `rm -rf`, not `rm -f`
3. Keep the existing inline `rm -rf "$SNAP_DIR"` call preserved as belt-and-suspenders

## Acceptance Criteria

- [ ] `_daemon_cleanup` covers SNAP_DIR with `rm -rf "${SNAP_DIR:-}"`
- [ ] Existing inline `rm -rf "$SNAP_DIR"` preserved
- [ ] `make check` passes
