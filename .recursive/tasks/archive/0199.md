# Add Recursive/lib to PROMPT_GUARD_DIRS

Already done. `.recursive/lib` is listed in the `PROMPT_GUARD_DIRS` array in `.recursive/engine/lib-agent.sh`. Confirmed during oversee triage 2026-04-09.

## Context

Pentest scan (2026-04-07, session #0105) identified that `Recursive/lib/` is not
monitored by the prompt guard system. Files in this directory (`costs.py`,
`cleanup.py`, `compact.py`, `config.py`, `evaluation.py`) are imported by daemon
post-agent python3 -c calls.

An agent could modify these files during a session. While `git reset --hard` at the
next cycle start restores them, within-cycle modifications affect post-agent
processing (cost tracking, budget enforcement, etc.).

**Severity**: THEORETICAL -- requires the agent to modify files in a guarded
framework directory (Recursive/), which the autonomous.md prompt prohibits for
target operators. But prompts are advisory, not enforcement.

## Fix

Add `"Recursive/lib"` to the `PROMPT_GUARD_DIRS` array in
`Recursive/engine/lib-agent.sh` (around line 46).
