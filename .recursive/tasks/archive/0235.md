# Fix typing in test_score_calibration.py fixture loader

Code-reviewer noted that `_load_fixture()` annotates `raw` as `dict[str, object]` but passes `raw.get("state")` (which resolves to `object | None`) to `ShiftArtifacts(state=...)` which expects `dict[str, object] | None`. Pyright flags this as a type error.

The fix: annotate `raw` as `dict[str, Any]` (with `from typing import Any`) since this is a JSON deserialization boundary, which aligns with the project's stated policy: "Any only at JSON deserialization boundaries." Also correct the docstring claim about typing.
