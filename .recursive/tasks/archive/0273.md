# Guard int() calls in _build_state counters block

## Problem

PR #267 added `isinstance(v, (int, float))` guards to the `category_counts` comprehension in `state.py` `_build_state()`. However, the `counters` block at lines 54-61 uses bare `int(counters_raw.get(..., 0))` without similar type guards. A corrupt state file with `"fixes": "bad"` would raise `ValueError` at load time.

This was flagged as an advisory note by the safety reviewer on PR #267.

## Fix

Apply the same `isinstance` guard pattern to each counter field in the `counters` block. For each `int(counters_raw.get(key, 0))` call, wrap it so non-numeric values fall back to 0 instead of crashing:

```python
def _safe_int(val: object, default: int = 0) -> int:
    """Convert to int, falling back to default for non-numeric values."""
    if isinstance(val, (int, float)) and not isinstance(val, bool):
        return int(val)
    return default
```

Or use inline `isinstance` checks consistent with the category_counts pattern.
