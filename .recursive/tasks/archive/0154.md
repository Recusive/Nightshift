# Fix `local origin_rc=$?` outside function in daemon.sh

## Problem

PR #143 (a68ba4c) introduced `local origin_rc=$?` at two sites in the main
`while true` loop of `scripts/daemon.sh` (lines 233 and 313). These are NOT
inside any function -- the only functions in daemon.sh are `cleanup()`,
`pick_session_role()`, `build_prompt()`, `build_pentest_prompt()`, and
`reset_repo_state()`.

On bash 3.2 (the only bash on macOS), `local` outside a function:
1. Prints "local: can only be used in a function" to stderr
2. Does NOT set the variable
3. With `set -u` (line 15), the subsequent `if [ "$origin_rc" -eq 1 ]` triggers
   "origin_rc: unbound variable" and exits the shell

The daemon cannot complete any cycle. No cycle has run since PR #143 was merged.

## Fix

Replace `local origin_rc=$?` with `origin_rc=$?` at both sites (lines 233 and 313).

- [ ] `origin_rc=$?` (not `local`) at both pentest-preflight and post-builder guard sites
- [ ] `/bin/bash -c 'set -uo pipefail; true; origin_rc=$?; echo $origin_rc'` exits 0
- [ ] `bash -n scripts/daemon.sh` passes
