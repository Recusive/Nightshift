# Sub-agent coordination module for Loop 2

Loop 2 "Sub-agent coordination" is at 0% in the tracker with no existing task. When `nightshift build` spawns multiple sub-agents for a wave, there is no coordination layer: agents can conflict on shared files, produce incompatible changes, or step on each other's work.

Build a `nightshift/coordination.py` module that manages sub-agent execution within a wave:
1. Assign non-overlapping file sets to each sub-agent based on the task decomposition
2. Detect merge conflicts when collecting sub-agent outputs before integration
3. Provide a retry/reassign mechanism when a sub-agent fails its task
4. Track per-agent status (pending, running, succeeded, failed) for observability

Wire into `build_feature()` between task decomposition and wave integration.
