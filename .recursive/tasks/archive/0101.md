# Evaluation #0001: preserve rejected-cycle findings in human-readable output

The evaluated run produced real fixes and logged issues, but because both cycles were rejected the day-team-facing shift log stayed a template file. The useful findings only survived in the runner log and nested state JSON, which is too buried for routine review.

## Acceptance Criteria
- [ ] Rejected cycles still surface their discovered fixes/issues in a durable human-readable artifact
- [ ] The artifact distinguishes accepted fixes from rejected findings without pretending rejected work shipped
- [ ] Re-running the Phractal evaluation leaves a readable summary without requiring the reviewer to inspect raw runner logs

## Resolution

Completed on 2026-04-06 by writing a test-mode runtime artifact
(`$TMPDIR/nightshift-test-runs/.../YYYY-MM-DD.md`) before rejected cycles are
reverted, and by preserving Claude summary/count fields as readable notes so
the artifact still explains what the rejected cycle found.
