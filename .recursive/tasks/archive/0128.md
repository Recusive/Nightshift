# Open PR from previous session requires full review before merge

Superseded by v2 brain architecture. In v2, the brain agent (not the build sub-agent) handles open PR recovery at the start of each cycle. `brain.md` line ~39 already contains: "Open PR recovery: If the `<open_prs>` block shows PRs from previous sessions, process them BEFORE starting new work. Review framework-zone PRs first (higher risk), then project-zone PRs. For each: read the diff, run appropriate reviewers, merge or close." The rule exists at the right layer (brain orchestration, not build SKILL.md). Task was filed for the v1 daemon where the builder was the only agent.

## Problem

When the daemon finds an open PR from a previous session, it auto-merges after checking CI. No diff review, no review panel. This bypasses the quality gate.

## Fix

Add to `Recursive/operators/build/SKILL.md` or `Recursive/prompts/autonomous.md`:

> OPEN PR RULE: When you find an open PR from a previous session, do NOT auto-merge. First: read the full diff with `gh pr diff`, run `make check` on the branch, and verify CI passes. If the PR touches `Recursive/engine/`, `Recursive/operators/`, or `nightshift/` core modules, run the full review panel before merging. Only fast-merge if it is a docs-only change with < 20 lines changed.

## Acceptance Criteria

- [ ] Rule added to the appropriate operator/prompt file
- [ ] Agent reads diff before merging open PRs
- [ ] Non-trivial PRs get review panel
- [ ] Docs-only PRs can fast-merge
