# Add symlink check before rmtree in eval_runner clone cleanup

`run_eval_full()` in `nightshift/owl/eval_runner.py` calls `shutil.rmtree()` on `EVALUATION_CLONE_DEST` (a fixed `/tmp/nightshift-eval` path) without checking if the path is a symlink first. On shared CI infrastructure, a malicious local user could pre-create a symlink at that path targeting an arbitrary directory, and the rmtree would follow it.

Fix: check `clone_dest.is_symlink()` before calling `shutil.rmtree()`. If it's a symlink, refuse to delete and raise an error.

Source: safety-reviewer advisory note on PR #231.
