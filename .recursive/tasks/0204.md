# Add commitment tracking to brain sessions (commitment log is empty)

## Problem

The commitments/log.md file exists but contains only the header -- it has never
been written to in 76 sessions. The brain.md instructions say to append to this
file at the end of every session with a Prediction/Actual/Result entry. This is
not happening.

This means:
1. The commitment hit rate is unmeasurable (no data)
2. The brain has no feedback loop on prediction accuracy
3. Checkpoint 3 (Pre-Commitment Metric) and Checkpoint 4 (Commitment Check) are
   effectively dead -- they happen in the brain's thinking but are never persisted

The decisions/log.md also has only one entry (the spike test from 2026-04-07).

## Root Cause

The v2 brain architecture was only recently deployed (first real session 2026-04-07).
However, the brain.md instructions for the Session Report section include commitment
logging and should be working. This needs investigation.

Possible causes:
- Brain is writing commitments but they're not being committed to main (selective
  git add may be missing the commitments/ dir -- verified it IS in daemon.sh)
- Brain is not completing the full Session Report section
- Output truncation is preventing the end-of-session writes
