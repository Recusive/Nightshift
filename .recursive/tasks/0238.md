# Sanitize exit_code and duration_min in append_session_index_row

Code-reviewer and safety-reviewer noted that `exit_code` and `duration_min` are not sanitized inside `append_session_index_row()` in lib-agent.sh. Both are provably integers from current call sites (PIPESTATUS and arithmetic), but the function is not safe-by-construction if a future caller passes arbitrary strings.

Add either:
- `tr -d '|\n\r' | head -c 8` on both fields (consistent with other fields), or
- `[[ "$exit_code" =~ ^[0-9]+$ ]]` guard with fallback

## Acceptance Criteria

- [ ] `exit_code` and `duration_min` sanitized inside the function
- [ ] Existing behavior unchanged for integer inputs
- [ ] `make check` passes
