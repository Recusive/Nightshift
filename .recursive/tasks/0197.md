# Validate cost_usd >= 0 in costs.py to prevent budget counter poisoning

## Context

Pentest scan (2026-04-07, THEORETICAL): `costs.json` survives `git clean -fd`
and `reset_repo_state` (gitignored file). Once the budget limiter is fixed
(task #0194), an agent could pre-write entries with negative `cost_usd` values
to suppress the budget counter.

Mitigation: requires direct file write to `.recursive/sessions/costs.json`
which is outside the normal agent workflow. Low probability.

## Acceptance Criteria

- [ ] `total_cost()` in `Recursive/lib/costs.py` skips or clamps entries where
      `cost_usd < 0`
- [ ] `record_session_bundle()` validates that computed cost is non-negative
- [ ] Test covers a ledger with negative entries and verifies they are excluded
      from the total
- [ ] `make check` passes
