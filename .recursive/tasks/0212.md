# Move _SHELL_METACHAR_RE to core/constants.py

## Context

Code review of PR #202 noted that `_SHELL_METACHAR_RE` (a compiled regex pattern) lives in
`nightshift/core/shell.py` as a private module-level constant. CLAUDE.md convention says
"Regex patterns ... go in `core/constants.py`". Since it is private (`_` prefix) and only
used by `validate_verify_command`, this is borderline acceptable, but strict convention
compliance says it should be in `constants.py`.

## Acceptance Criteria

- [ ] Move `_SHELL_METACHAR_RE` to `nightshift/core/constants.py`
- [ ] Import it in `nightshift/core/shell.py`
