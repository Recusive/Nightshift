# Make Claude eval reruns scorable from Claude Code sessions

## Problem

The fresh Phractal eval rerun for `#0020` could not complete in this environment because the spawned `claude` subprocess failed inside an existing Claude Code session. Unsetting `CLAUDECODE` was not enough to produce a scorable 2-cycle run.

## Fix

Make the eval runner or launch path resilient to Claude-in-Claude execution so a default `nightshift test --agent claude --cycles 2 --cycle-minutes 5` rerun can complete and produce a report without manual shell workarounds.

## Acceptance Criteria

- [ ] `nightshift test --agent claude --cycles 2 --cycle-minutes 5` can complete from a Claude Code shell without the nested-session failure
- [ ] If Claude-in-Claude remains unsupported, the runner automatically falls back to a supported agent or emits a clear, actionable failure before starting cycles
- [ ] Regression coverage exists for the nested-session path
- [ ] A fresh eval rerun produces a scorable report instead of halting after two agent failures
