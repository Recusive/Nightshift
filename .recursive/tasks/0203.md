# Update ROLE-SCORING.md to reflect v2 brain-delegates-to-sub-agents architecture

## Problem

ROLE-SCORING.md is a v1 document. It still instructs the agent to:
1. Run PHASE 1: ASSESS (read files manually)
2. Run PHASE 2: DECIDE (compute scores manually using a formula)
3. Run PHASE 3: EXECUTE (read the operator SKILL.md directly)

In v2, the brain is a separate Opus agent that reads the dashboard (pre-computed
signals) and the advisory recommendation (pre-computed scoring). The brain does NOT
manually compute role scores -- pick-role.py does that and outputs an advisory.
The brain's 4 checkpoints (Signal Analysis, Forced Tradeoff, Pre-Commitment,
Commitment Check) are defined in brain.md, not ROLE-SCORING.md.

ROLE-SCORING.md currently confuses anyone reading it into thinking they should
manually implement a scoring algorithm, when in v2 that work is done by pick-role.py
and the brain reads the result as advisory context.

## Acceptance Criteria
- [ ] ROLE-SCORING.md header clarifies it describes the advisory scoring algorithm used by pick-role.py, not a manual process for the brain
- [ ] PHASE 1/2/3 structure is removed or clearly labeled as v1-only
- [ ] All 8 roles (including security-check, evolve, audit) have scoring rules documented
- [ ] The examples section is updated to show v2 brain decision-making
- [ ] No references to EXECUTING ROLE: or reading SKILL.md directly
