# Dynamic PROMPT_GUARD_FILES expansion for .github/workflows

Currently `PROMPT_GUARD_FILES` in `lib-agent.sh` lists each workflow file explicitly (`ci.yml`, `notify-orbitweb.yml`). Any future workflow file added to `.github/workflows/` would not be individually diffed by `check_prompt_integrity` — only new-file creation is detected via `PROMPT_GUARD_DIRS`. This means a modification to a newly-added workflow file would go undetected until it was added to `PROMPT_GUARD_FILES` manually.

## Acceptance Criteria
- [ ] All `*.yml` files under `.github/workflows/` are guarded without manual `PROMPT_GUARD_FILES` updates
- [ ] Either: enumerate all current files dynamically at snapshot time, OR implement glob expansion in `save_prompt_snapshots`
- [ ] Existing explicit entries (`ci.yml`, `notify-orbitweb.yml`) remain guarded
