# Fix session-id zero-padding in test_module_map.py helper

## Context

PR #241 fixed the date column zero-padding in `_make_session_index` (`f"2026-04-{i+1:02d}"`). The code reviewer noted the session-id column on the same line has the identical defect: `f"2026040{i+1}-000000"` produces `202604010-000000` at `row_count >= 10`.

This is currently harmless because:
1. Only `row_count=3` is used in tests
2. The parsing logic only checks `cols[2]` for `"CIRCUIT-BREAK"`, not session-id format

But it should be fixed for consistency and future safety.

## Fix

In `nightshift/tests/test_module_map.py`, in `_make_session_index`, change the session-id column from `f"2026040{i+1}-000000"` to `f"2026040{i+1:01d}-000000"` or use a proper zero-padded format like `f"20260{4:01d}{i+1:02d}-000000"`.

## Acceptance Criteria

- [ ] Session-id column in test helper produces valid format for any row_count
- [ ] `make check` passes
