# Fix worktree cleanup: remove .claude/worktrees/agent-* after every session

## Problem

After daemon sessions, `.claude/worktrees/agent-*` directories accumulate in the repo and appear as untracked files in `git status`. The daemon's existing `cleanup_worktrees` call does not catch all cases. Human had to manually run `git worktree remove` after sessions (issue #0223). The current working directory for strategize agent session #0123 is `.claude/worktrees/agent-a4cb88b7`, confirming the leak is live.

## Fix

1. In `daemon.sh` or `lib-agent.sh`, add a post-session cleanup step that enumerates all `git worktree list` entries matching `.claude/worktrees/agent-*` and removes each with `git worktree remove --force`.
2. Add `git worktree prune` after the removal loop.
3. In the housekeeping step at the start of each cycle, also clean up stale agent-* worktrees left by crashed sessions.

This is a framework fix (engine zone). Must be delegated to evolve, not build.

## Acceptance Criteria

- After a clean daemon session, `git worktree list` shows no `.claude/worktrees/agent-*` entries
- After a crashed session, next cycle's housekeeping removes the stale worktree
- `git worktree prune` runs after cleanup
- `git status` shows no `?? .claude/worktrees/` untracked entry
