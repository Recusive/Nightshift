# Fix misleading comment in test_parse_error_includes_subpackage_context

## Problem

In `nightshift/tests/test_module_map.py`, the test `test_parse_error_includes_subpackage_context` has a misleading inline comment on the `"broken.py"` assertion (lines 168-172). The comment says "Top-level error uses bare filename (package_dir=None path in _parse_modules is never reached here since package_dir is always provided)" which is internally contradictory.

The real reason `"broken.py"` appears is that `path.relative_to(package_dir)` for `nightshift/broken.py` relative to `nightshift/` yields `broken.py` -- the same string as a bare filename. The `package_dir=None` branch is never exercised by `generate_module_map`.

## What To Do

1. Simplify the comment to explain that the relative path of a top-level file already looks like a bare filename
2. Remove the confusing reference to the `package_dir=None` code path
3. make check passes
