# Decide on category_counts accuracy for multi-fix count-only cycles

## Problem

PR #262 code reviewer noted: When the agent returns a count-only payload with `fixes_count_only=3` and `categories=["Security"]`, `category_counts["Security"]` becomes 1 (not 3). Each category in the list gets counted once, regardless of how many fixes ran. This is a known simplification because the agent doesn't report per-fix category breakdowns in count-only mode.

## Decision Needed

Either:
1. Accept the simplification (document it) -- category_counts is approximate in count-only mode
2. Use `fixes_count_only` to scale the count when there's a single category
