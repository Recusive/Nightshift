# Add category string allowlist validation in state.py

## Problem

PR #262 safety reviewer noted (pre-existing gap): `category_counts` dict accepts any non-empty string as a key. An adversarial agent could inject arbitrary category strings. While low-severity (agent runs in isolated worktree), validating against `CATEGORY_ORDER` from constants.py before writing to `category_counts` would be a defense-in-depth improvement.

## Fix

In `nightshift/core/state.py`, validate category strings against `CATEGORY_ORDER` before inserting into `category_counts`. Unknown categories should be mapped to a catch-all like "Other" or silently skipped.

## Acceptance Criteria

- [ ] Category strings validated against CATEGORY_ORDER in state.py
- [ ] Unknown categories handled (skip or map to "Other")
- [ ] Test for unknown category added
- [ ] `make check` passes
