# Fix cost classifier to recognize role-based session types in cost_analysis()

## Context

**Path update (2026-04-07):** costs.py moved to `Recursive/lib/costs.py`.

`cost_analysis()` in `Recursive/lib/costs.py` classifies sessions by
branch/commit prefix (feat/, fix/, docs/, test/, overseer). Daemon sessions
run by role (build, review, achieve, oversee, strategize) produce feature
strings like "success (pentest: success) [PROMPT MODIFIED]" that do not
match any branch prefix. These fall into "unknown" at $25.40/session
average — the most expensive bucket with zero visibility.

From `cost_analysis('.recursive/sessions')`:
- unknown: 21 sessions, $534 total, $25.40/session average
- fix: 8 sessions, $4.49/session
- test: 32 sessions, $1.81/session

The two biggest outliers are both role-based sessions: 20260406-103855
(overseer, $27.89, 28.7x peer average) and 20260406-124154 (build, $34.37,
5.4x peer average). Cost intelligence cannot surface these outliers because
the classifier lumps them into "unknown."

## Fix

In `nightshift/costs.py`, add a fallback classification step: when the
branch/commit prefix produces "unknown", check the Role column from the
session index row and use it as the task_type:
- "build" -> "build"
- "review" -> "review"
- "achieve" -> "achieve"
- "oversee" -> "oversee"
- "strategize" -> "strategize"

The session index already has a Role column (column 3 in the markdown table,
parsed by `parse_session_index()` in `pick-role.py`). The cost module should
read this column as a fallback when no branch prefix matches.

## Acceptance Criteria

- [ ] `cost_analysis()` returns role-named task types for daemon sessions
      that previously fell into "unknown"
- [ ] "unknown" count drops from 21 to near 0 for sessions with a Role

      strategize)
- [ ] `cost_per_test_added_usd` and `cost_per_tracker_delta_usd` are
      computable for build sessions that were previously "unknown"
