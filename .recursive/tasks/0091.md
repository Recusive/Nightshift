# Evaluation dry-run CLI command

The evaluation system (`nightshift/evaluation.py`) runs automatically via the daemon every N sessions, but there is no way to trigger it manually from the CLI. Developers and the agent should be able to run `nightshift eval --dry-run` to test the evaluation pipeline without cloning repos or running shifts.

## Design

Add an `eval` subcommand to the CLI that:
1. `nightshift eval` -- runs a full evaluation against the configured target repo
2. `nightshift eval --dry-run` -- runs evaluation with synthetic artifacts (no clone, no shift)
3. Reports scores to stdout and optionally writes to `.recursive/evaluations/`

## Verification

1. `nightshift eval --dry-run` prints scores for all 10 dimensions
2. `nightshift eval --dry-run` exits 0 without network access
3. `nightshift eval` (full mode) clones, runs, scores, writes report

## Acceptance Criteria

- [ ] `eval` subcommand added to CLI
- [ ] `--dry-run` flag generates synthetic artifacts and scores them
- [ ] Full mode delegates to `evaluate()` from `evaluation.py`
- [ ] Output is human-readable (dimension table with scores)
- [ ] Tests cover both dry-run and CLI argument parsing
