# Guard _all_tasks_done against vacuous truth on empty list

## Context

Code review advisory note (PR #218): `_all_tasks_done([])` returns `(True, [])` which is
vacuous truth. Currently `_version_ready` guards against this with `if not task_files:` check
before calling `_all_tasks_done`, so no live bug exists. However, the vacuous truth is encoded
as a test contract. If the guard is ever removed, it would silently permit releasing a version
with zero tasks.

## Acceptance Criteria

- [ ] Add a docstring warning in `_all_tasks_done` about empty list behavior, OR
- [ ] Raise `ValueError` inside `_all_tasks_done` for empty list
- [ ] Update the test `test_empty_file_list_returns_true` accordingly
- [ ] `make check` passes
