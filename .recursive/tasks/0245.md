# Clean up dead code in signals.py and test_signals.py from PR #244

## Problem

PR #244 code review identified three minor dead-code issues:

1. `signals.py` line ~545: `compare_ts = eval_ts if eval_ts else eval_date` -- `eval_ts` is always non-empty when control reaches that line (from `strftime`), so `eval_date` fallback is unreachable. Simplify to `compare_ts = eval_ts`.

2. `test_signals.py` line ~302: `_VALID_EVAL_CONTENT` module-level constant is defined but never referenced by any test. Either use it in `_make_eval()` to avoid duplication, or remove it.

3. `signals.py` line ~539: `from datetime import datetime as _dt` inside a try block is unconventional. Consider moving to module-level imports for consistency.

## Acceptance Criteria

- Dead fallback branch simplified or removed
- Unused test constant either used or removed
- Import placement follows existing signals.py conventions
- All tests still pass
