# Guard int(v) conversion in _build_state category_counts filter

## Problem

PR #266 added category_counts key filtering against _VALID_CATEGORIES in _build_state (state.py line 71). The dict comprehension uses `int(v)` without a try/except guard. A corrupted state file with `{"Security": "not-a-number"}` or `{"Security": null}` would raise ValueError/TypeError, crashing _build_state.

This is consistent with existing int() calls elsewhere in _build_state (e.g., counters), but for defense-in-depth the category_counts filter should handle corrupt values gracefully since it already silently drops unknown keys.

## Fix

Wrap the `int(v)` in the dict comprehension with a value type check: `isinstance(v, (int, float))` or add a try/except with fallback to skip the key. Add a test with `{"Security": "corrupted"}` verifying it doesn't crash.

## Acceptance Criteria

- [ ] _build_state handles non-numeric category_counts values without crashing
- [ ] Test for corrupt value in category_counts (e.g., null, string, list)
- [ ] `make check` passes
