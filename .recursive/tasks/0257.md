# Extend daemon.sh EXIT trap to cover temp files

## Problem

PR #255 reviewers (code-reviewer, meta-reviewer, safety-reviewer) all noted the same advisory: the EXIT trap in daemon.sh (line 72) only registers `$LOCKFILE` for cleanup. Temp files created with `mktemp` (`COST_MSG_TMP` at line 254, `CONTEXT_FILE` at line 148) are cleaned up with inline `rm -f` calls but are NOT covered by the trap.

Under `set -euo pipefail`, if a future edit inserts an unguarded command between a `mktemp` and its `rm -f`, the temp file would be orphaned on abort.

## What To Do

1. Extend the EXIT trap to a general-purpose cleanup function that removes `$LOCKFILE` and any temp files created during the session
2. Register each `mktemp` variable with the cleanup function
3. Keep the existing inline `rm -f` calls as belt-and-suspenders (the trap is the safety net)

## Acceptance Criteria

- [ ] EXIT trap handler covers LOCKFILE, CONTEXT_FILE, and COST_MSG_TMP
- [ ] Trap is robust to variables being unset (use `${VAR:-}` or similar)
- [ ] Existing inline rm -f calls preserved
- [ ] `make check` passes
