# Add unit test for sync_github_tasks stale .next-id advance path

The stale `.next-id` while-loop fix in `lib-agent.sh` (PR #175) has no unit
test. The inline Python in `sync_github_tasks` is not covered by any test. A
regression in the advance logic would be silent.

## Acceptance criteria

- Extract the issue-to-task conversion logic from the inline Python block in
  `sync_github_tasks` into a standalone Python helper (e.g.,
  `nightshift/task_sync.py` or `scripts/task_sync.py`).
- OR add a shell integration test in `tests/` that exercises the stale-advance
  path directly.
- Test must confirm: when `.next-id` points to an already-existing task file,
  the next issue is written to the next available slot (not silently dropped).
- `make check` passes.
