# Session-since counters don't track v2 sub-agent delegations

## Problem

The dashboard shows "78 sessions since evolve/audit/security" even though all three were delegated as sub-agents in recent sessions (#0107-#0111). The brain correctly dismisses these as "stale tracking gap" but the alerts are noise.

The v1 session counter in `sessions/index.md` tracks operator roles, not v2 brain sub-agent delegations. When the brain delegates to evolve, the session gets recorded as "brain" not "evolve" in the index.

## Fix

The daemon should extract which sub-agents were delegated from the brain's output (or from the delegation log) and record them in the session index. Alternatively, the dashboard should read the decisions log to find the actual last time each agent type was delegated.

## Acceptance Criteria

- `sessions_since_evolve` reflects actual evolve sub-agent delegations, not just v1 operator sessions
- Same for audit, security, strategize, oversee, achieve
- Dashboard alerts are accurate, not perpetually "overdue"
