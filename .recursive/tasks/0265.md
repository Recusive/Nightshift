# Add positive-path test for dominance guard with valid category

## Problem

PR #265 added a test verifying unknown categories don't trigger dominance violations. The complementary assertion -- that a valid category with enough fixes DOES trigger a dominance violation -- is not tested. This positive-path test strengthens confidence the allowlist guard didn't accidentally break the underlying dominance check.

## Fix

Add a test to `TestCycleCategoryAllowlist` where 4 fixes tagged with a valid CATEGORY_ORDER member (e.g., "Security") do trigger a dominance violation when total_fixes >= 4 and one category exceeds 50%.
