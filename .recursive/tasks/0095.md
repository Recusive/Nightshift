# Stabilize session index row formatting and feature capture

The last several entries in `docs/sessions/index.md` have blank feature cells and broken multiline table rows, which makes the agent's trend scan unreliable. The session index is supposed to be the quick operational dashboard for speed, cost, and repeated failures.

## Acceptance Criteria

- [ ] Each new session appends exactly one markdown table row to `.recursive/sessions/index.md`
- [ ] The Feature column is always a single-line value (`-` if unknown, never a wrapped row fragment)
- [ ] The PR column is always a single-line value (`-`, `#NN`, or a URL)
- [ ] A validator or test catches multiline/broken table rows before merge
- [ ] Operational docs are updated if the writer or format rules change
