# Auto-changelog entry generation from PR metadata

Self-Maintaining "Auto-changelog update" is at 0% in the tracker. Currently changelog entries are manually written by the builder agent during Step 6c, which is error-prone and time-consuming. The agent reconstructs what it built from memory, risking inaccuracies.

Build a utility function that generates a draft changelog entry from structured data the builder already has:
- PR title and description (from `gh pr create` output)
- Files changed (from `git diff --stat`)
- Test count delta (from pytest output before/after)
- Task number and description (from the task file)

The function takes these inputs and returns a formatted changelog entry following the existing convention (`- **[tag]** Description. (task #NNNN)`).

This does NOT auto-commit the changelog -- it produces a draft that the builder inserts into the version file. The builder retains editorial control.

## Acceptance Criteria
- [ ] Function `draft_changelog_entry()` exists (in a new or existing module)
- [ ] Accepts PR title, files changed, test delta, task reference
- [ ] Returns a formatted string matching existing changelog conventions
- [ ] Builder can call it during Step 6c instead of writing from scratch
- [ ] Tests cover various entry types (feat, fix, refactor, etc.)
- [ ] `make check` passes
