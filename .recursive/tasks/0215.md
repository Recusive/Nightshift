# Add test coverage for pentest signal functions in signals.py

## Context

Meta-reviewer advisory note (PR #205): `count_pending_pentest_framework_tasks()` and
`count_recent_pentest_tasks()` in `.recursive/engine/signals.py` have no test coverage
in `.recursive/tests/test_signals.py`. These functions are security-signal-critical.

Tests should cover:
1. Exact match `source: pentest` is counted
2. Suffixed variants like `source: pentest-runner` are NOT counted (anchoring behavior)
3. CRLF line endings still match
4. Tasks with wrong `target:` are excluded
5. `count_recent_pentest_tasks` respects the date cutoff

## Acceptance Criteria

- [ ] Tests for `count_pending_pentest_framework_tasks()` in test_signals.py
- [ ] Tests for `count_recent_pentest_tasks()` in test_signals.py
- [ ] Edge cases: suffixed values rejected, CRLF handled, date cutoff works
- [ ] `make check` passes
