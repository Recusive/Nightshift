# Auto-clone target repo if missing in `nightshift test` command

## Problem

Eval #0017 required a manual pre-step: `git clone https://github.com/fazxes/Phractal /tmp/nightshift-eval-0017` before the `nightshift test` command could run. Without it, the tool exits with:

```
FileNotFoundError: [Errno 2] No such file or directory: '/private/tmp/nightshift-eval-0017'
```

The task description and documentation imply the command should work end-to-end. The tool knows the eval target URL from `eval_targets.py`.

## What To Do

1. Read `nightshift/cli.py` and `nightshift/infra/worktree.py` to find where `validate_repo_checkout()` is called.
2. Before the validation, check if `repo_dir` exists. If it does not exist, either:
   a. Clone from the configured eval target URL (preferred for the `test` subcommand), or
   b. Print a clear actionable error message with the exact `git clone` command to run.
3. If auto-cloning, use the URL from `eval_targets.py` `PHRACTAL_URL` when available.
4. Add a test verifying the error message includes the clone command when repo_dir is missing.
5. Run `make check` to verify.

## Acceptance Criteria

- `nightshift test --repo-dir /tmp/nonexistent` either auto-clones or prints an actionable error with the exact `git clone` command
- No FileNotFoundError traceback exposed to the user
- `make check` passes
