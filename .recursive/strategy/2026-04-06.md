## Recommendations

### 1. Fix the autonomy score first-match bug (pick-role.py:52)

**Problem**: `re.search()` returns the FIRST `TOTAL:` match in the autonomy
file. Dual-section files (baseline + updated) cause `pick-role.py` to read
the lower baseline score rather than the final updated score. With a 76/100
read vs actual 81/100, ACHIEVE is mildly over-scheduled. If a future
baseline drops below 70, the over-boost is +50 instead of +20 — a 30-point
ACHIEVE inflation that crowds out BUILD cycles.

**Evidence**: `scripts/pick-role.py:52` — `re.search(r"TOTAL:\s*(\d+)\s*/\s*100", text)`;
`docs/autonomy/2026-04-06c.md` has TOTAL: 76 at line 34 and TOTAL: 81 at
line 93. Confirmed via grep in pre-build pentest scan. Handoff #0092
also notes this.

**Prompt refs**: None — this is a code fix, not a prompt issue.

**Fix**: Change `re.search(...)` to `matches = re.findall(...); return int(matches[-1]) if matches else None` in `read_latest_autonomy_score()`.
Add a regression test with a dual-TOTAL file asserting the last match is
returned. 1-line code change + 1 test. New task #0176.

**Impact**: Role picker reads the correct autonomy score. ACHIEVE is not
over-scheduled when autonomy files have baseline + updated sections. Score
accuracy improves for all future ACHIEVE reports.

### 3. Fix cost classifier to recognize role-based session types

**Problem**: `cost_analysis()` classifies sessions by branch/commit prefix
(feat/, fix/, docs/, test/, overseer). Daemon sessions tagged by role
(build, review, achieve, oversee, strategize) produce feature strings like
"success (pentest: success)" that fall into "unknown" at $25.40/session
average — the most expensive bucket with the least visibility.

**Evidence**: `cost_analysis('docs/sessions')` — unknown: 21 sessions,
$25.40 average, $534 total. Top outliers (20260406-103855 at $27.89, 
20260406-124154 at $34.37) are both role-based sessions invisible to
the classifier.

**Prompt refs**: None — code fix.

**Fix**: In `nightshift/costs.py`, add a fallback classification step: when
branch/commit prefix gives "unknown", read the Role column from the session
index row and use it as the task_type (build → build, oversee → oversee,
etc.). Add tests covering each role type mapping. New task #0178.

**Impact**: Cost intelligence covers the full spend. Outlier detection works
on actual BUILD/ACHIEVE sessions. Session #0092 (ACHIEVE, $3.55) would
appear as "achieve" type rather than "unknown," making cost-per-autonomy-
point a computable metric.
