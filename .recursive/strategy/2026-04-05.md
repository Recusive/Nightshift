## Recommendations

### 1. Freeze Prompt Churn And Pay Down Evaluation Debt
**Problem**: Loop 1's real-repo acceptance signal is still unreliable, but the system keeps spending sessions on docs, prompt, and queue hygiene instead of closing the repeated evaluation failures.
**Evidence**: Evaluations `#0001`-`#0005` repeat the same low dimensions; `docs/vision-tracker/TRACKER.md` still keeps Loop 1 at 99%; handoffs `#0045` and `#0046` both re-report the same unresolved evaluation gaps.
**Prompt refs**: `docs/prompt/evolve.md:63-77`, `docs/prompt/evolve-auto.md:7-27`
**Fix**: Create an urgent tranche that forces tasks `#0098`, `#0099`, `#0100`, `#0101`, and `#0102` ahead of remaining low-numbered cleanup until Nightshift produces two consecutive Phractal evaluations at or above 60/100.
**Impact**: The system stops gathering the same evidence and starts restoring a trustworthy acceptance loop.

### 2. Cut Task-Generation Pressure
**Problem**: Mandatory end-of-session task creation is inflating backlog faster than the builder can retire it.
**Evidence**: Active queue state is 50 unresolved tasks; task metadata shows 60 creations vs 24 completions on 2026-04-05; handoff `#0046` still generated `#0104` and `#0105` while 37 `v0.0.8` tasks remained pending.
**Prompt refs**: `docs/prompt/evolve.md:323-353`
**Fix**: Edit Step 6o so it creates `0-2` tasks max, only after a duplicate check and only when unresolved active work is below a ceiling such as 35. Above that ceiling, the builder should tighten or reprioritize existing tasks instead of adding new ones.
**Impact**: The queue becomes prioritizable again, and strategist or overseer recommendations stop getting buried under auto-generated follow-up work.

### 3. Add A Queue-Integrity Gate Before Task Selection
**Problem**: The system treats the queue as authoritative even when task metadata is malformed or incomplete.
**Evidence**: Session `20260405-080655` skipped `#0045` because `## status:` hid it from parsing; active tasks `#0073`, `#0081`, `#0093`, `#0094`, and `#0103` still lack `target`, and 19 active tasks still lack `vision_section`.
**Prompt refs**: `docs/prompt/evolve.md:139-148`, `docs/prompt/evolve-auto.md:7-21`
**Fix**: Add a pre-selection check that scans active tasks for malformed frontmatter or missing required fields. If any are found, the builder must either pick the existing repair task (`#0058`, `#0060`, `#0064`, `#0105`) or create a repair task before normal selection continues.
**Impact**: Queue order becomes truthful, release accounting improves, and vision-alignment logic stops operating on partial metadata.

### 4. Turn The Reviewer Daemon On As A Real Lane
**Problem**: There is no independent quality-review cycle in practice.
**Evidence**: `docs/sessions/index-review.md` reports zero reviewer sessions while 58 PRs have already merged; review work is being absorbed by builder-side review panels and advisory tasks instead of a separate maintenance lane.
**Prompt refs**: none
**Fix**: Create a task that schedules `daemon-review.sh` every N merged PRs or at least daily, and require it to write durable entries to `docs/sessions/index-review.md`.
**Impact**: Code-quality work stops competing directly with feature/build sessions, and the system gains a second independent signal before debt accumulates.
