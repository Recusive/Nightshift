# Triage Rules

## CLOSE: Already Done
Check `git log` and merged PRs. If the work was already shipped in a different PR or task, mark `status: done` with "Superseded by PR #NN / task #NNNN."

## CLOSE: Duplicate
Two tasks describe the same work? Keep the lower-numbered one (builder sees it first), mark the other `status: done` with "Duplicate of #NNNN."

## CLOSE: Wontfix
Mark `status: wontfix` with a specific reason if the task is:
- `priority: low` AND older than 2 weeks AND never picked
- Describes work that is no longer relevant (system changed)
- Auto-generated by builder's "Generate Work" but adds no real value

Every wontfix needs evidence, not just a label.

## KEEP: Reorder
For surviving tasks:
- Ensure priorities are correct (security/reliability = urgent, polish = low)
- Ensure version targeting matches current version
- Fix any malformed frontmatter

## KEEP: Decompose
Large tasks being avoided (pending 10+ sessions)? Break into 2-3 smaller subtasks the builder can complete in one session.
